## Description



**What happened**:

I found the performance bug that we list all workloads here: https://github.com/kubernetes-sigs/kueue/blob/fc238ea3383d724ad9640c796736cc18edb88ce8/pkg/controller/tas/node_controller.go#L311-L314

This may take really long on large clusters hosting 20k workloads, because the List operation calls DeepCopy for each object, even if listed from memory. 

We had similar issues in the past for Nodes and Pods: https://github.com/kubernetes-sigs/kueue/issues/9337

**What you expected to happen**:

We need to have much more scoped filter for the Workloads. With a dedicated indexer we could list only the workloads which have TAS Topology assignment (currently running or admitted) which would already be a good filter. An additional filter is to filter by node.
