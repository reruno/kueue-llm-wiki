## Description

**What would you like to be added**:
1. Kueueviz lacks resource usage reporting even thought the backend exposes (partially) this information. It is nice to have this info to get a high level overview of Cohort/Queue resource utilization: nominalQuota + Borrowing. 
2. The tables in kueueviz don't enforce specific sorting. Due to auto refresh, the rows keep shifting place. This makes them hard to read/inspect, especially when there more than 10 items in the table.
3. The resource flavor details page lists the nodes matching the flavor. But in a large cluster with lots of labels on the nodes, the table explodes in size and becomes hard to read. 

**Why is this needed**:

Quality of life improvements. Better UX.
