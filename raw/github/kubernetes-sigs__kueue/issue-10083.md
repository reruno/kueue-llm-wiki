## Description

<!-- Please only use this template for submitting enhancement requests -->

**What would you like to be added**:
Ensure that MK controller places MultiKueue specific labels on worker Pods running under MK Workloads.
This issue is focused on MK **worker** clusters. For MK manager clusters, see #10093.

**Why is this needed**:
The goal is to make MultiKueue usage observable. Initial research shows that there are no existing MK-specific labels, unlike, for example, for Kueue. Currently, the Kueue usage can be observed thanks to placing Kueue-specific labels on the Pods, for example `kueue.x-k8s.io/podset=main`. Similar labels can be placed on worker Pods for MK, for example: `kueue.x-k8s.io/multikueue-worker-workload=true`.
