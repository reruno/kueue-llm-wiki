## Description

**What would you like to be added**:

A ClusterQueue on a MultiKueue manager cluster should track the **resource usage** (on the worker side) of workers submitted to it, and present a summary of those to the user through the manager cluster API, without a need for the user to log in to all the workers.

A nice addition would be also to track the **quotas** on the worker side, to let users compare the usage against the quotas.
(Note: this part is related to #9988, though distinct from it in *not* involving a customizable multiplier).

**Why is this needed**:

MultiKueue users often would like to treat the manager cluster as a single "management panel", without needing to log in to the worker cluster (except for when troubleshooting). On the other hand, understanding the picture of resource usage vs. capacity is central to using Kueue.
