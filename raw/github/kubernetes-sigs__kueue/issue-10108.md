## Description

**What would you like to be added:**
When a DRA driver publishes devices with `SharedCounters` and `ConsumesCounters` fields ([KEP-4815](https://github.com/kubernetes/enhancements/issues/4815)), the kube-scheduler tracks counter consumption to prevent conflicting device allocations. Kueue needs counter-based quota management for these partitionable devices instead of device-count quota. For example, an NVIDIA H100 GPU can be partitioned into MIG profiles (1g.10gb, 2g.20gb, 7g.80gb) that consume different amounts of GPU memory and compute slices. Device-count quota treats all profiles equally, while counter-based quota accurately reflects that a 7g.80gb profile consumes 8x more GPU memory than a 1g.10gb profile.

**Why is this needed:**

GPU partitioning via MIG is a key use case for maximizing GPU utilization. Without counter-based quota, admins cannot enforce fair sharing across teams using different MIG profile sizes. A team using four 1g.10gb profiles (40Gi total memory) would consume the same quota as a team using four 7g.80gb profiles (320Gi total memory), even though the latter uses 8x more physical resources.
