## Description

**What happened**:
Kueue selects node with existing Pod not managed by Kueue. The existing Pod is already using all the GPUs on that node. As a result, new pod remains in Pending state 

**What you expected to happen**:
Only node which has requested amount of GPU resources should be selected.

**How to reproduce it (as minimally and precisely as possible)**:
1. Cordon all nodes except 21. 
2. Deploy Statefulset POD which requests one full node.  This Statefulset is not managed by Kueue
3.  Deploy Statefulset POD which requests all GPU resources to a node.  This Statefulset managed by Kueue

Observed: Kueue will sometimes choose the node which is running Pod from Step 2

### Comment by [@mimowo](https://github.com/mimowo) — 2026-03-26T06:10:21Z

@rduser thank you for the issue. Actually, we have a mechanism which accounts for non-TAS pods in principle. This is using the nonTasUsageCache, and populates the cache based on non-TAS Pod events, see [here](https://github.com/kubernetes-sigs/kueue/blob/main/pkg/controller/tas/non_tas_usage_controller.go#L74).

So I'm wondering maybe the issue is only if (2.) and (3.) are immediately one after another before the mechanism has a chance to react?

If the issue is indeed with immediate creation of non-TAS and TAS pods then it should be rare, and is hard to prevent, so one mitigation suggestion for such cases is to configure waitForPodsReady timeout which will recycle (evict and re-admit) the TAS workload..

### Comment by [@mimowo](https://github.com/mimowo) — 2026-03-26T07:02:27Z

Please double check the non-TAS pods information which might be relevant for our non-TAS controller: deletionTimestamp, spec.NodeName, status.phase.

If there is no deletionTimetam, the nodeName is set, and status.phase is Running, then I suspect some cache corruption, but I haven't seen that case in practice yet. If we suspect that, then try to restart Kueue, and see if the issue goes away. This will not solve the problem, but will confirm that the problem was due to cache corruption so that we can focus on investigating how this could have happened.
