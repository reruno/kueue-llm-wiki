## Description

**What happened**:

Suppose the following structure

```
- Root-Cohort
  - Cohort-A (10 CPUs)
    - CQ-A
  - Cohort-B
    - CQ-B (5 CPUs)
    - CQ-C
```

Then:

```
- submit 10cpu workload to CQ-C
- workload admits to CQ-C
- submit 5 cpu workload to each of CQ-A and CQ-B
- workload from CQ-B admits
```

**What you expected to happen**:
workload from CQ-A should admit, as Cohort-A is not borrowing, while Cohort-B is.

**How to reproduce it (as minimally and precisely as possible)**:
See (LLM generated) test case: https://github.com/kubernetes-sigs/kueue/commit/be3e92c984ce93b772f557200d6e1e36ae84d45b

```
go test ./pkg/scheduler/ -run "TestSchedule/with_fair_sharing:_hierarchical_nominal-first" -count=1
```

This test passes with flag disabled.
