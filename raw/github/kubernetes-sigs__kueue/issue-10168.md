## Description

**What would you like to be added**:

Add a lightweight simulator for Kueue. kube-scheduler integration can be considered later as an optional extension, but it should not block the initial simulator.

**Why is this needed**:

The main goal of this simulator is to **simulate a large number of workloads virtually and cheaply without building a real Kubernetes cluster(ex. kwok)**.

Today, it is expensive and slow to validate Kueue behavior with real workloads because it usually requires setting up a real cluster and running real workloads.

A lightweight simulator would make it much easier to:

- test Kueue policy and admission behavior
- compare queueing / preemption / fair sharing configurations
- reproduce workload waiting / admission / eviction scenarios
- run large-scale what-if analysis with low cost
- do regression testing for Kueue changes

For the MVP, it is more important to model **Kueue decision-making** accurately than to model full Kubernetes end-to-end execution.
