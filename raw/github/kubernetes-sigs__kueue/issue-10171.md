## Description

**What would you like to be added**:

A new policy value `LowerPriorityBorrowersOnly` for `borrowWithinCohort.policy` (Classical Preemption only).

Today `borrowWithinCohort` supports `Never` and `LowerPriority`. `LowerPriority` allows a borrowing preemptor to preempt any lower-priority workload in the cohort, including workloads that are running within their CQ's nominal quota. 

The proposed `LowerPriorityBorrowersOnly` policy adds a constraint: a workload is only a valid preemption target if **the target CQ is borrowing** for the contested resources, and **removing that specific workload would not take the target CQ's usage below its nominal quota**.

**Concrete example:**

CQ-A (preemptor) needs to borrow 4 GPUs. CQ-B has nominal=10 GPUs, current usage=13 GPUs (borrowing 3):
- Workload W1 in CQ-B uses 2 GPUs → **valid target** (13-2=11, still ≥ nominal 10)
- Workload W2 in CQ-B uses 5 GPUs → **not a valid target** (13-5=8, below nominal 10)

CQ-C has nominal=10 GPUs, current usage=9 GPUs (not borrowing):
- No workloads in CQ-C are candidates, regardless of priority.

`maxPriorityThreshold` composes with this: both constraints must be satisfied (target must be lower priority, at or below the threshold, AND must be a borrower whose removal keeps the CQ at/above nominal).

**Why is this needed**:

In multi-tenant clusters, teams are allocated nominal quota as a guarantee. When a CQ borrows unused capacity from the cohort, it's understood that this capacity is best-effort and may be reclaimed. However, the current `LowerPriority` policy doesn't distinguish between borrowed and guaranteed capacity, meaning a borrowing preemptor can evict workloads that are running entirely within a CQ's guaranteed allocation.

`LowerPriorityBorrowersOnly` provides a middle ground between `Never` (no preemption while borrowing) and `LowerPriority` (unrestricted preemption of lower-priority workloads): borrowing preemptors can reclaim borrowed capacity from other CQs, but guaranteed/nominal capacity is protected. This aligns with the principle that nominal quota should be a hard guarantee.
