## Description


**What happened**:

Preemptions can occassionally get stuck due to unsatisfied preemption expectations, see https://github.com/kubernetes-sigs/kueue/pull/10179

**What you expected to happen**:

* preemptions never get stuck on preemptionExpectations


**How to reproduce it (as minimally and precisely as possible)**:

test: https://github.com/kubernetes-sigs/kueue/pull/10179

**Anything else we need to know?**:

The root cause is that the workload_controller removes the expctations before the scheduler sees the Evicted=true condition. As a result scheduler may repeat the request, and repeat adding the preemptionExpectation. The repeated expectation is never removed because the victim workload has already transitioned to Evicted.
