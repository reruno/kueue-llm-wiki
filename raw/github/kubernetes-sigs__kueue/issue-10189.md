## Description

<!-- Please only use this template for submitting enhancement requests -->

**What would you like to be added**:
A new condition type to be added to the WorkloadStatus of the Manager Cluster Workloads denoting and summarizing their status based on what is happening to them in the broader MultiKueue context (outside of just the Manager Cluster and its Cluster Queue).

**Why is this needed**:
The Workload status in queue in the Management Cluster must reflect the true state (Pending, Admitted, Running, Failed, Succeeded) derived from the remote cluster, including a human-readable message explaining the state (e.g., "Waiting for quota in cluster X").
