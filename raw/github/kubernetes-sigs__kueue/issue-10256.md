## Description

**What would you like to be cleaned**:

I would propose to put interim logs and xml resports into a folder like "build-reports" folder, reflecting also that they should be saved by the CI job as "build-reports".

**Why is this needed**:

The auxiliary files like xml or logs usually are not a good fit for "artifacts" nor "bin" folder, because:
- artifacts usually implies outputs worth keeping, publishing, or consuming downstream. Mixing auxiliary artifacts with release artifacts requires people responsible for the release process to be very careful not to upload junk files during the release process.
- bin strongly suggests executables or binary blobs.
