## Description

<!-- Please only use this template for submitting clean up requests -->

**What would you like to be cleaned**:

I would like to use the same pattern for ungating Pods for ElasticJobs as in [TopologyUngater](https://github.com/kubernetes-sigs/kueue/blob/main/pkg/controller/tas/topology_ungater.go) for TAS. 

Currently the ungating is triggered by the job Reconciler by calling [StartWorkloadSlicePods](https://github.com/kubernetes-sigs/kueue/blob/main/pkg/workloadslicing/workloadslicing.go)

By analogy to TopologyUngater we could call the controller ElasticJobUngater.

**Why is this needed**:

1. single path for handling Failed Pods and Pods created for the first time, avoiding bugs like https://github.com/kubernetes-sigs/kueue/issues/9879
2. code which works for all CRDs OOTB, current short-term fix is only for RayCluster: https://github.com/kubernetes-sigs/kueue/pull/10035
3. no direct dependency from jobframework to elasticworkloads directory, so easier package-level dependency management
4. the same pattern used for similar cases (TAS & ElasticJobs)
