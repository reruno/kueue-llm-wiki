## Description

**What would you like to be added**:

I’d like ClusterQueue to support automatic quota detection based on node selectors. By passing a match expression (or an empty selector to watch all nodes), the quota can dynamically sync with the cluster's current capacity. This removes the manual overhead of adjusting limits during node pool scaling.

**Why is this needed**:

Currently, users must manually sync ClusterQueue quotas with the underlying infrastructure. I would like Kueue to support a mode where limits default to the actual available resources in the cluster (with configurable headroom for system resources). This allows capacity management to be handled at the node pool level (e.g., via max nodes settings), with Kueue dynamically inheriting those boundaries rather than requiring redundant manual updates."
