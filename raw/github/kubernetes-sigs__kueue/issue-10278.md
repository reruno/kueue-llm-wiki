### Comment by [@kannon92](https://github.com/kannon92) — 2026-04-02T14:27:03Z

I investigated this issue and found the root cause in the genref markdown templates.

**Root cause**: The `hack/genref/markdown/pkg.tpl` template uses `$isConfig := (eq .GroupName "")` to detect config packages and give them special handling (rendering all types including the root `Configuration` type). However, the config packages declare `+groupName=config.kueue.x-k8s.io` in their `doc.go`, so `GroupName` is non-empty and the guard never triggers.

This causes the template to fall through to the regular API branch, which only renders types that are `IsExported` (has `+genclient` or `+k8s:deepcopy-gen:interfaces=...Object` annotations) or `Referenced` (referenced by another type). The `Configuration` type has neither — it's the root type so nothing references it, and it lacks those specific annotations. As a result, the entire `Configuration` type definition and all its direct fields are omitted from the generated docs.

**Fix**: Change `$isConfig` from `(eq .GroupName "")` to `(eq .GroupName "config.kueue.x-k8s.io")` and restructure the rendering branch so config packages include all visible types. Also update `type.tpl` so the `Configuration` type gets `apiVersion`/`kind` header rows. The same fix applies to both v1beta1 and v1beta2 config docs.

I have a working fix locally — will put up a PR.
