## Description

<!-- Please only use this template for submitting clean up requests -->

**What would you like to be cleaned**:
Some tests allow/expect admissions without any flavors assigned to be created in the tested scenarios.
This is not something that should ever happen in the production environment for actual users.
The main pain-point this causes is the logic assigning/verifying Admission Checks on Admissions.

These tests should be amended and the possibility of flavorless admissions being created - eliminated.

**Why is this needed**:
Tests should present scenarios as close to reality as possible. If a test relies on an illegal state to occur, it is fundamentally flawed. This risks some tests producing false positives/negatives despite the logic working as expected.
