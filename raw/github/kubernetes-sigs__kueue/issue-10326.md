**What would you like to be added**:
Currently when a workload is created for a LeaderWorkerSet, the PodSets are created without any metadata (https://github.com/kubernetes-sigs/kueue/blob/main/pkg/controller/jobs/leaderworkerset/leaderworkerset_reconciler.go#L283-L285).

If a user specifies some Labels/Annotations at the LeaderWorkerTemplate, these metadata should be propagated to the PodSet. This will be consistent with what we do for the Jobs (https://github.com/kubernetes-sigs/kueue/blob/main/pkg/controller/jobs/job/job_controller.go#L253)

 I think we should not propagate the labels that are managed by the lws-system (eg `leaderworkerset.sigs.k8s.io/group-index`).
