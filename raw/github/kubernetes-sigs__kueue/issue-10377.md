**What would you like to be cleaned**:

The GH action image format should be image hash + version comment like `kubernetes-sigs/release-actions/setup-bom@8753ea6bdadb814d779c6ec34eaca689dbfb492b # v0.4.3`.

- Ideal pattern: https://github.com/kubernetes-sigs/kueue/blob/e65ae5e5cc3ed8fb956200f0a30c9f7d6bc7560f/.github/workflows/sbom.yaml#L30
- Not good patterns
  - tag version comment is missing
    - [ ] https://github.com/kubernetes-sigs/kueue/blob/e65ae5e5cc3ed8fb956200f0a30c9f7d6bc7560f/.github/workflows/sbom.yaml#L24
    - [ ] https://github.com/kubernetes-sigs/kueue/blob/e65ae5e5cc3ed8fb956200f0a30c9f7d6bc7560f/.github/workflows/openvex.yaml#L22
  - not a hash-based image
    - [ ] https://github.com/kubernetes-sigs/kueue/blob/e65ae5e5cc3ed8fb956200f0a30c9f7d6bc7560f/.github/workflows/krew-release.yml#L10
    - [ ] https://github.com/kubernetes-sigs/kueue/blob/e65ae5e5cc3ed8fb956200f0a30c9f7d6bc7560f/.github/workflows/krew-release.yml#L12
    - [ ] https://github.com/kubernetes-sigs/kueue/blob/e65ae5e5cc3ed8fb956200f0a30c9f7d6bc7560f/.github/workflows/openvex.yaml#L23
    - [ ] https://github.com/kubernetes-sigs/kueue/blob/e65ae5e5cc3ed8fb956200f0a30c9f7d6bc7560f/.github/workflows/sync-dependabot.yaml#L18

**Why is this needed**:

The hash-based image specification allows us to protect the Kueue repository from supply-chain attacks. Because the tag could be modified. If the gh action library repository is taken over by malicious developers, they often ship the malicious code with the existing shipped tag. 

The version comment like `# v0.4.3` allows us to automatically update image by dependabot even when we use hash-based image.
