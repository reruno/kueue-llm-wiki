## Description

<!-- Please only use this template for submitting enhancement requests -->

**What would you like to be added**:

Introduce a global knob in the ConfigMap for determining if short workload names, or workload names should be used.

This is a follow up to https://github.com/kubernetes-sigs/kueue/issues/9872. It seems just graduating the feature gate ShortWorkloadNames may not be ideal for some users which value the extra information contained in the suffix of the workload name. 

**Why is this needed**:

Short workload names are good in case someone wants to use them in internal systems as labels.

Long workload names might be better in case some user wants to keep some relevant information visible in the workload name. For example, short workload names may lose the index number of the LWS group as this is added as a suffix.

**Completion requirements**:

I think a small KEP listing both use-cases and some ConfigMap choice for the default which will replace the FG in the long term. I imagine 

```yaml
workloadNames:
  mode: Long / Short
   # potentially other configuration parameters
```
