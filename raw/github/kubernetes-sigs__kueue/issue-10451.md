## Description

<!-- Please only use this template for submitting clean up requests -->

**What would you like to be cleaned**:

The intention was to run e2e tests in parallel, using the E2E_NPROC param, but it does not seem to work (I think), looking at the timestamps for a selected build they all look sequential: https://storage.googleapis.com/kubernetes-ci-logs/pr-logs/pull/kubernetes-sigs_kueue/10440/pull-kueue-test-e2e-main-1-35/2043949072166948864/build-log.txt

Also, there is no GINKGO_ARGS="", and I don't see E2E_NPROC logged anywhere, so I suspect this is not working well.

**Why is this needed**:

The e2e tests are currently taking too long: https://prow.k8s.io/job-history/gs/kubernetes-ci-logs/pr-logs/directory/pull-kueue-test-e2e-main-1-35

Sometimes even 41min.
