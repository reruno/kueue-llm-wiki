**What would you like to be documented or improved**:

I would like to add a section in the Installation page about upgrade policy. 

Sometimes users when upgrading from 0.15.x to 0.16.y may only check the upgrade steps for  0.16.y  skipping the upgrade steps for 0.16.0. 

We should explicitly state in the section, that when upgrading to any patch release > .0 the admin should also check the upgrade steps to the .0 release. 

It would be helpful to even give an example with auto-generated links. 

For example, in order to upgrade to the latest 0.17.1 please also follow the upgrade steps for 0.17.0 (with the appropriate link). 

We could also give this information in the release notes itself.

**Location** (URL, file path, or section if applicable):

https://kueue.sigs.k8s.io/docs/installation/
