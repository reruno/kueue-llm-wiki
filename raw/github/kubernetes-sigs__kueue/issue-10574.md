**What would you like to be added**:
Currently in TAS, the domains on each level (in the `updateCountsToMinimum` function) are selected using a "smart greedy" algorithm (pick domains greedily until fit + optimize the last domain). 

So, if we have to assign `20` units and capacities are `[19, 10, 10]`, we would choose the `19, 10` domains

But its possible to find an optimal selection (there is a pseudo-polynomial algorithm for knapsack problem using the dynamic programming). 

Such optimal selection was implemented as part of BalancedPlacement algorithm for TAS. The question is if we want to provide it as an alternative to `Best-Fit`.

**Why is this needed**:
It would lead to smaller cluster fragmentation. 

Its probably not worth it in the general case, but could be useful for some very costly and scarce resources like GPUs.
