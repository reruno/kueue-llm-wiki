## Description

**What happened**:

We observe a lot of ERROR level logs due to flavors not matching to PodSet for TAS due to 
- "Flavor "xyz" does not contain the requested level"
- "Flavor "xyz" does not support TopologyAwareScheduling"

Logged from https://github.com/kubernetes-sigs/kueue/blob/main/pkg/scheduler/flavorassigner/flavorassigner.go#L944

**What you expected to happen**:

Such a non-compatible flavor should not be reported as an error. It might be a useful V3 log for debugging. It only means that the PodSet needs to be matched against another flavor.

**How to reproduce it (as minimally and precisely as possible)**:

For example the unit test "workload requiring TAS skips the non-TAS flavor"
