## Description

**What happened**:
When we change name of `wl-a` workload to `wl-z` in this test case - https://github.com/kubernetes-sigs/kueue/blob/c1cf6e4db7041e1ccec14b7f209da08bccf50322/pkg/scheduler/scheduler_fs_test.go#L462

I looks like workloads are chosen based on alphabetical order instead of choosing a workload that will not borrow resources to satisfy fair sharing constraints.

Reproduction: https://github.com/kubernetes-sigs/kueue/pull/10612

**What you expected to happen**:
I expected `wl-a` to be chosen as its cohort doesn't borrow any resources while `wl-b`'s cohort already borrows.
