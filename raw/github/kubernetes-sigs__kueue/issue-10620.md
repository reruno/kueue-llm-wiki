## Description

**What would you like to be added**:

I would like to properly populate the underlying_cause for the Eviction-oriented metrics.

Currently we only expose "" as the underlying reason for AdmissionCheck retry. However, some complex controllers may have 2, 3 or more reasons for Retry. Users would like to gain visibility into statistics for different reasons.

I think this likely will require extending the AdmissionCheckState with another field like "Reason" which could be used to set the underlying_cause in the metric.

This is related to https://github.com/kubernetes-sigs/kueue/issues/10618, but goes beyond just messages.

**Why is this needed**:

To improve observability, so that the user can track how many ACs were retried due to different reasons.
