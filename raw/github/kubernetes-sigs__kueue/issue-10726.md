## Description

<!-- Please only use this template for submitting enhancement requests -->

**What would you like to be added**:

I propose adding an `nodeAffinity` field to the ResourceFlavor API. This would allow administrators to use standard Kubernetes operators (e.g. NotIn, In) at the flavor level to centrally evaluate node properties during workload admission.

**Why is this needed**:

Currently, ResourceFlavor objects in Kueue only support strict key-value equality matching via the `nodeLabels` field. This is a significant limitation for cluster administrators who need to define flavors based on exclusion (anti-affinity) rather than inclusion.

This feature is useful for instance during health-aware scheduling: where nodes are dynamically labeled as degraded/unhealthy, using anti-affinity would allow the admin to route the workloads away from them automatically.
