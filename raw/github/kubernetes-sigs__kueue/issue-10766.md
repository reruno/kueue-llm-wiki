**What would you like to be added**:
We need a composable dispatcher that allows for different routing policies. The overall goal is to treat clusters like nodes with concepts like filtering and scoring, and also node anti/affinity. 

**Why is this needed**:
MultiKueue is a powerful concept but today it has major shortcomings. Today its limited to submitting the workload to all worker clusters and allowing the first one that admits it to pick up the workload. At most you can define some limited set of clusters to submit to. 

I need stronger policy setting for training clusters. Users want to be able to do things like :
1. Target highest quality of quota (ex: target guarantees or best burst, decreasing probability of preemption)
2. Have affinity to things like data locality or specific services installed on the cluster, etc. 

(These two are out of scope and require separate keps that I will link. But you need the foundation for adding these sort of plugins.)
