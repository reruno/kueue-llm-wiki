## Description

When RayService that uses Redis deleted, the RayCluster creates Redis cleanup job. I observed that the Redis cleanup job was able to create a redis cleanup pod, but then it was removed and the job was stuck in suspended state and was not able to execute.

The job stuck in suspended state because it could not find the RayService kueue workload. 
Since the RayCluster has ray.io/gcs-ft-redis-cleanup-finalizer it stuck in suspended state as well.
