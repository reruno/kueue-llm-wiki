## Description

<!-- Please only use this template for submitting enhancement requests -->

**What would you like to be added**:

Kueue should be able to take a VPA matching the Job/CronJob into account for calculating the workload size.

**Why is this needed**:

Using the VerticalPodAutoscaler in combination with Kueue does not work properly in that sense, because the VPA is modifying the scheduled pod, but not the workload object, thus not taking the full possibility of "requests" into account. Having that integration would allow to schedule more workloads/or less on the underlying cluster.
