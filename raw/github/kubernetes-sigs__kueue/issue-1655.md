## Description

Add e2e tests that cover high-availability of Kueue webhooks and visibility extension API, as discussed in https://github.com/kubernetes-sigs/kueue/pull/1554#pullrequestreview-1818175088.

These tests should be as close as real-world use cases, and guarantee among other things:
* The results return by the visibility API is correctly served by all replicas (eventually)
* In case the current leading replica is terminated, reconciliation correctly fails over to the newly elected leader replica
