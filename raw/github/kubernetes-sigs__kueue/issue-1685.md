## Description

**What would you like to be added**:

The Kueue deployment should use a uniquely identifying label selector for its Pods.

We may want to adopt Kubernetes [recommended labels](https://kubernetes.io/docs/concepts/overview/working-with-objects/common-labels/) for defining this label selector.

Note that label selector is immutable, so existing users will have to recreate the deployment. This should be documented in the release note.

**Why is this needed**:

The Kueue deployment is configured with the default label selector, as generated by the kubebuilder scaffolding, i.e.:  

```yaml
spec:
  selector:
    matchLabels:
      control-plane: controller-manager
```

This selector does not uniquely identify the Kueue Pods, and chances are it intersects with other deployments, which leads to issues where selection based on this selector returns more than expected.

This also affects the Prometheus ServiceMonitor and the auth proxy Service.

**Completion requirements**:

This enhancement requires the following artifacts:

- [ ] Design doc
- [ ] API change
- [ ] Docs update

The artifacts should be linked in subsequent comments.
