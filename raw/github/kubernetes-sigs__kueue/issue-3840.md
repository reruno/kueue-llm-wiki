## Description

**What happened**:

MultiKueue keeps regular worklaods in its cache currently which is the source of https://github.com/kubernetes-sigs/kueue/issues/3818

**What you expected to happen**:

Regular workloads are no kept in MultiKueue cache.

**How to reproduce it (as minimally and precisely as possible)**:

Create workloads not managed by Kueue, observe they are present in the cache with debugger, or with extra logging.

**Anything else we need to know?**:

https://github.com/kubernetes-sigs/kueue/pull/3835 is quick fix, check the comment
