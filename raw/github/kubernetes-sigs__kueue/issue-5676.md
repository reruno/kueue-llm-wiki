## Description

<!-- Please only use this template for submitting enhancement requests -->

**What would you like to be added**:

The current bucket sizes defined in the kueue admission wait time metrics [here](https://github.com/kubernetes-sigs/kueue/blob/main/pkg/metrics/metrics.go#L436) are statically generated. Requesting to allow custom bucket ranges to be provided for metric generation, so that the users can have better analytics built over the metrics.

It would also be great if we can provide these bucket ranges either through helm values or through env variables when configuring Kueue.

**Why is this needed**:

To build better analytics and observability over Kueue metrics, based on the custom usecases
