## Description

<!-- Please only use this template for submitting clean up requests -->

**What would you like to be cleaned**:
While users could start by looking at e2e tests, Kueue could use more demo resources where they can apply yaml manually to help evaluate Kueue for a scoped down set of simple scheduling scenarios. Both when I was evaluating Kueue and when I look at community slack questions, it seems like this could be useful. 

In docs under Tasks, most examples seem to stop at a single workload scope. We need more scheduling / quota bursting examples.

**Why is this needed**:
Help make evaluating Kueue as a solution easier.
