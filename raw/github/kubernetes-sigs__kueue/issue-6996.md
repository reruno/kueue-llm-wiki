## Description

<!-- Please only use this template for submitting enhancement requests -->

**What would you like to be added**: 
Option to ignore labels that don't exist on nodes in topology levels.

**Why is this needed**:
Currently, if a label that's used in one of the topology levels does not exist on any node, TAS doesn't work because it says it couldn't create the domain structure.

We have some topology labels added to only specific nodes. We want to be able to use them _if_ they exist. And if not, continue with the next level in the topology when `podset-preferred-topology` is used.
