<blockquote>
<h2>v2.5.0</h2>
<h1>🚀 Spark on Kubernetes Operator v2.5.0</h1>
<p>We're excited to announce the release of <strong>Spark on Kubernetes Operator v2.5.0</strong>! This release introduces
alpha feature gates, namespace label-based watching, Python API generation, SparkConnect webhook validation,<br />
and several important bug fixes including an OOM prevention fix.</p>
<hr />
<h2>✨ New Features</h2>
<ul>
<li><strong>Feature gate mechanism</strong> — Alpha feature gates are now supported, enabling experimental features to be<br />
toggled safely (<a href="https://redirect.github.com/kubeflow/spark-operator/pull/2794">#2794</a>)</li>
<li><strong><code>LoadSparkDefaults</code> feature gate</strong> <em>(Alpha)</em> — Adds <code>--load-spark-defaults</code> to spark-submit for<br />
Spark 4.0+ (<a href="https://redirect.github.com/kubeflow/spark-operator/pull/2798">#2798</a>)</li>
<li><strong><code>PartialRestart</code> feature gate</strong> <em>(Alpha)</em> — Skips reconcile for webhook-patched executor fields,<br />
reducing unnecessary restarts (<a href="https://redirect.github.com/kubeflow/spark-operator/pull/2786">#2786</a>)</li>
<li><strong>Watch namespaces by labels</strong> — Operator can now watch namespaces based on label selectors<br />
(<a href="https://redirect.github.com/kubeflow/spark-operator/pull/2808">#2808</a>)</li>
<li><strong>Configurable timestamp precision</strong> for <code>ScheduledSparkApplication</code> names<br />
(<a href="https://redirect.github.com/kubeflow/spark-operator/pull/2827">#2827</a>)</li>
<li><strong>Python API generation</strong> for Spark Operator CRDs<br />
(<a href="https://redirect.github.com/kubeflow/spark-operator/pull/2828">#2828</a>)</li>
</ul>
<hr />
<h2>🐛 Bug Fixes</h2>
<ul>
<li><strong>OOM prevention</strong> — Add label selector to ConfigMap cache to prevent OOM via informer flooding<br />
(<a href="https://redirect.github.com/kubeflow/spark-operator/pull/2881">#2881</a>)</li>
<li>Fix duplicate webhook patch and add missing <code>ScheduledSparkApplication</code> patches<br />
(<a href="https://redirect.github.com/kubeflow/spark-operator/pull/2875">#2875</a>)</li>
<li>Fix: correct schedule parse error logging in <code>ScheduledSparkApplication</code> controller<br />
(<a href="https://redirect.github.com/kubeflow/spark-operator/pull/2841">#2841</a>)</li>
<li>Fix: handle nil <code>Executor.Instances</code> in <code>GetExecutorRequestResource</code><br />
(<a href="https://redirect.github.com/kubeflow/spark-operator/pull/2834">#2834</a>)</li>
<li>Fix: correct filename typo in metrics package<br />
(<a href="https://redirect.github.com/kubeflow/spark-operator/pull/2830">#2830</a>)</li>
<li>Fix: propagate batch scheduler initialization errors to trigger retries<br />
(<a href="https://redirect.github.com/kubeflow/spark-operator/pull/2783">#2783</a>)</li>
<li>Reset <code>SparkApplication</code> status when transitioning from <code>SUCCEEDING</code>/<code>FAILING</code> → <code>PENDING_RERUN</code><br />
(<a href="https://redirect.github.com/kubeflow/spark-operator/pull/2773">#2773</a>)</li>
<li>Server-side apply CRDs by forcing conflicts<br />
(<a href="https://redirect.github.com/kubeflow/spark-operator/pull/2800">#2800</a>)</li>
<li>Fix <code>SparkConnect</code> nil executor template panic<br />
(<a href="https://redirect.github.com/kubeflow/spark-operator/pull/2814">#2814</a>)</li>
</ul>
