<h2>v0.16.4</h2>
<p>Changes since <code>v0.16.3</code>:</p>
<h2>Changes by Kind</h2>
<h3>Feature</h3>
<ul>
<li>Helm: Allow setting log level (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9944">#9944</a>, <a href="https://github.com/gabesaba"><code>@​gabesaba</code></a>)</li>
<li>TAS: Extend the support for handling NoSchedule taints when the TASReplaceNodeOnNodeTaints feature gate is enabled. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/10003">#10003</a>, <a href="https://github.com/j-skiba"><code>@​j-skiba</code></a>)</li>
<li>VisibilityOnDemand: Introduce a new Kueue deployment argument, --visibility-server-port, which allows passing custom port when starting the visibility server. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9976">#9976</a>, <a href="https://github.com/Nilsachy"><code>@​Nilsachy</code></a>)</li>
</ul>
<h3>Bug or Regression</h3>
<ul>
<li>
<p>LWS integration: Fixed a bug that the <code>kueue.x-k8s.io/job-uid</code> label was not set on the workloads. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/10010">#10010</a>, <a href="https://github.com/mbobrovskyi"><code>@​mbobrovskyi</code></a>)</p>
</li>
<li>
<p>MultiKueue: Enable AllowWatchBookmarks for remote client watches to prevent idle watch connections from being terminated by HTTP proxies with idle timeouts (e.g., Cloudflare 524 errors). (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9990">#9990</a>, <a href="https://github.com/trilamsr"><code>@​trilamsr</code></a>)</p>
</li>
<li>
<p>Scheduling: fix the issue that scheduler could indefinitely try re-queueing a workload which was once
inadmissible, but is admissible after an update. The issue affected workloads which don't specify
resource requests explicitly, but rely on defaulting based on limits. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9913">#9913</a>, <a href="https://github.com/mimowo"><code>@​mimowo</code></a>)</p>
</li>
<li>
<p>Scheduling: fixed SchedulingEquivalenceHashing so equivalent workloads that become inadmissible through
the preemption path with no candidates are also covered by the mechanism.</p>
<p>As a safety measure while the broader fix is validated, the beta SchedulingEquivalenceHashing feature gate
is temporarily disabled by default. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/10007">#10007</a>, <a href="https://github.com/mimowo"><code>@​mimowo</code></a>)</p>
</li>
<li>
<p>StatefulSet integration: Fixed a bug that the <code>kueue.x-k8s.io/job-uid</code> label was not set on the workloads. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9902">#9902</a>, <a href="https://github.com/mbobrovskyi"><code>@​mbobrovskyi</code></a>)</p>
</li>
<li>
<p>TAS: Fixed a bug where pods could become stuck in a <code>Pending</code> state during node replacement.
This may occur when a node gets tainted or <code>NotReady</code> after the topology assignment phase, but before
the pods are ungated. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9978">#9978</a>, <a href="https://github.com/j-skiba"><code>@​j-skiba</code></a>)</p>
</li>
<li>
<p>TAS: fix the bug that workloads which only specify resource limits, without requests, are not able to perform
the second-pass scheduling correctly, responsible for NodeHotSwap and ProvisioningRequests. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9947">#9947</a>, <a href="https://github.com/mimowo"><code>@​mimowo</code></a>)</p>
</li>
<li>
<p>VisibilityOnDemand: Fix non-deterministic workload ordering with UsageBasedAdmissionFairSharing enabled. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9955">#9955</a>, <a href="https://github.com/sohankunkerkar"><code>@​sohankunkerkar</code></a>)</p>
</li>
</ul>
<h3>Other (Cleanup or Flake)</h3>
<ul>
<li>Restore the FlavorFungibilityImplicitPreferenceDefault feature gate. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9991">#9991</a>, <a href="https://github.com/mimowo"><code>@​mimowo</code></a>)</li>
</ul>
<h2>v0.16.3</h2>
<p>Changes since <code>v0.16.2</code>:</p>
<h2>Changes by Kind</h2>
<h3>Feature</h3>
<ul>
<li>Observability: Add scheduler logs for the scheduling cycle phase boundaries. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9813">#9813</a>, <a href="https://github.com/sohankunkerkar"><code>@​sohankunkerkar</code></a>)</li>
<li>Scheduling: Add the alpha SchedulerLongRequeueInterval feature gate (disabled by default) to increase the
inadmissible workload requeue interval from 1s to 10s. This may help to mitigate, on large environments with
many pending workloads, issues with frequent re-queues that prevent the scheduler from reaching schedulable
workloads deeper in the queue and result in constant re-evaluation of the same top workloads. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9819">#9819</a>, <a href="https://github.com/mbobrovskyi"><code>@​mbobrovskyi</code></a>)</li>
<li>Scheduling: Add the alpha SchedulerTimestampPreemptionBuffer feature gate (disabled by default) to use
5-minute buffer so that workloads with scheduling timestamps within this buffer don’t preempt each other
based on LowerOrNewerEqualPriority. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9837">#9837</a>, <a href="https://github.com/mbobrovskyi"><code>@​mbobrovskyi</code></a>)</li>
</ul>
