### Comment by [@mbobrovskyi](https://github.com/mbobrovskyi) — 2026-04-07T08:07:59Z

> Also, could you explain how this fix relates to the disabled test "correctly handles cohort metrics when workload admitted with admission check"? I don't see this test removing a cohort, but maybe I'm missing something.

I already explained this here: https://github.com/kubernetes-sigs/kueue/pull/10080#discussion_r2986458874.

The issue is that other tests create and delete cohorts. When the cohort is explicit, it works fine because we clear metrics on deletion. However, with implicit cohorts, we have a problem – the metrics are not cleared.

As a result, a previous test leaves metrics behind, which causes flakiness. We expect 2 active workloads but get 5 (left over from a previous test).

That’s why when we delete a Cohort or ClusterQueue, we need to ensure that all Cohort metrics are cleared.
