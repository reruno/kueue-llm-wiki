#### What this PR does / why we need it:
Alongside resolving #7343, this PR also finishes the DRA test cleanup that was started by #7529:

| File | Change |
| --- | --- |
| `extended_resources_test.go` | Error assertions in `TestResolveExtendedResourceQuota()` converted from `wantErr bool` / `wantErrCount int` to structured `field.ErrorList` comparison using `cmp.Diff` |
| `claims_test.go` | `reflect.DeepEqual` (that was flagged in the review of #7529) replaced with `cmp.Diff` on the success path |

#### Which issue(s) this PR fixes:
Fixes #7343 

#### Special notes for your reviewer:
- The only error-path test case that needed changing in `extended_resources_test.go` is "workload with non-integer extended resource quantity":

  - Its assertion now actually checks the exact `field.Invalid` error _type_ and _field path_. 
  - `Detail` and `BadValue` are excluded via `cmpopts.IgnoreFields`, which makes sense given these fields don't affect correctness; this also makes the code consistent with `claims_test.go`.

- The `reflect.DeepEqual` → `cmp.Diff` change in `claims_test.go` also switches from `t.Fatalf` to `t.Errorf` to be consistent with the other two `pkg/dra` test files (as well as the technicality that this assertion failing shouldn't hide other assertions in the same subtest, although in practice there are no subsequent assertions here).
