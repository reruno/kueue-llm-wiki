#### What this PR does / why we need it:
As described in the issue, the problem occurs if the flavor assigner assigns mixed modes `(Preempt, Fit)`, and the `Fit` mode has to be updated to `Preempt` because of TAS rules (fragmentation). 

In this PR if the RepresentativeMode is `Preempt`, we update the mode to `Preempt` in all PodSets and all Resources of this workload. This ensures that we consider for preemption, the candidates that only use the resource that originally was assigned the `Fit` mode.
