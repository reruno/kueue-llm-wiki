## Pull request overview

This PR addresses a flaky unit test in the TAS scheduler by making the expected vs. recorded Kubernetes events comparison order-independent in the affected preemption scenario.

**Changes:**
- Updates `TestScheduleForTASPreemption` to sort event slices during comparison (while still ignoring event messages) to avoid failures due to nondeterministic event emission order.
