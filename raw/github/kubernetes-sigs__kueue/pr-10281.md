## Description

#### What type of PR is this?

/kind documentation

#### What this PR does / why we need it:

The generated reference docs for `config.kueue.x-k8s.io/v1beta1` and `config.kueue.x-k8s.io/v1beta2` were missing the root `Configuration` type and all its direct fields (`namespace`, `manageJobsWithoutQueueName`, `managedJobsNamespaceSelector`, `featureGates`, etc.).

The root cause was that `Configuration` lacked the `+k8s:deepcopy-gen:interfaces=k8s.io/apimachinery/pkg/runtime.Object` marker comment, which the genref tooling uses to identify top-level resource types. This PR adds that marker to both `v1beta1` and `v1beta2` `Configuration` types and regenerates the reference docs.

Changes:
- Add `+k8s:deepcopy-gen:interfaces=k8s.io/apimachinery/pkg/runtime.Object` marker to `apis/config/v1beta1/configuration_types.go` and `apis/config/v1beta2/configuration_types.go`
- Regenerate `kueue-config.v1beta1.md` and `kueue-config.v1beta2.md` which now include the `Configuration` type with all its fields and proper "Appears in" cross-references

#### Which issue(s) this PR fixes:

Fixes #10278

#### Special notes for your reviewer:

The marker `+k8s:deepcopy-gen:interfaces=k8s.io/apimachinery/pkg/runtime.Object` is already present on other root types (e.g. in the core API types). Adding it here is consistent with the existing pattern and is what the genref tool uses to identify types that should appear in the "Resource Types" section with `apiVersion`/`kind` rows.
