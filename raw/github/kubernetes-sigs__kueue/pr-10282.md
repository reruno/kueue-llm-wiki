## Description

#### What type of PR is this?

/kind bug
/area tas

#### What this PR does / why we need it:

`IsExplicitTAS()` only checks `PodSetRequiredTopologyAnnotation` and `PodSetPreferredTopologyAnnotation`. Pods using `PodSetSliceRequiredTopologyAnnotation` and `PodSetSliceRequiredTopologyConstraintsAnnotation` (for multi-slice TAS scheduling) are not recognized as TAS pods.

The topology ungater's pod handler calls `IsTAS()` to determine whether a pod should trigger a reconcile. When `IsTAS()` returns false, the ungater ignores the pod, and the `kueue.x-k8s.io/topology` scheduling gate is never removed. This leaves pods permanently in `SchedulingGated` state.

Add `PodSetSliceRequiredTopologyAnnotation` and `PodSetSliceRequiredTopologyConstraintsAnnotation` to `IsExplicitTAS()` so pods with slice topology requests are properly recognized and ungated.

#### Which issue(s) this PR fixes:

Fixes #

#### Special notes for your reviewer:

The `kueue.x-k8s.io/podset-slice-required-topology` and `kueue.x-k8s.io/podset-slice-required-topology-constraints` are mutually exclusive, but both need to be checked in `IsExplicitTAS()`.

#### Does this PR introduce a user-facing change?

```release-note
TAS: fix a bug that Pods which only contain the `kueue.x-k8s.io/podset-slice-required-topology` or `kueue.x-k8s.io/podset-slice-required-topology-constraints` as the TAS annotation are not ungated.
```
