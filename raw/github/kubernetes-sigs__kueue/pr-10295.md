### Comment by [@Piyushkhobragade](https://github.com/Piyushkhobragade) — 2026-04-06T14:00:59Z

> I didn’t quite get the difference. Could you please show a before-and-after example?

Before:
f.log.V(3).Info(
    "Returning workload from ClusterQueue without Cohort",
    "clusterQueue", klog.KRef("", string(cq.GetName())),
    "workload", klog.KObj(entry.Obj),
)

After:
f.log.
    WithValues(
        "clusterQueue", klog.KRef("", string(cq.GetName())),
        "workload", klog.KObj(entry.Obj),
    ).
    V(3).
    Info("Returning workload from ClusterQueue without Cohort")

The difference is that the updated version attaches structured fields via WithValues on the context-aware logger before emitting the log. This ensures consistent structured output and better alignment with controller-runtime logging patterns, which helps maintain JSON-lines compatibility.
