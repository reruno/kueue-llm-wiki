#### What this PR does / why we need it:
Add explicit `nodeSelectorAssertMode` enum to clarify exact vs count-based verification in TAS tests
Test now requires `nodeSelectorAssertMode` field that must be one of:

- Exact: for rank-based ordering tests where each pod has a unique rank, making pod-to-domain assignments deterministic.
- CountsOnly: for greedy (non-rank) assignment tests, where the specific pod-to-domain mapping is non-deterministic
