#### What this PR does / why we need it:
Adds GroupNameAnnotation and PrebuiltWorkloadAnnotaion annotations which are available with newly introduced WorkloadIdentifierAnnotations feature gate.
Which helps to run LWS with longer name then before.
 
I tested changes using e2e tests.
I discovered that LWS tests are passing with workload name of 39 characters or less.
After moving to annotations for pod group name and prebuilt workload name limit, I retested.
I was able to pass tests with workload name length of 54 characters or less.

```release-note
Introduced the `WorkloadIdentifierAnnotations` feature gate. When enabled, Kueue stores pod-group and prebuilt-workload names on the annotations (`kueue.x-k8s.io/pod-group-name` and `kueue.x-k8s.io/prebuilt-workload-name`). The labels are still read as a fallback if the annotations are absent, and with the gate disabled the labels continue to be used for both reading and writing.
```
