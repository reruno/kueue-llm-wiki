## Description

#### What type of PR is this?
/kind bug

#### What this PR does / why we need it:
There is no validation for `AdmissionCheckStrategy.OnFlavors` to check that the specified flavors are listed in quota. This PR adds that validation so such `ClusterQueue` configurations fail fast. For pre-existing `ClusterQueue`s with invalid `onFlavors`, updates to unrelated fields are still allowed.

#### Which issue(s) this PR fixes:
Fixes #7259
