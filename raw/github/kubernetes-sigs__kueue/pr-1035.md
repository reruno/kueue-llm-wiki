  1.: https://kubernetes.io/docs/concepts/overview/working-with-objects/labels/#syntax-and-character-set
  but in our case they are also CRD names, so need to conform to:
  2.  https://kubernetes.io/docs/concepts/overview/working-with-objects/names/
  
  So basically, we are looking for a label that passes (1.) but fails (2.). 
  
  The spectrum is narrow, but I think "indexed_job" would do. It passes (1.) as `could contain dashes (-), underscores (_), dots (.), and alphanumerics between.`, but fails (2.) because it contains `_`, which is not enumerated here: `contain only lowercase alphanumeric characters, '-' or '.'`,
