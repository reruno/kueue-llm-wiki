<blockquote>
<h1>Release Process</h1>
<h2>Plugin OCI images</h2>
<p>Plugin binaries are released as OCI images built with <a href="https://docs.docker.com/build/buildx/">Docker Buildx</a>. The plugin binary is placed under <code>/bin/&lt;plugin_name&gt;-plugin</code> (e.g. <code>/bin/secretreader-plugin</code>); when the image is mounted at <code>/plugin</code>, the executable path is <code>/plugin/bin/&lt;plugin_name&gt;-plugin</code>.</p>
<h3>Versioning</h3>
<p>Plugin OCI images are tagged with the same version as the repository release tag. For example, pushing <code>v0.2.0</code> publishes all plugin images with tag <code>0.2.0</code>.</p>
<h3>How to release</h3>
<ol>
<li>Push a repository tag <code>v*</code> (e.g. <code>v1.0.0</code>). This triggers the <code>Release</code> workflow.</li>
<li>The workflow scans <code>plugins/*/</code> and checks whether the image already exists in the container registry for that version. Only plugins that haven't been published yet are built and pushed.</li>
<li>Container images are published to:
<ul>
<li><code>ghcr.io/kubernetes-sigs/cluster-inventory-api/&lt;plugin_name&gt;:&lt;version&gt;</code></li>
<li>Example: <code>ghcr.io/kubernetes-sigs/cluster-inventory-api/secretreader:1.0.0</code></li>
</ul>
</li>
</ol>
<h3>Image signing</h3>
<p>Each released image is signed using <a href="https://docs.sigstore.dev/cosign/signing/overview/">Cosign</a> keyless signing with GitHub Actions OIDC. No long-lived signing keys are used; an ephemeral key pair is generated for each signing event and the public key is recorded in the <a href="https://docs.sigstore.dev/logging/overview/">Sigstore transparency log</a>.</p>
<h4>Verifying image signatures</h4>
<p>Install <a href="https://docs.sigstore.dev/cosign/system_config/installation/">Cosign</a> and run:</p>
<pre lang="bash"><code>cosign verify \
  --certificate-oidc-issuer=https://token.actions.githubusercontent.com \
  --certificate-identity-regexp=&quot;^https://github\.com/kubernetes-sigs/cluster-inventory-api/\.github/workflows/release\.yml@refs/tags/v.*$&quot; \
  ghcr.io/kubernetes-sigs/cluster-inventory-api/&lt;plugin&gt;:&lt;version&gt;
</code></pre>
<p>Example:</p>
<pre lang="bash"><code>cosign verify \
  --certificate-oidc-issuer=https://token.actions.githubusercontent.com \
  --certificate-identity-regexp=&quot;^https://github\.com/kubernetes-sigs/cluster-inventory-api/\.github/workflows/release\.yml@refs/tags/v.*$&quot; \
  ghcr.io/kubernetes-sigs/cluster-inventory-api/secretreader:0.1.0
</code></pre>
<h3>SBOM and provenance</h3>
<p>Each released image includes attestations as OCI referrers:</p>
<ul>
<li><strong>SBOM</strong> (SPDX): <code>cosign download sbom ghcr.io/kubernetes-sigs/cluster-inventory-api/&lt;plugin&gt;:&lt;version&gt;</code></li>
<li><strong>Provenance</strong> (SLSA-style): attached by Buildx; verify with <a href="https://docs.sigstore.dev/cosign/verify-attestation/">cosign verify attestation</a> or your preferred policy engine.</li>
</ul>
