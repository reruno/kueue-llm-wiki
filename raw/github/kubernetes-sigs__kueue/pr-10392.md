## Pull request overview

Cherry-picks upstream refactor to move elastic job pod ungating out of the generic job reconciler into a dedicated controller, addressing cases where pods could remain scheduling-gated after scale-up.

**Changes:**
- Introduces the `ElasticJobUngater` controller (feature-gated) to ungate elastic workload-slice pods based on Workload admission/quota reservation.
- Removes the workload-slice pod-start/ungating path from `JobReconciler.ReconcileGenericJob`.
- Updates integration/e2e coverage and adds unit tests for the new controller behavior.

```diff
+func (r *elasticJobUngater) listGatedPods(ctx context.Context, wl *kueue.Workload) ([]*corev1.Pod, error) {
+	sliceName := workloadslicing.SliceName(wl)
+	var podList corev1.PodList
+	if err := r.client.List(ctx, &podList,
+		client.InNamespace(wl.Namespace),
+		client.MatchingFields{indexer.WorkloadSliceNameKey: sliceName},
+	); err != nil {
+		return nil, fmt.Errorf("listing pods for workload slice: %w", err)
+	}
+
+	var gated []*corev1.Pod
+	for i := range podList.Items {
+		if utilpod.HasGate(&podList.Items[i], kueue.ElasticJobSchedulingGate) &&
+			podList.Items[i].Annotations[kueue.WorkloadAnnotation] == wl.Name {
+			gated = append(gated, &podList.Items[i])
+		}
+	}
+	return gated, nil

+	}
+	wlName, found := pod.Annotations[kueue.WorkloadAnnotation]
+	if !found {
+		return
+	}
```

  `queueReconcileForPod` currently enqueues reconciles for every Pod update/delete that has `kueue.WorkloadAnnotation`, which is broadly applied across the system. This controller only operates on elastic workload-slice pods (which are identified via `kueue.WorkloadSliceNameAnnotation` and/or the elastic scheduling gate), so this watch can create significant unnecessary reconcile load. Consider filtering early (e.g., return unless `kueue.WorkloadSliceNameAnnotation` is present) to match `listGatedPods` and avoid reconciling unrelated workloads.
