#### What this PR does / why we need it:

This PR replaces the shell-based e2e Kind cluster lifecycle in `cmd/experimental/kueue-populator/` with a Go implementation.

The previous setup used `run-e2e.sh` to manage cluster creation, image loading, manifest application, and readiness checks. This logic is now implemented in Go test setup.

The goal is to simplify the flow and make lifecycle handling (cleanup, errors, readiness) easier to maintain.

This change is limited to `cmd/experimental/kueue-populator/`.
