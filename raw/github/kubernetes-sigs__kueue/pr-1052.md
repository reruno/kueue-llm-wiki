In an asynchronous system as kubernetes, it is possible that we will observe new Workload objects before their PodTemplate objects.

As such, we need a mechanism that:
- marks incomplete Workloads so they cannot be added to the queues, or, at least they cannot be popped into the scheduler.
- allows to complete a previously incomplete Workload.

In a way, this is a similar we have handling ClusterQueues when the ResourceFlavor object doesn't exist.

But I think we can have a simpler implementation for this case:
I think we can:
1. Index Workloads by the PodTemplate objects they refer to.
2. In the PodTemplate event handler, we can trigger the update of the corresponding Workload object. See how we trigger updates from one reconciler into another (like Workload->ClusterQueue).
