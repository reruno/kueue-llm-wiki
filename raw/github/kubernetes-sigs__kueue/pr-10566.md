Bumps [sigs.k8s.io/kueue](https://github.com/kubernetes-sigs/kueue) from 0.17.0 to 0.17.1.
<details>
<summary>Release notes</summary>
<p><em>Sourced from <a href="https://github.com/kubernetes-sigs/kueue/releases">sigs.k8s.io/kueue's releases</a>.</em></p>
<blockquote>
<h2>v0.17.1</h2>
<p>Changes since <code>v0.17.0</code>:</p>
<h2>Urgent Upgrade Notes</h2>
<h3>(No, really, you MUST read this before you upgrade)</h3>
<ul>
<li>AdmissionChecks: Add the alpha <code>RejectUpdatesToCQWithInvalidOnFlavors</code> feature gate (disabled by default) to reject updates to existing ClusterQueues with invalid <code>AdmissionCheckStrategy.OnFlavors</code> references.
when enabling this feature gate, fix any existing invalid <code>OnFlavors</code> references before updating the affected ClusterQueues. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/10512">#10512</a>, <a href="https://github.com/tenzen-y"><code>@​tenzen-y</code></a>)</li>
</ul>
<h2>Changes by Kind</h2>
<h3>Bug or Regression</h3>
<ul>
<li>
<p>AdmissionChecks: ClusterQueue validation now checks that the flavors specified in <code>AdmissionCheckStrategy.OnFlavors</code> are listed in quota. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/10369">#10369</a>, <a href="https://github.com/ShaanveerS"><code>@​ShaanveerS</code></a>)</p>
</li>
<li>
<p>AdmissionChecks: fix the bug that on backoff admission checks which are spanning all ResourceFlavors, such as MultiKueue, may be missing in the Workload’s status.</p>
<p>For MultiKueue that manifested with a bug, when aside from the MultiKueue admission check there was another non-MultiKueue admission check. In the scenario when eviction on the management cluster happened the manager that had temporarily lost connection to a worker, the remote workload would keep running on the reconnected worker, despite the workload staying without reservation on the manager cluster. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9359">#9359</a>, <a href="https://github.com/Singularity23x0"><code>@​Singularity23x0</code></a>)</p>
</li>
<li>
<p>AdmissionFairSharing: Fixed a bug in entry penalties by reducing them when workload is admitted and also clearing them up if all the resources on the admission entry penalty have value zero. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/10455">#10455</a>, <a href="https://github.com/MaysaMacedo"><code>@​MaysaMacedo</code></a>)</p>
</li>
<li>
<p>ElasticJobs: Fix a bug where pods stay gated after scale-up by allowing finished workloads to ungate their own pods. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/10364">#10364</a>, <a href="https://github.com/sohankunkerkar"><code>@​sohankunkerkar</code></a>)</p>
</li>
<li>
<p>FailureRecoveryPolicy: Fixed an issue where pods could remain stuck terminating if their node became unreachable only after the force-termination timeout had already elapsed. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/10500">#10500</a>, <a href="https://github.com/kshalot"><code>@​kshalot</code></a>)</p>
</li>
<li>
<p>Fix a bug in HA mode that caused follower replicas to retain stale workload caches after deletion. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/10521">#10521</a>, <a href="https://github.com/Ladicle"><code>@​Ladicle</code></a>)</p>
</li>
<li>
<p>Fix a bug where the batch/v1 Job mutating webhook could still run even when the batch/job integration was disabled. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/10328">#10328</a>, <a href="https://github.com/Ladicle"><code>@​Ladicle</code></a>)</p>
</li>
<li>
<p>Fix handling of orphaned workloads which could result in the accumulation of stale workloads
after PodsReady timeout eviction for Deployment-owned pods. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/10274">#10274</a>, <a href="https://github.com/sebest"><code>@​sebest</code></a>)</p>
</li>
<li>
<p>LeaderWorkerSet integration: fix the bug that the PodTemplate metadata wasn't propagated to the Workload's PodSets. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/10399">#10399</a>, <a href="https://github.com/pajakd"><code>@​pajakd</code></a>)</p>
</li>
<li>
<p>MultiKueue: Fixes the bug where a job, after being dispatched to a worker, would not sync correctly after being evicted there. This would also cause its workload to be incorrectly labeled as admitted.</p>
<p>Now the workload and the manager job instance will correctly reflect the evicted state and MultiKueue will perform a fallback, then dispatch remote workloads to all eligible workers again after being evicted from the Worker it was successfully admitted to before. An example of such a case is if the remote instance got preempted on the worker. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/10340">#10340</a>, <a href="https://github.com/Singularity23x0"><code>@​Singularity23x0</code></a>)</p>
</li>
<li>
<p>MultiKueue: fix the bug that when custom admission checks are configured on the manager cluster, other than
the MultiKueue admission check, then the Job may start running on the selected worker before the other admission
checks are satisfied (Ready). We fix the issue by deferring the dispatching of workload until all non-MultiKueue AdmissionChecks become Ready. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/10398">#10398</a>, <a href="https://github.com/mszadkow"><code>@​mszadkow</code></a>)</p>
</li>
<li>
<p>Observability: Fix a bug where kueue_cohort_subtree_admitted_workloads_total and kueue_cohort_subtree_admitted_active_workloads metrics could include results for an implicit root Cohort after deletion of a child Cohort or ClusterQueue. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/10395">#10395</a>, <a href="https://github.com/mbobrovskyi"><code>@​mbobrovskyi</code></a>)</p>
</li>
<li>
<p>Observability: Fix excessive memory overhead in hot code paths by reusing the named logger in NewLogConstructor and avoiding unnecessary logger cloning. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/10393">#10393</a>, <a href="https://github.com/MatteoFari"><code>@​MatteoFari</code></a>)</p>
</li>
<li>
<p>Observability: avoid logging update failures as &quot;error&quot; when they are caused by concurrent object modifications, especially when multiple errors are present.</p>
<p>Example log message: &quot;failed to update MultiKueueCluster status: Operation cannot be fulfilled on multikueueclusters.kueue.x-k8s.io &quot;testing-cluster&quot;: the object has been modified; please apply your changes to the latest version and try again after failing to load client config: open /tmp/kubeconfig no such file or directory&quot; (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/10348">#10348</a>, <a href="https://github.com/mbobrovskyi"><code>@​mbobrovskyi</code></a>)</p>
</li>
<li>
<p>TAS: Fix empty slices for count=0 podSets causing infinite scheduling loop (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/10502">#10502</a>, <a href="https://github.com/jzhaojieh"><code>@​jzhaojieh</code></a>)</p>
</li>
<li>
<p>TAS: fix the bug that Pods which only contain the <code>kueue.x-k8s.io/podset-slice-required-topology</code> or <code>kueue.x-k8s.io/podset-slice-required-topology-constraints</code> as the TAS annotation are not ungated. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/10442">#10442</a>, <a href="https://github.com/tg123"><code>@​tg123</code></a>)</p>
</li>
<li>
<p>TAS: reduce the churn on the TAS-enabled controller, called NonTasUsageReconciler, by skipping triggering
of the Reconcile on Pod changes which are irrelevant from the controller point-of-view. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/10508">#10508</a>, <a href="https://github.com/MatteoFari"><code>@​MatteoFari</code></a>)</p>
</li>
</ul>
</blockquote>
