#### What this PR does / why we need it:

This PR reduces the memory footprint of Kueue's informer caches through two complementary optimizations:

**1. Strip non-scheduling fields from cached Workloads**

Registers a `TransformFunc` on the Workload informer that removes PodTemplateSpec fields Kueue never reads at runtime. For each PodSet template, only scheduling-relevant fields are retained:

- **ObjectMeta**: Labels, Annotations
- **PodSpec**: NodeSelector, Affinity, Tolerations, SchedulingGates, RuntimeClassName, Overhead, ResourceClaims, TopologySpreadConstraints, Priority
- **Containers/InitContainers**: Name, Resources, Ports only

Fields like Image, Command, Args, Env, VolumeMounts, and Volumes are stripped since Kueue only needs resource requests and scheduling constraints for admission and preemption decisions.

The PodSet equality comparison (`ComparePodSets`) is updated to compare only container Resources instead of full container specs, consistent with the stripped cache entries.

**2. Configurable node label prefix filtering for TAS node cache**

Adds a `Resources.ExcludeNodeLabelPrefixes` configuration field that strips infrastructure labels from cached node objects in the TAS node cache. Nodes in large clusters can carry dozens of labels set by cloud providers and infrastructure controllers (e.g., `kubectl.kubernetes.io/`, `cloud.google.com/`, `eks.amazonaws.com/`) that are irrelevant for topology, flavor, or workload scheduling decisions.

A sensible set of default prefixes is provided. Operators can override the list via the Kueue configuration to match their environment.

In large clusters with thousands of Workloads and hundreds of nodes, informer caches become the dominant memory consumer. A significant portion of the cached data consists of fields Kueue never inspects. Stripping these fields at ingestion time reduces per-object memory without any behavioral change.

#### Special notes for your reviewer:

The two optimizations are independent and target different cache domains:
- Commit 1 targets the Workload informer cache (`pkg/config/config.go`, `pkg/util/equality/podset.go`)
- Commit 2 targets the TAS node cache (`pkg/cache/scheduler/tas_flavor.go`, `apis/config/v1beta2/`)

They can be reviewed and tested independently. Happy to split into separate PRs if preferred.

#### Does this PR introduce a user-facing change?

```release-note
Added Resources.ExcludeNodeLabelPrefixes configuration field to strip infrastructure labels from the TAS node cache, reducing memory usage. A default set of common infrastructure label prefixes is provided. Added informer TransformFunc to strip non-scheduling PodTemplateSpec fields from cached Workloads, reducing per-workload memory footprint.
```
