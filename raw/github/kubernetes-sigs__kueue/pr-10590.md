## Description

#### What type of PR is this?

/kind feature

#### What this PR does / why we need it:

Registers a `TransformFunc` on the Workload informer that removes PodTemplateSpec fields Kueue never reads at runtime, reducing per-Workload memory in the informer cache.

For each PodSet template, only scheduling-relevant fields are retained:

- **ObjectMeta**: Labels, Annotations
- **PodSpec**: NodeSelector, Affinity, Tolerations, SchedulingGates, RuntimeClassName, Overhead, ResourceClaims, TopologySpreadConstraints, Priority
- **Containers/InitContainers**: Name, Resources, Ports only

Fields like Image, Command, Args, Env, VolumeMounts, and Volumes are stripped since Kueue only needs resource requests and scheduling constraints for admission and preemption decisions.

The PodSet equality comparison (`ComparePodSets`) is updated to compare only container Resources instead of full container specs, consistent with the stripped cache entries.

In large clusters with thousands of Workloads, informer caches become a dominant memory consumer. A significant portion of the cached PodTemplateSpec consists of fields Kueue never inspects. Stripping these fields at ingestion time reduces per-object memory without any behavioral change.

This PR contains **no API changes**—only internal cache optimization.

**Performance data:**

Good point - I'll run the scheduler performance tests (`make run-performance-scheduler` with `SCALABILITY_MEM_PROFILE=1`) on `main` vs this branch to produce an A/B heap profile comparison. Will post the results.

**Methodology for confirming excluded fields are not used:**

I did a systematic audit of all code paths in `pkg/cache/`, `pkg/scheduler/`, `pkg/controller/core/`, `pkg/controller/tas/`, and `pkg/config/` (excluding test files). Searched for direct access to every stripped Container field (`Image`, `Command`, `Args`, `Env`, `EnvFrom`, `VolumeMounts`, `SecurityContext`, `Lifecycle`, `LivenessProbe`, `ReadinessProbe`, `StartupProbe`, `WorkingDir`, etc.) and PodSpec field (`Volumes`, `RestartPolicy`, `DNSPolicy`, `ServiceAccountName`, `SecurityContext`, `ImagePullSecrets`, etc.).

**Result: zero hits in scheduling-critical code.** The only Container fields accessed are `Resources`, `Ports`, and `Name`. The only PodSpec fields accessed are `NodeSelector`, `Affinity`, `Tolerations`, `SchedulingGates`, `RuntimeClassName`, `Overhead`, `ResourceClaims`, `TopologySpreadConstraints`, and `Priority` - all of which are preserved by the transform.

For DeepEqual safety:
- `comparePodTemplate` in `pkg/util/equality/podset.go` (updated in this PR) now compares only `Tolerations` + container `Resources`.
- The full-Spec comparison in `pkg/cache/queue/cluster_queue.go` is safe because both sides originate from the informer cache, so both are identically stripped.
- All `DeepCopy()` calls in the scheduler operate on already-transformed objects from the queue/informer cache.

Happy to add a more formal "preserved fields" doc comment on `StripWorkloadTransform` to make this explicit for future contributors.

### Comment by [@shshr](https://github.com/shshr) — 2026-04-17T19:42:17Z

## Performance Results: Workload Informer Cache TransformFunc

Benchmark: 15,000 workloads (5 cohorts × 6 queues × [350 small + 100 medium + 50 large]) with realistic container specs including:
- Container images, 15-25 environment variables, 5-10 volume mounts, 2-3 init containers per workload
- Uses the existing `test/performance/scheduler` framework (`minimalkueue` + envtest)

### Results

| Metric | Baseline (`main`) | With TransformFunc | Delta |
|--------|-------------------|-------------------|-------|
| MaxRSS | 1,619,608 KB | 1,609,920 KB | −10 MB (0.6%) |
| **Go heap in-use** | **578.08 MB** | **528.41 MB** | **−49.67 MB (8.6%)** |
| Wall time | 426s | 437s | +2.5% (within noise) |

### Top heap allocation changes (flat)

| Allocation site | Baseline | Optimized | Delta |
|----------------|----------|-----------|-------|
| `reflect.growslice` | 190.74 MB | 171.72 MB | −19.0 MB |
| `FieldsV1.DeepCopyInto` | 17.10 MB | 10.56 MB | −6.5 MB |
| `PodSpec.DeepCopyInto` (cum) | 18.51 MB | 14.01 MB | −4.5 MB |
| `mapassign_faststr0` | 52.03 MB | 49.03 MB | −3.0 MB |
| `WorkloadSpec.DeepCopyInto` | 2.00 MB | <threshold | −2.0 MB |

### Methodology

- Both builds use the same Go version (1.25.9), same test config, same hardware
- Baseline is `main` branch (v0.17.0-dev), optimized is `strip-workload-cache` branch
- Workloads carry realistic PodTemplateSpec payloads (images, env vars, volume mounts, init containers) rather than the default minimal config (CPU-only containers)
- The TransformFunc strips `ManagedFields`, container `Image`/`Env`/`VolumeMounts`/`LivenessProbe`/`ReadinessProbe`/`StartupProbe`/`Lifecycle`, init containers, and volumes from cached Workload objects
- MaxRSS includes Go runtime overhead, OS buffers, and envtest (etcd+apiserver) — the heap delta better isolates the informer cache impact

### Analysis

The 8.6% heap reduction (~50 MB) at 15K workloads demonstrates measurable savings from stripping non-scheduling fields. In production clusters with 50-100K+ workloads carrying heavier PodTemplateSpecs (sidecar containers, dozens of env vars from ConfigMaps/Secrets, complex volume configurations), the savings scale proportionally. The stripped fields (managed fields, probes, lifecycle hooks, images, env vars, volumes, init containers) are confirmed unused by the scheduling path per the field audit in my previous comment.
