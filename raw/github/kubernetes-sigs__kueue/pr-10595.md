## Description

#### What type of PR is this?
/kind cleanup

#### What this PR does / why we need it:
This PR addresses the long execution time of the `test-e2e-customconfigs` suite (which was taking ~32 minutes) by splitting it into two separate, independent suites that can run in parallel:

1. `test/e2e/sequential/baseline/`: Contains tests for core Kueue features that do not require external third-party integrations.
2. `test/e2e/sequential/extended/`: Contains tests requiring external integrations (like Spark, JobSet, etc.), saving significant CI time by skipping integration installations when only core functionality needs testing.

Additionally, this PR includes the following specific runtime optimizations:
- **SparkApplication:** Bumped the CPU and memory requests for the dummy workloads so the pods compute and finish faster.

- Updated `Makefile-test.mk` build targets and `testing.md` documentation to reflect the new structure.

#### Which issue(s) this PR fixes:
Fixes #10472
