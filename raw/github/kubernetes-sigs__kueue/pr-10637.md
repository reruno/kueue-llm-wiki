#### What this PR does / why we need it:
Fixes metric label inconsistency by renaming `detailed_reason` to `underlying_cause` in the `EvictedWorkloadsOnceTotal` metric.

The `EvictedWorkloadsOnceTotal` metric was using a different label name (`detailed_reason`) compared to other eviction metrics (`EvictedWorkloadsTotal` and `LocalQueueEvictedWorkloadsTotal`), which use `underlying_cause`. This inconsistency makes it harder for users to query and correlate eviction metrics.

This PR ensures all eviction-oriented metrics use the same label name and description for consistency.
