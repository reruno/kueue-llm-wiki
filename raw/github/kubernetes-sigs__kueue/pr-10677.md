  When upgrading Kueue, review the upgrade notes according to the minor release

  - If your target version is a patch release, also review the patch release notes

  You do not need to review patch release notes from earlier minor lines
  separately; those changes are covered by the next minor `.0` release notes.

  Let's generate the following text:
  
  For example:
  
  - When upgrading from `v0.15.1` to `v0.17.2`, review the upgrade notes for
    `v0.16.0`, `v0.17.0`, `v0.17.1` and  `v0.17.2`.

  to the latest patch release of the previous minor line, where the user should
  also review the patch release notes in that minor line.
