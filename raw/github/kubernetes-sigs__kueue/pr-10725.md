#### What this PR does / why we need it:
This PR fixes a MultiKueue bug where the manager workload could miss `status.accumulatedPastExecutionTimeSeconds` after a remote workload finished.

When a remote workload finishes, the manager now copies the past execution time and updates the local workload before marking it finished.

### Review by [@kshalot](https://github.com/kshalot) (commented) — 2026-04-24T14:02:10Z

Here's my understanding - Kueue calls `SyncAdmittedCondition` upon eviction:
https://github.com/kubernetes-sigs/kueue/blob/78109e3799c9ff98ada439b99fb92f00c70c634c/pkg/controller/core/workload_controller.go#L559
and `SyncAdmittedCondition` is what updates `AccumulatedPastExecutionTimeSeconds`:
https://github.com/kubernetes-sigs/kueue/blob/78109e3799c9ff98ada439b99fb92f00c70c634c/pkg/workload/admissionchecks.go#L75-L87

So we won't see `AccumulatedPastExecutionTimeSeconds` increase if we do not evict. Eviction also wasn't included in the reproduction steps.

@MatteoFari Could you confirm that if the reproduction steps involve an eviction, the time is increased?
