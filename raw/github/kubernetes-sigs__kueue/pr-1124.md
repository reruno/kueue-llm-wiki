### Review by [@mimowo](https://github.com/mimowo) (commented) — 2023-09-18T07:50:46Z

It looks ok, but I have a slight concern, that if there is a panic thrown between Lock and Unlock the lock will remain. It is usually not an issue if the code in between is very simple (sort of like here), but I have a feeling this code may evolve.

I would suggest to wrap the code into a function call and use Lock / defer Unlock pattern for the duration of the function. This could be either in a named function, like "snapshotQueueData", or an unnamed, like we do for example here: https://github.com/kubernetes-sigs/kueue/blob/0ee53de7f2f10ea7eb129f34c143126bdc002ee1/client-go/informers/externalversions/factory.go#L148-L159.
