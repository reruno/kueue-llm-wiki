  ```golang
  }
  ```
  
  While it is also a double for loop, it's as if it was just one loop, because `j` never decreases.

  I had something like this:
  ```go
          // Match the current state of pod sets
  	// to the pod set info in the workload
  	lastFoundPodSetIdx := -1
  	for i := range jobPodSets {
  		jobPodSetIsFound := false
  		for j := lastFoundPodSetIdx + 1; j < len(wl.Spec.PodSets); j++ {
  			jobPodSet := jobPodSets[i]
  			wlPodSet := wl.Spec.PodSets[j]
  
  			if wlPodSet.Name == jobPodSet.Name {
  				if !workloadFinished && wlPodSet.Count < jobPodSet.Count {
  					return false
  				}
  				jobPodSetIsFound = true
  				lastFoundPodSetIdx = j
  			}
  		}
  		// If actual pod set info has a role that workload doesn't have,
  		// consider workload not an equivalent to the pod group
  		if !jobPodSetIsFound {
  			return false
  		}
  	}
  ```
  
  But decided to drop it because it was harder to read and understand. If you want I can go back to this implementation.
