### Comment by [@alculquicondor](https://github.com/alculquicondor) — 2023-12-04T17:47:26Z

IMO, looking at the "pending workloads" of a (Local/Cluster)Queue makes a lot of sense, and we should keep these endpoints long term.

For the case of workload, it might make sense to just have an endpoint for "workload", period. Although it's unclear which fields we would need for an admitted Workload that isn't already in the Workload status. My main worry is that a user does a GET for "pending workload X" and it gets 404, just because the workload was admitted.
All these implications need to be explored in a future KEP.
