/kind bug

#### What this PR does / why we need it:

Add a hold in the CQ, observe all workloads in metrics and then remove the hold.

This is more reliable than the older approach of creating the localQueue after the workloads, because we can't guarantee Kueue observes workloads and queue updates in order.
