### Review by [@agilgur5](https://github.com/agilgur5) (commented) — 2024-04-26T02:26:42Z

Nice guide, this is helpful!

I couldn't comment on the PR in-line as the fork was deleted (GH was throwing an error), so a comment below:

For the very last line: Did you mean `plugin` instead of `resource`?
`http` and `plugin` actually do create Pods, it's just not documented much (I found this out only after being a contributor for many months. it's very vaguely mentioned [in the `http` template docs](https://argo-workflows.readthedocs.io/en/release-3.4/http-template/#argo-agent) and manifests linked therein). They create an `agent` Pod if one does not exist in the Workflow's namespace. If one already exists, it gets re-used.
