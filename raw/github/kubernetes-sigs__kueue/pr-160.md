  in kube-scheduler, we only create an event if the workload didn't fit, not when there is an error. Errors are usually conflicts on the resource version, or we are trying to schedule something that we already admitted. These errors are internal, in the sense that the user doesn't need to do anything to change the situation (the system will recover).
  
  On the other hand, if the workload doesn't fit, the user could do something, like terminate some running jobs, or request more quota to the admin, etc.
  That would be lines 124-128.
