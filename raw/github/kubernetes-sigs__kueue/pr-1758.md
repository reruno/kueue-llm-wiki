## Description

#### What type of PR is this?

/kind bug

#### What this PR does / why we need it:

Removes fields that are not relevant to queueing or autoscaling from role hash calculation.

In particular, users of pod groups might have labels to identify a particular instance within a group.

In general, Pod labels don't have an influence in kueue or in cluster-autoscaler, so the easiest solution is to remove labels from hash calculations.

We can add pod labels or other fields back in a case-by-case fashion, if we identify any that might lead to different autoscaling behavior.
