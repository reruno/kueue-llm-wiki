#### What this PR does / why we need it:
When provisioning request has Accepted=True and Provisioned is nil we cannot dereference the Provisioned status. Fixing it by checking that Provisioned is explicitly set to False. Testing it by separately setting Accepted=True (leaving Provisioned empty) and only then changing Provisioned=False.
