  Why not just `if p.DeletionTimestamp != nil && p.Spec.NodeName == ""`? 
  
  We know there are no `schedulingGates` if `p.Spec.NodeName != ""` because of this [check](https://github.com/kubernetes/kubernetes/blob/5ceb99dc6bae8a3013192439bdbee942d14de190/pkg/apis/core/validation/validation.go#L4987). Also, a pod with `p.Spec.NodeName == ""` is Pending (or Failed), so no need to check for Pending explicitly.
  
  This would also match the logic in `gc_controller` [link](https://github.com/kubernetes/kubernetes/blob/5ceb99dc6bae8a3013192439bdbee942d14de190/pkg/controller/podgc/gc_controller.go#L303).
  
  For the context, the users may want to implement the analogous logic in their controllers which create replacement pods, so that the replacement pods are safe (not handled as excess pods). So, the simpler the logic, the easier it would be for users to replicate.
