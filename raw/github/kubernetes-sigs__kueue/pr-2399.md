#### Special notes for your reviewer:
Fixes for https://github.com/kubernetes-sigs/kueue/issues/2353#issuecomment-2159897688.

`Active` status calculated by `meta.IsStatusConditionTrue(cq.Status.Conditions, v1beta1.ClusterQueueActive)`. Sometime we don't have time to set this condition to `True` between creation clusterqueues (an `JustBeforeEach`) and print list of clusterqueues. That's why we have `active = false`. So we need to wait time to be sure that status set to `true`.
