  Hm, I think ultimately we will want the MultiKueue Jobs to be transparent to the users, as much as possible. For regular Jobs a user can add a custom condition and it is preserved. If we want to have the same logic for MultiKueue Jobs we would probably need to use SMP (Strategic Merge Patch), and make it strict. Otherwise we may drop user's conditions. 
  
  IIUC dropping user conditions would also happen prior to this PR, so I'm not concerned. However, maybe using strict is more aligned with the long-term goal of mimicking the regular Job API. WDYT @alculquicondor ?
