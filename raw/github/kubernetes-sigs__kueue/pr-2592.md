## Description

#### What type of PR is this?
/kind cleanup

#### What this PR does / why we need it:
In preparation for #79, we will change the Scheduler, FlavorAssigner, and Preemption logic to only interact with the `ClusterQueueSnapshot`'s capacity through high level queries. E.g. "am I borrowing" or "how much capacity do I have left". This PR prepares us to bring all access to `ResourceQuota`/`Usage` behind the `ClusterQueueSnapshot` interface in a subsequent PR.

We don't want to rely on iteration through `ResourceGroups` to find our `ResourceQuota`, but rather, should be able to make O(1) queries to a flat data structure. See [this code](https://github.com/kubernetes-sigs/kueue/blob/542b90cfe294e41b0d677b2606fd2485c704b48a/pkg/scheduler/scheduler.go#L388-L395) for an example how we currently have to find a `ResourceQuota`, if we are not already looping through `ResourceGroups`.

#### Special notes for your reviewer:
I intentionally avoided _any_ simplifications when possible, to keep the diff small
