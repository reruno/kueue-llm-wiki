#### What type of PR is this?

#### What this PR does / why we need it:
Follow up to #2592. In preparation for https://github.com/kubernetes-sigs/kueue/issues/79.

Copied from #2592: we will change the Scheduler, FlavorAssigner, and **Preemption** logic to only interact with the ClusterQueueSnapshot's capacity through high level queries. E.g. "am I borrowing" or "how much capacity do I have left".

Additionally, we fix the test `preempting locally and borrowing other resources in cohort, without
cohort candidates`. Since we were only looping over CQ resource groups, we didn't
find out that we had no capacity in CQ for this flavor [code](https://github.com/kubernetes-sigs/kueue/blob/6b936946662c65e5454821ef74d9c9f5b2c1c546/pkg/scheduler/preemption/preemption.go#L574-L611). We fix this by adding the alpha flavor to that CQ, and adding a CQ which lends resources to those FlavorResources

```release-note
NONE
```
