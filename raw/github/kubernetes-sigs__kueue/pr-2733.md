  I suspect what happens is that we may occasionally hit with SIGTERM the pod before the SIGTERM trap is registered, and then it takes by default 30s before sending SIGKILL. 
  Maybe we can try first setting for this case `TerminationGracePeriod(1).` on the PodSpec and use longer timeout only if the issue repeats?
  
  I agree it is not responsibility of Kueue to test kubelet, but we should also try to keep the tests executing fast, and extending the timeout may cover some possible optimizations.
