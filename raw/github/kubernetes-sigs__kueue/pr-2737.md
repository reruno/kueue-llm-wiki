> This is an interesting extension to the state machine for a Workload. It is getting pretty close to what one might need for a fairly general fault detection/recovery mechanism. Is there interest in pursuing that angle?

Nice to hear we are converging on the approaches for failure recovery :). This KEP is originally motivated for failure recovery in TAS, where the replacement pod is unlikely to find another slot in the failure domain (especially if node is the lowest level). Another idea (but vague at the moment) for TAS is to adjust the admission without eviction, but we are not sure how to do it yet.
