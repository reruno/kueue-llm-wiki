  It makes the current implementation quadratic in complexity. I know the trees are not deep, but if we are going to implement it, then probably better to follow the standard method: https://www.geeksforgeeks.org/detect-cycle-in-a-graph/ .
  
  Also, I'm not convinced we need cycle detection at all:
  1. would it be wrong if there is a cycle as long as a user enables both?
  2. we don't have a use-case that yet.
  
  So I would prefer code simplicity first. We can add it later if really needed.
