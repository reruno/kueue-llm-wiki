1. for consistency - reasons for all other conditions (for example WorkloadEvicted) are in the API package
2. to enable Kueue to make decisions based on the conditions from every package. In particular, when working on https://github.com/kubernetes-sigs/kueue/pull/2807 I consider to use the reclaimWhileBorrowing reason
