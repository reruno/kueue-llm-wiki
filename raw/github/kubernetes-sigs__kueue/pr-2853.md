<h2>v0.24.0</h2>
<p>Highlights from this release include network policy support using <a href="https://github.com/kubernetes-sigs/kube-network-policies">sigs.k8s.io/kube-network-policies</a> (thanks <a href="https://github.com/aojea"><code>@​aojea</code></a>!) and support for building node images from pre-compiled Kubernetes releases (thanks <a href="https://github.com/dims"><code>@​dims</code></a>!).</p>
<p>For building images, see the docs at <a href="https://kind.sigs.k8s.io/docs/user/quick-start/#building-images">https://kind.sigs.k8s.io/docs/user/quick-start/#building-images</a></p>
<!-- raw HTML omitted -->
<ul>
<li>The default node image is now Kubernetes v1.31.0: <code>kindest/node:v1.31.0@sha256:53df588e04085fd41ae12de0c3fe4c72f7013bba32a20e7325357a1ac94ba865</code></li>
</ul>
<!-- raw HTML omitted -->
<ul>
<li><strong>Out-of-the-box support for network policy via <a href="https://github.com/kubernetes-sigs/kube-network-policies">sigs.k8s.io/kube-network-policies</a></strong>
<ul>
<li>Advanced users can still disable kindnetd and install your own CNI with <a href="https://kind.sigs.k8s.io/docs/user/configuration/#disable-default-cni">https://kind.sigs.k8s.io/docs/user/configuration/#disable-default-cni</a>, however note that this is a &quot;power user&quot; feature and KIND does not offer support for any third-party CNI</li>
</ul>
</li>
<li><strong>Support for building node images from existing Kubernetes releases</strong>
<ul>
<li><strong>NOTE</strong>: For Kubernetes releases before v1.31.0, this will result in larger images because kind opted in to compiling out remaining in-tree cloud providers with a build tag when publishing images. For v1.31.0+ there is no difference.</li>
<li>See: <a href="https://kind.sigs.k8s.io/docs/user/quick-start/#building-images">https://kind.sigs.k8s.io/docs/user/quick-start/#building-images</a></li>
</ul>
</li>
<li>Support for loading multiple image archives in <code>kind load image-archive</code>
<ul>
<li><strong>NOTE</strong>: it is still more efficient to do a single archive in most cases</li>
</ul>
</li>
<li>Migrated to skipPhase in Init/JoinConfiguration instead of the kubeadm flags, making it possible for users to cutomize the phases (at your own risk!) with config patches</li>
<li>powershell completion</li>
<li>Updated dependencies, including, but not limited to:
<ul>
<li>containerd 1.7.18</li>
<li>runc 1.1.13</li>
<li>go 1.22.6</li>
<li>CNI plugins to 1.5.1</li>
<li>pause 3.10</li>
</ul>
</li>
<li>Docs and clarification for third party install options including scoop and winget</li>
</ul>
