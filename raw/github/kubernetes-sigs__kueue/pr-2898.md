<h2>v0.6.0</h2>
<h2>Highlights</h2>
<ul>
<li>New JobSet <a href="https://github.com/kubernetes-sigs/jobset/tree/main/keps/262-ConfigurableFailurePolicy">Failure Policy API</a> - allows users to configure different behavior for different types of errors, enabling them to use compute resources more efficiently and improve ML training goodput.</li>
<li>Add <a href="%5B#617%5D(https://redirect.github.com/kubernetes-sigs/jobset/issues/617)">Coordinator field</a> to JobSet spec, enabling user to define a global coordinator pod for distributed ML/HPC workloads. The stable network endpoint for this pod will be added as a label and annotation to every Job and Pod in the JobSet for easy use in application code. A common use case for this is TPU Multislice training with multiple different Job templates. See linked issue for details.</li>
<li>Add <a href="https://redirect.github.com/kubernetes-sigs/jobset/issues/649">global Job index</a> label/annotation to every Job and Pod, which is needed to support TPU Multislice training with multiple different Job templates. See linked issue for details.</li>
<li>Added <a href="https://redirect.github.com/kubernetes-sigs/jobset/issues/613">new metrics</a></li>
<li><a href="%5B#476%5D(https://redirect.github.com/kubernetes-sigs/jobset/issues/476)">Improved test coverage</a></li>
<li>Bug fixes</li>
<li>New examples and documentation</li>
</ul>
