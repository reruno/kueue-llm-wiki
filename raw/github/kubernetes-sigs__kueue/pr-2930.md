## Description

This is a draft PR with a prototype of a resource abstraction capability that we will be discussing in the Batch WG call on 8/29.  It is motivated by improving/simplifying quota management for MIG partitions, and as such currently has some hardwired treatment of those resources for easy prototyping.

After getting feedback from the Batch WG tomorrow on how best to configure a generalized form of this resource abstraction, I will implement the preferred configuration option and eliminate all hardcoded resource names from the patch.

The key idea is to enable a flexible abstraction from the concrete resources specified by the PodSets into resources that are better suited for quota management of highly configurable accelerators like MIG-enabled GPUs.
