#### What this PR does / why we need it:
Before https://github.com/kubernetes-sigs/kueue/pull/263, we can delete a resourceFlavor without the impact of a clusterQueue, but now we added the finalizer to resourceFlavor, we'll wait in termination until the referenced clusterQueue deleted first. The test clusterQueue `Should be inactive when corresponding flavors deleted` is no longer suitable any more.
