<h2>v0.134.0</h2>
<p>Hugo v0.134.0 brings render hooks for tables, ContentWithoutSummary, scoped content render and Obsidian style callout alerts.</p>
<h2>ContentWithoutSummary and scoped content render</h2>
<p>This release finally brings a <a href="https://gohugo.io/methods/page/contentwithoutsummary/"><code>Page.ContentWithoutSummary</code></a> method. This has been a long sought after method where the use case is obvious: render the summary with a <em>more</em> button that renders the ... content without summary. To enable this we have reimplemented the content summary handling in Hugo. This consolidates the 3 <a href="https://gohugo.io/content-management/summaries/#comparison">summary types</a> (<code>auto</code>, <code>manual</code> and <code>frontmatter</code>) – they are now all HTML. This is a slightly breaking change (<code>manual</code> was plain text before), but the new behaviour should be much less confusing and easier to handle in the templates. If you want plain text, pipe it into <code>plainify</code>: <code>{{ .Summary | plainify }}</code>.</p>
<p>Before this release there was one instance of <code>.Content</code> for a given page, it was e.g. not possible to render a page's content slightly different (e.g. different heading levels) on the list pages or the home page. This release anables that with a new <code>Page.Markup</code> with an optional <code>scope</code> argument.</p>
<p>In the list template you would do something like this:</p>
<pre lang="handlebars"><code>{{ range .Pages | first 5 }}
  {{ with .Markup &quot;list&quot; }}
     {{ with .Render }}
       &lt;div&gt;
           &lt;li&gt;{{ .Summary }}&lt;/li&gt; 
           &lt;li&gt;{{ .ContentWithoutSummary }}&lt;/li&gt; 
           &lt;li&gt;{{ .WordCount }}&lt;/li&gt; 
           &lt;li&gt;etc ...&lt;/li&gt; 
       &lt;/div&gt;
     {{ end }}
  {{ end }}
{{ end }}
</code></pre>
<p>You can think of the custom scope <code>list</code> as a cache key: Multiple invocations on the same page with the same scope will give the same return value. You can now pass down some data or configuration to in <code>.Page.Store</code> to the templates that render this markup (shortcodes, render hooks), or you can use the new global <code>hugo.Context.MarkupScope</code> function made for this purpose:</p>
<pre lang="handlebars"><code>{{ if eq hugo.Context.MarkupScope &quot;list&quot; }}
  {{/* Render some list markup. */}}
{{ else }}
  {{/* default */}}
{{ end }}
</code></pre>
<h2>Render hooks for tables</h2>
<p>This has been a long sought after feature. Now you can have full control over how you render your Markdown tables with <a href="https://gohugo.io/render-hooks/tables">table render hooks</a>.</p>
<h2>Obsidian style callout alerts</h2>
<p>Hugo <code>v0.133.0</code> added <a href="https://gohugo.io/render-hooks/blockquotes/">blockquote render hooks</a> with GitHub styled <a href="https://gohugo.io/render-hooks/blockquotes/#alerts">alerts</a>. We have since then gotten some feedback about extended alert syntax in <a href="https://help.obsidian.md/Editing+and+formatting/Callouts#Change+the+title">Obsidian</a>, and in Hugo <code>v0.134.0</code> we now support both.</p>
