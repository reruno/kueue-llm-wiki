### Comment by [@kerthcet](https://github.com/kerthcet) — 2022-08-04T08:10:59Z

This is because we used to generate workloads via job controller, then the priority of the workload will be updated according to the priorityClassName, but now, we create the workloads via `MakeWorkload()` and we don't specify the priority values, If so,  the priority value will always be 0, so the earlier created one will be located at the top of the heap and pop out first.
```
func Priority(w *kueue.Workload) int32 {
	if w.Spec.Priority != nil {
		return *w.Spec.Priority
	}
	// When priority of a running workload is nil, it means it was created at a time
	// that there was no global default priority class and the priority class
	// name of the pod was empty. So, we resolve to the static default priority.
	return constants.DefaultPriority
}
```

What we want to do is just change like below, if so, `PriorityClass` creation seems no longer needed.
From:
```
wlLowPriority := testing.MakeWorkload().PriorityClass(lowPriorityClass.Name).Obj()
```
To:
```
wlLowPriority := testing.MakeWorkload().Priority(priority).Obj()
```
