### Review by [@alculquicondor](https://github.com/alculquicondor) (commented) — 2024-09-17T18:11:57Z

```diff
             cpu: 500m
             memory: 512Mi
       serviceAccountName: controller-manager
+      tolerations:

```

  Yeah, generally, to send a Pod to a specific node or node group, you would use a toleration and a node selector, otherwise the Pod could land on any other node. A "generic" toleration isn't generally problematic because the pod would still schedule if the node doesn't have the taint. We can't say the same about node selectors. Adding a "generic" node selector would make the Kueue pod unschedulable in almost every cluster.
  
  Regardless, I haven't seen other projects that provide a toleration as part of their release manifests. You are already expected to modify a release manifest to match your needs (like increasing resource requests or changing configurations), or using kustomize or helm charts. I don't think we should be adding this by default.
