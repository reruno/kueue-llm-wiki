  > "ensure the field is entirely absent from API responses when empty (optional fields must be pointers)
  
  https://github.com/kubernetes/community/blob/master/contributors/devel/sig-architecture/api_changes.md#new-field-in-existing-api-version
  
  Addtiaionally, the status document doesn't mention anything for the pointer: https://github.com/kubernetes/community/blob/master/contributors/devel/sig-architecture/api-conventions.md#spec-and-status
  
  The key point is, can we get rid of this empty status field to avoid confusion? After that, can we add the status field as a non-pointer field?
  @alculquicondor Do you have any knowledge?
