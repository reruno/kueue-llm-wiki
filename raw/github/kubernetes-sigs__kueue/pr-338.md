#### What this PR does / why we need it:

Also:
- validate that queueName is a valid reference name.
- validate that admission can be unset

  The current logic says: when the pod is admitted, the queue name can't be changed. This implies:
  
  - When a workload is being admitted (from `admission==nil` to `admission!=nil`) the queueName shouldn't be changed (this includes from empty queueName to something).
  - When a workload is being preempted (from `admission!=nil` to `admission==nil`) the queueName can be changed.
  - When a workload stays admitted, the queueName can't be changed. If we added `&& oldObj.Spec.QueueName != ""`, you could end up in a case where the queueName changes from empty to something, which should be invalid.
  
  Did I miss a case?
