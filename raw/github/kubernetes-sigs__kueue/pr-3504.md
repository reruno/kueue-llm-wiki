<blockquote>
<h2>v0.138.0</h2>
<p>This release soft deprecates <a href="https://gohugo.io/methods/page/scratch/">Page.Scratch</a> and aliases it to <a href="https://gohugo.io/methods/page/store/">Page.Store</a>. The only difference between the two has been that the <code>Page.Scratch</code> was reset on rebuilds, which becomes problematic when state is written by cached content (typically <code>.Content</code>). The original motivation behind adding a new and not changing the existing method was to avoid breakage, but in hind-sight, it would have fixed more than it broke.</p>
<p>This release also includes a fix for a data race that I, <a href="https://github.com/bep"><code>@ÔÇőbep</code></a>, introduced in <code>v0.137.0</code>.</p>

<h2>v0.137.0</h2>
<p>Note that we have no longer build the <a href="https://gohugo.io/hosting-and-deployment/hugo-deploy/">deploy feature</a> in the standard and extended archives. If you need that, download archives with <code>withdeploy</code> in the filename. See <a href="https://gohugo.io/installation/">instalation</a> for details. You can read <a href="https://redirect.github.com/gohugoio/hugo/issues/12994">#12994</a> for more on the motivation behind this, but for one, it shaves off about 40% of the binary size.</p>
