  > I'm not still sure the reason. For example, why not change the typed to ResourceFlavorReference in the following?
  
  That change is indeed in this PR: https://github.com/kaisoz/kueue/blob/4a77f59347fac56daa2260e6e730178f64d7f4c7/pkg/cache/clusterqueue.go#L605
  
  > I'm wondering if we can get rid of this converter once we replace all string flavor names with ResourceFlavorReference typed.
  
  The converter is not needed. My initial approach was to use `ResourceFlavorReference` everywhere. Then I thought that casting the `ResourceFlavor.Name` (a string) to a `ResourceFlavorReference` could be written in a more concise way (it can get lengthy, especially in test code). For example, this:
  
  ```
  c.resourceFlavors[kueue.ResourceFlavorReference(rf.Name)] = rf
  ```
  could be written like this
  
  ```
  c.resourceFlavors[rf.Ref()] = rf
  ```
  
  However, as I discussed with @mimowo on Slack, I was concerned about this approach since I don't see methods being used in API files. That's why I wrote this proposal to get your opinions. My bad for not adding this concern to the PR description 🤦🏻
  
  After hearing your thoughts, I’ll use the cast to ResourceFlavorReference everywhere instead of the converter 😊
