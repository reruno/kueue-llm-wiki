<details>
<summary>Release notes</summary>
<p><em>Sourced from <a href="https://github.com/fkhadra/react-toastify/releases">react-toastify's releases</a>.</em></p>
<blockquote>
<h2>v11</h2>
<h2>What is new in v11</h2>
<p>I’m super excited about this release! The main focus was on customization, and my goal was to empower you (and myself) so you can fully personalize the look and feel for the notifications.</p>
<p>In short, react-toastify should be able to blend into any design system.</p>
<p><img src="https://github.com/user-attachments/assets/6e3db43b-59e5-44e6-b950-061c0dc30fe4" alt="Screenshot 2024-12-12 at 22 21 52" /></p>
<h3>No need to import the css file anymore</h3>
<p>The stylesheet is now injected automatically, so you no longer need to import it. The CSS file is still exported by the library.</p>
<pre lang="tsx"><code>  import { ToastContainer, toast } from 'react-toastify';
<p>function App(){
const notify = () =&gt; toast(&quot;Wow so easy !&quot;);</p>
<pre><code>return (
  &amp;lt;div&amp;gt;
    &amp;lt;button onClick={notify}&amp;gt;Notify !&amp;lt;/button&amp;gt;
    &amp;lt;ToastContainer /&amp;gt;
  &amp;lt;/div&amp;gt;
);
</code></pre>
<p>}
</code></pre></p>
<h3>Easy customization!</h3>
<p>One of the top requests has been how to customize notifications. To be fair, until this release, it was quite challenging because it required users to override numerous CSS classes.</p>
<p>I’ve simplified the DOM structure of the notification by removing extraneous div elements, nested elements, etc... It’s a significant breaking change, but it’s truly worth the effort. I can confidently say that the library can now seamlessly integrate into any design system.</p>
<p>Below, I’ve implemented a couple of different designs using only Tailwind. <strong>I didn’t override a single CSS class from react-toastify 🤯!</strong></p>
<p><img src="https://github.com/user-attachments/assets/6e3db43b-59e5-44e6-b950-061c0dc30fe4" alt="Screenshot 2024-12-12 at 22 21 52" /></p>
<blockquote>
<p>[!NOTE]<br />
Head to <a href="https://stackblitz.com/edit/react-toastify-how-to-style?file=src%2FApp.tsx">stackblitz</a> to check the code.</p>
</blockquote>
<p>How does it work in practice? On the left side, we have the old DOM structure vs the new one on the right side.</p>
<p><img src="https://github.com/user-attachments/assets/32570fb3-ebbc-4497-b647-1cd6f4b124d9" alt="noti-struct" /></p>
<ul>
<li><code>Toastify__toast-body</code> and its child are now completely gone.</li>
<li>The <code>CloseButton</code> now uses an absolute position.</li>
</ul>
<p>Thanks to those changes, <strong>nothing will interfere with your content.</strong></p>
