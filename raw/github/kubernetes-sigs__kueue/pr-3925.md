  For reproducible and reliable deployment of kueue in production environments, I think users (including us) would consider it important to follow standard chart versioning practices. If every published chart is overwriting the same version, users have no control or visibility over changes, and no way to roll back to a previous working version if an update causes an issue. Moreover, you could have multiple installations of "v0.1.0" e.g. on different clusters, each of which could be completely different depending on the time it was installed.
  
  Updating only the appVersion effectively makes the problem worse, in the sense that there are more kinds of changes in the content of the chart that are not represented in the version of the chart.
