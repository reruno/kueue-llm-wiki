<blockquote>
<h2>v0.140.2</h2>
<p>The timing of this release comes from the security fix in <code>golang.org/x/net</code>'s <code>html.Parse</code> function. This is used in two places in Hugo:</p>
<ol>
<li>Extracting table of contents from Asciidoctor rendered output.</li>
<li>Collecting HTML classes etc. when build stats is <a href="https://gohugo.io/getting-started/configuration-build/#configure-build-stats">enabled</a></li>
</ol>
<p>It's a little bit of a stretch to see how this could be exploited in Hugo, but we understand that many want a clean security report. See <a href="https://redirect.github.com/golang/go/issues/70179">this issue</a> for details.</p>
