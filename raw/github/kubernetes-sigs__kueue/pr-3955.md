  Actually, after the second thought I think this test is ok in the current form:
  1. delete the topology
  2. remove the finalizer
  3. wait for the topology to be gone
  
  I was thinking about testing also the other order:
  1. remove the finalizer
  2. delete the topology
  3. wait for the topology to be gone
  
  However, in fact, this flow will not work by design, as after (1.)  the user will race with the controller - the controller will try to add back the finalizer while the user tries to delete the topology.
  
  So, I think the coverage is good.
