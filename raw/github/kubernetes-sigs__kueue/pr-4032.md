#### What this PR does / why we need it:

This adds [kubeflow/spark-operator](https://github.com/kubeflow/spark-operator)'s `SparkApplication` integration in Kueue. 

Technically, Kueue's [Plain Pods](https://kueue.sigs.k8s.io/docs/tasks/run/plain_pods/) integration can handle `SparkApplication` pods by putting `kueue.x-k8s.io/*` labels both in driver/executor pod template. However, `SparkApplication` integration can give simpler, better UX and deeper integrations for some Kueue's unique features: [Topology Aware Scheduling](https://kueue.sigs.k8s.io/docs/concepts/topology_aware_scheduling/), [All-or-Nothing Scheduling](https://kueue.sigs.k8s.io/docs/tasks/manage/setup_wait_for_pods_ready/), etc.
