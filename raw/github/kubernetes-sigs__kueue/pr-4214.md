### Comment by [@mimowo](https://github.com/mimowo) — 2025-02-11T06:57:26Z

My view is consistent with @tenzen-y here:
1. we don't recommend users to rely on staging,
2. we often ask our users before a release to evaluate a fix just after merge to the main branch, so having the images pushed is very useful
3. the build time of the main branch and release branches is quite imprortant for optimizing the release process, as it requires a couple of steps waiting for completion sequentially
4. with the parallelism of 3 we expect to be able to utilize the stronger machines well, not just book them.

To validate 4. I assume we need to merge and compare the build time gain and the actual resource utilization ( I don't think it can be assessed reliably without merging and an actual experiment).
