### Comment by [@tenzen-y](https://github.com/tenzen-y) — 2025-02-11T11:16:24Z

@mimowo It seems that we need to revert these changes since the cloudbuild or DinD does not support multiarch-image building by docker-buildx: https://prow.k8s.io/view/gs/kubernetes-ci-logs/logs/post-kueue-push-images/1889255570226024448

As I checked other repositories with concurrent image buildings, they build image only one arch, then merge image manifests into one to support multi-architecture by one tag.

https://storage.googleapis.com/kubernetes-ci-logs/logs/post-cluster-api-push-images/1889239459732393984/artifacts/build.log

```
make ARCH=amd64 docker-build
make ARCH=arm docker-build
make ARCH=arm64 docker-build
make ARCH=ppc64le docker-build
make ARCH=s390x docker-build
```

```
docker manifest create --amend gcr.io/k8s-staging-cluster-api/capd-manager:v20250211-v1.9.0-rc.0-344-g516671b41 gcr.io/k8s-staging-cluster-api/capd-manager-amd64:v20250211-v1.9.0-rc.0-344-g516671b41 gcr.io/k8s-staging-cluster-api/capd-manager-arm:v20250211-v1.9.0-rc.0-344-g516671b41 gcr.io/k8s-staging-cluster-api/capd-manager-arm64:v20250211-v1.9.0-rc.0-344-g516671b41 gcr.io/k8s-staging-cluster-api/capd-manager-ppc64le:v20250211-v1.9.0-rc.0-344-g516671b41 gcr.io/k8s-staging-cluster-api/capd-manager-s390x:v20250211-v1.9.0-rc.0-344-g516671b41
```
