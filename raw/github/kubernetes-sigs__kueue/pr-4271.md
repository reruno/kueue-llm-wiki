Thanks for working on it. I think in the current form it remains inaccurate (still better than it was) as the code accounts 1 pods for each TAS workload domain. This is not necessarily true since the domains may have more pods assigned.

I see two approaches to make this fully correct:
1. propagate the exact numbers via a couple of structures, based on assignment
2. account for all pods one-by-one. Currently we only do it for non-TAS pods, but the same way we could iterate over TAS pods.

I believe (2.) is easier from the code perspective, but I would be leaning to start with (1.) , as (2.) will require iterating over more pods.
