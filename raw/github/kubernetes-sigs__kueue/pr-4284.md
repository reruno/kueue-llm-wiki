### Review by [@tenzen-y](https://github.com/tenzen-y) (commented) — 2025-02-17T18:43:50Z

```diff

 		elapsedTime = r.clock.Since(podsReadyCond.LastTransitionTime.Time)
 	}
-	waitFor := r.waitForPodsReady.timeout - elapsedTime
-	if waitFor < 0 {
-		waitFor = 0
-	}
-	return true, waitFor
+	return true, r.waitForPodsReady.timeout - elapsedTime
```

  As I checked the affection by this change, this change does not have any bug for now since we use this time only if the time is positive number:
  https://github.com/kubernetes-sigs/kueue/blob/7a496666ec137cc08315df1884de623e0d68cd2a/pkg/controller/core/workload_controller.go#L546-L566
  
  However, in the future, we do not have any guarantee. So, I would recommend using the `max` function as @mimowo mentioned above.
  
  ```suggestion
  	return true, max(r.waitForPodsReady.timeout - elapsedTime, 0)
  ```
