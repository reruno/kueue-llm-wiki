  Actually, I'm not worried about -1 in message, we could just do `fitCount = max(fitCount, 0)`.
  
  However, I'm wondering if the -1 might be a sign we are double-counting usage from TAS workloads at both the cache level and the snapshot level. Because for a TAS workload we will call `updateUsage`, and then when creating the snapshot we will call `addUsage` and both functions will increment by 1. So, we may be running out of Pods, which would be a bug.
  
  Let me put on hold to make sure we don't merge accidentally.
  /hold
  I think one option is to pass some extra flag in `addUsage` to discriminate if this is for TAS or non-TAS, but at this point I'm equally good to cherry-pick https://github.com/kubernetes-sigs/kueue/pull/4200 first. I'm leaning to cherry-pick the refactoring, but let me check with @PBundyra .
