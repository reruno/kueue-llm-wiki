### Comment by [@nasedil](https://github.com/nasedil) — 2025-03-03T16:38:26Z

I used `String()` for serialization.  It results in outputs like this.
```
test-just-resource	{"flavor-resource1": "{\"Flavor\":\"testFlavour1\",\"Resource\":\"testResource1\"}"}
test-resource-quantity	{"flavor-resource-quantities": "{{\"Flavor\":\"testFlavour2\",\"Resource\":\"testResource2\"}:5,{\"Flavor\":\"testFlavour1\",\"Resource\":\"testResource1\"}:3}"}
 ```
Not sure how if there's a straightforward way to avoid escape slashes in the output.

I also debugged how `FlavorResource` and `FlavorResourceQuantities` are outputted, they both lead to reflect encoder here [vendor/go.uber.org/zap/zapcore/json_encoder.go](https://github.com/uber-go/zap/blob/6d482535bdd97f4d97b2f9573ac308f1cf9b574e/zapcore/json_encoder.go#L179) which uses go builtins for encoding.  And it automatically encodes `FlavorResource` but not `map[FlavorResource]int64`.  I don't see a straightforward way to generalize encoding for maps, and seems like `String()` is a good enough approach here.

@mimowo @tenzen-y  what do you think?
