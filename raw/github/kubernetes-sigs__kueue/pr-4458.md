<details>
<summary>Release notes</summary>
<p><em>Sourced from <a href="https://github.com/kubernetes-sigs/kind/releases">sigs.k8s.io/kind's releases</a>.</em></p>
<blockquote>
<h2>v0.27.0</h2>
<p><strong>This release moves kind to containerd 2.x</strong> and contains fixes for nerdctl. It also moves Kubernetes to 1.32.2 by default.</p>
<!-- raw HTML omitted -->
<p><strong>WARNING</strong>: kind v0.27.0+ will be required to use <code>kind load ...</code> subcommands with these new containerd 2.0+ images (built by kind v0.27+). For other use cases, the new images should still work with recent kind releases, but are not guaranteed. As always we <em>strongly</em> recommend consuming images by their <code>sha256</code> digest for security and reliability. <a href="https://redirect.github.com/kubernetes-sigs/kind/issues/3853">kubernetes-sigs/kind#3853</a></p>
<p>Older images from recent releases should continue to work with kind v0.27.0+.</p>
<p><strong>NOTE</strong>: As <a href="https://github.com/kubernetes-sigs/kind/releases/tag/v0.20.0">previously warned</a> containerd 2.x requires that you must be using <code>config_path</code> mode for containerd registry config. If you're using the <a href="https://kind.sigs.k8s.io/docs/user/local-registry/">local registry script</a> at, or more recent than <a href="https://github.com/kubernetes-sigs/kind/commit/791b3dc43cb7cc5cd1d1763fa92c95bab4120903">https://github.com/kubernetes-sigs/kind/commit/791b3dc43cb7cc5cd1d1763fa92c95bab4120903</a> (kind v0.20.0 / May 22, 2023) then no changes should be necessary.</p>
<p>The default node image is now <code>kindest/node:v1.32.2@sha256:f226345927d7e348497136874b6d207e0b32cc52154ad8323129352923a3142f</code></p>
<!-- raw HTML omitted -->
<ul>
<li>Updated to containerd 2.x</li>
<li>Updated default node image to Kubernetes 1.32.2</li>
<li>Updated go to 1.23.6</li>
</ul>
<p>Images pre-built for this release:</p>
<ul>
<li>v1.32.2: <code>kindest/node:v1.32.2@sha256:f226345927d7e348497136874b6d207e0b32cc52154ad8323129352923a3142f</code></li>
<li>v1.31.6: <code>kindest/node:v1.31.6@sha256:28b7cbb993dfe093c76641a0c95807637213c9109b761f1d422c2400e22b8e87</code></li>
<li>v1.30.10: <code>kindest/node:v1.30.10@sha256:4de75d0e82481ea846c0ed1de86328d821c1e6a6a91ac37bf804e5313670e507</code></li>
<li>v1.29.14: <code>kindest/node:v1.29.14@sha256:8703bd94ee24e51b778d5556ae310c6c0fa67d761fae6379c8e0bb480e6fea29</code></li>
</ul>
<p><strong>NOTE</strong>: You <em>must</em> use the <code>@sha256</code> digest to guarantee an image built for this release, until such a time as we switch to a different tagging scheme. Even then we will highly encourage digest pinning for security and reproducibility reasons.</p>
<p>See also:</p>
<ul>
<li><a href="https://kind.sigs.k8s.io/docs/user/quick-start/#creating-a-cluster">https://kind.sigs.k8s.io/docs/user/quick-start/#creating-a-cluster</a></li>
<li><a href="https://kind.sigs.k8s.io/docs/user/quick-start/#building-images">https://kind.sigs.k8s.io/docs/user/quick-start/#building-images</a></li>
</ul>
<p>NOTE: These node images support amd64 and arm64, both of our supported platforms. <strong>You must use the same platform as your host,</strong> for more context see <a href="https://redirect.github.com/kubernetes-sigs/kind/issues/2718">kubernetes-sigs/kind#2718</a></p>
<!-- raw HTML omitted -->
<ul>
<li>Compatibility fixes for containerd 2.x</li>
<li>Fix <code>kind get clusters</code> with nerdctl</li>
<li>Statically link CNI binaries to match upstream</li>
<li>Fix no-arguments validation for multiple subcommands</li>
<li>Update shellescape dependency to current vanity import</li>
<li>When building node images: wait for containerd to be ready, and retry image pulls</li>
</ul>
