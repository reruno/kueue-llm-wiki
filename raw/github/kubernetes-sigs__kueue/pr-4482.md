### Comment by [@dgrove-oss](https://github.com/dgrove-oss) — 2025-03-04T18:44:59Z

@mbobrovskyi -- I think this is the root cause of #4477.   We were appending to `activePods` without ever clearing it.  
If we're expecting 2 Pods, but only 1 of them happens to be ready the first time we look, we add 1 to the list.  Then the next time through if both Pods are ready, we appended them both. Thus the length of the array keeps growing: 1, 3, 5, 7, .....

### Review by [@mimowo](https://github.com/mimowo) (commented) — 2025-03-05T07:56:33Z

Indeed, this could be problematic. Assume we wait for two running pods. If we see only one running then we add it to the `activePods`. In the next run we add it again if the second one is not Running yet. As a consequence we may end up with the same pod twice on the list `activePods`. Then, the other pod will not be stopped at all.
