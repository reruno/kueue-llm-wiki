  Yeah, I didn't ask it for performance reasons at all. 
  
  I just considered kind of idiomatic expression in golang to hint the size if known up front, improving readability by setting clear expectations. 
  
  However, the golang style guide discusses this and explicitly says it is not a readability requirement:
  https://google.github.io/styleguide/go/best-practices.html#size-hints
  
  Most code does not need a size hint or preallocation, and can allow the runtime to grow the slice or map as necessary. **It is acceptable to preallocate when the final size is known (e.g. when converting between a map and a slice) but this is not a readability requirement**, and may not be worth the clutter in small cases.
  
  So, up to you.
