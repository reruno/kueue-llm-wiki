```diff
 	BehaviorExitFast = []string{"entrypoint-tester"}

+	// Starts a container runs simple webserver, but does not register any signal handler.
+	// Useful for preemption tests, when SIGTERM signal can come before signal handlers are registered.
+	BehaviorWaitForDeletionRunning = []string{"test-webserver"}
```

  Can we just use it to replace the pause? Searched for other uses of `BehaviorWaitForDeletionFailOnExit` and found this: https://github.com/kubernetes-sigs/kueue/blob/848b8ee9c21a437ce829b04ebeeec28663858857/test/e2e/singlecluster/e2e_test.go#L147
  
  I think this test could also flake if we continue to use the pause as it expects the workload to be failed: https://github.com/kubernetes-sigs/kueue/blob/848b8ee9c21a437ce829b04ebeeec28663858857/test/e2e/singlecluster/e2e_test.go#L207.
  
  
  So, I would suggest to just have 
  
  ```
  // Starts a container which always ends in failure on deletion.
  // To achieve this runs simple webserver, but does not register any signal handler.
  BehaviorWaitForDeletionFailOnExit = []string{"test-webserver"}
  ```
