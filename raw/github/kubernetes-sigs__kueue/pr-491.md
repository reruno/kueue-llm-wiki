#### What this PR does / why we need it:

There are two newly added tests in scheduler_test.go which are failing occasionally due to different reasons: 
- fails due to a conflict when adding `PodsReady` condition to a newly admitted workload when another `WorkloadAdmitted` condition is added later by workload_reconclier.  Fixed by retrying of adding the condition.
- fails due to a random order at which workloads are admitted in the same scheduling cycle. Solved by adding a second before creating the second workload to ensure the order of admission.

#### Which issue(s) this PR fixes:

-->
Fixes #488 

#### Special notes for your reviewer:

The proposed solutions are quick to do, but it seems both issues could be solved better.

First if we used SSA there should be no conflict between PodsReady and WorkloadAdmitted if used different field managers. 

Second we could modify the scheduler code to break the tie in a predictive manner, for example based on workload UIDs. Alternatively we could use a fake clock which will be handy anyway for testing timeouts.

Let me know if a follow up PRs are needed to improve if we are happy with the current solution.
