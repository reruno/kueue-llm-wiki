## Description

#### What type of PR is this?

/kind bug

#### What this PR does / why we need it:

This fixes an inconsistency between Helm chart version in OCI registry and version reported by the chart metadata (`Chart.yaml`)

#### Which issue(s) this PR fixes:

Fixes #4928 

#### Special notes for your reviewer:

Tested locally

* comment the last line of the script (to prevent pushing to registry)
* run `./hack/push-chart.sh`
* go to `bin`
* extract `kueue-0.11.3.tgz`
* `cat kueue/Chart.yaml`
```
> cat Chart.yaml

```yaml
apiVersion: v2
appVersion: v0.11.3-dirty
description: Kueue is a set of APIs and controller for job queueing. It is a job-level
  manager that decides when a job should be admitted to start (as in pods can be created)
  and when it should stop (as in active pods should be deleted).
name: kueue
type: application
version: 0.11.3-dirty
```

#### Does this PR introduce a user-facing change?

No: the chart version reported in the registry stays the same (no `v` prefix); only the chart metadata is changed.
