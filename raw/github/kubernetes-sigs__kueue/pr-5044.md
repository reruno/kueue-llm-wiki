#### What this PR does / why we need it:

[kube-prometheus-stack](https://github.com/prometheus-community/helm-charts/tree/main/charts/kube-prometheus-stack) is a common way to install Prometheus. However, by default, Prometheus only scrapes those ServiceMonitors that have the label `release: $PROMETHEUS_HELM_RELEASE`.

### Comment by [@kannon92](https://github.com/kannon92) — 2025-04-18T19:58:54Z

I don't think this is the right change. You should be able to add labels (kueue.labels) if this is needed.

Can you see if you get this working with our current helm configuration? I think hardcoding a label from other project is not going to scale in the long run. We should have general ways to inject labels.

### Comment by [@kevin85421](https://github.com/kevin85421) — 2025-04-18T20:08:20Z

> I don't think this is the right change. You should be able to add labels (kueue.labels) if this is needed.

Your point makes sense. My concern is that `kueue.labels` is used everywhere, and `release` is not an uncommon label name. I'm a bit concerned that it might overwrite something or trigger something unexpectedly.

### Comment by [@kevin85421](https://github.com/kevin85421) — 2025-04-18T20:12:59Z
