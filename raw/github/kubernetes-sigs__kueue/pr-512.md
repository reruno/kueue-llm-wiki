## Description

#### What type of PR is this?

/kind bug

#### What this PR does / why we need it:

Ensures atomicity when releasing quota and requeuing inadmissible workloads. If not, it's possible that a workload being requeued slips in between the time a workload is deleted from the cache and we requeue inadmissible workloads, violating priority.

#### Special notes for your reviewer:

This is adding a dependency from the queue manager lock into the cache lock. We already had an instance like this via the StatusChecker interface in the manager.
In the future, we need to be careful not to add a lock dependency the other way around.
