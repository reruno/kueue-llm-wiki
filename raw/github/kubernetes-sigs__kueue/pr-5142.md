  > Isn't duplicated with config/components/crd/patches/cainjection_in_clusterqueues.yaml?
  
  Yes, that’s correct. This approach was intentionally [chosen](https://github.com/kubernetes-sigs/kueue/pull/4800#discussion_r2034744666) during our work to add support for metrics certificates. We decided to completely isolate the cert-manager setup, which does result in some duplication. For example, in `test/e2e/config/certmanager`, we are duplicating the webhook patches, controller-manager configuration, and other related files. This was a deliberate design decision.IMO, it doesn't make sense to install those patches for the default setup.
