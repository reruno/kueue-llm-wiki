#### What this PR does / why we need it:

Turns out we didn't need a distinction between two types of preemption (within Cohort and ClusterQueue).

Furthermore, the distinction implied that we could not preempt both from the Cohort and ClusterQueue, which limits preemption.

For example, you could have a case with an incoming workload requiring 2 cores, where 1 can be reclaimed from the cohort and the other from the ClusterQueue.

On the other side, this implies that we need an additional sorting criteria for preemption candidates: we should prioritize preempting workloads from ClusterQueues that are borrowing quota.
