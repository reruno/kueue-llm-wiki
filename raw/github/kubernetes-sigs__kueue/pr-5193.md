#### What this PR does / why we need it:

1. Corrects some misconceptions in the build comments
2. Fixes multi-arch build to use the correct base image. 
   - The previous build "worked" because the base image is all static files, but it will have the metadata for the host platform instead of the target platform. Some container runtime builds out there may have rejected this image.
3. Avoid depending on implicit copy relative path while using explicit entrypoint full path
4. Optimizes iterative build caching for the docker image
5. Fixes development / kind image targets to build for `linux/$(go env GOARCH)` instead of `linux/amd64`, this will work on amd64 or arm64 currently.
