Cleanup motivated by the work on #5276

#### Special notes for your reviewer:

I think this PR greatly simplifies the reasoning about the TAS cache because:
1. When initializing the tasCache it does not rely on the order of adding ResourceFlavor or Topology
2. Eliminates the need for throwing Listing of ResourceFlavors from event handlers ([example](https://github.com/kubernetes-sigs/kueue/pull/5309/files#diff-2335ebd2fdb62e9d56dbd44087a20a915598a260557bd0afd92681e5529f47afL141-L143))
3. Removes the two-level write access to TAS cache, via cache [example](https://github.com/kubernetes-sigs/kueue/pull/5309/files) or tasCache [example](https://github.com/kubernetes-sigs/kueue/pull/5309/files#diff-7f688e0706bb6b6775d35b320ee0d496f92ed367d87cb845979731898ea9d1eaL177), greatly simplifying the flow

Because of (2.) I think this is actually a bugfix, because the error to List() resource flavor from tas/topology_controller.go could prevent initializing the caches.
