1. No drawbacks so far, this will simply not include stdout/stderr in the ginkgo report...
However we rather log it anyways with ginkgoWriter. There is very little or no extra stdout stuff in Kueue.
Usually this option speeds up tests as they capture less output.

For us there shouldn't be many drawbacks, bc we will still see everything printed in build_logs.
The only one might be that whatever we had from the stdout we would not see in the ginkgo reports (json and xml).
It's very hard to spot the difference, as reports differs in range of hundred of bytes (392 bytes to be exact).
