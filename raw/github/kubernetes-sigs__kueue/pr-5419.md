#### What this PR does / why we need it:

When we run `make update-helm`, we get an error

for Mac OS:

```
Error syncing logger: sync /dev/stderr: inappropriate ioctl for device
```

for Linux:

```
Error syncing logger: sync /dev/stderr: invalid argument
```


It indicates that the logger is attempting to call Sync() on a file descriptor like /dev/stderr (or /dev/stdout), which is connected to a console that doesn't support the sync operation. This operation is intended for file-based outputs but fails when the output is a non-file descriptor like a console.

I propose to skip error handling for Sync() operation to do not confuse contributors.
