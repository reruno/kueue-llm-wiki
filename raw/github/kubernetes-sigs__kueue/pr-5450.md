#### What this PR does / why we need it:

The LeastFreeCapacity (LFC) profile in Topology Aware Scheduling (TAS) was incorrectly failing to schedule workloads with 'Required' topology requests in certain scenarios. The previous logic would strictly prioritize the domain with the absolute lowest free capacity, even if that domain was too small to accommodate the entire workload. This led to premature failures when larger, but still suitable domains were available.

This commit refines the `findLevelWithFitDomains` function for the `LeastFreeCapacity` profile when a `Required` topology request is present. It now iterates through the topology domains (which are sorted by increasing free capacity) and selects the *first* domain encountered that has sufficient capacity to fit the entire workload's requested pod count. This guarantees that if a viable single domain exists, it will be chosen, and it will be the one with the least free capacity among all candidates that can fulfill the requirement.

Two new test cases have been added to verify this behavior one for the happy path, the other one for the error handling.
