## Description

In a MultiKueue setup, deleting a Job in the manager cluster does not remove pods in the worker cluster, leading to orphaned pods that continue consuming resources.

#### What this PR does / why we need it:
This change adds `PropagationPolicy: Background` to the `remoteClient.Delete()` call in the MultiKueue adapter to ensure the remote Job's pods are cleaned up properly.

Regarding why the "orphan" mode does not delete the Job - I think this is because in this mode the Job gets the "orphan" finalizer, and awaits for kubernetes GC to add the finalizer to Pods, and only then the Job is deleted.
