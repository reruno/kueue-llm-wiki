```diff
+	IgnoreAnnotation = "kueue.x-k8s.io/ignore"
```

  Can you consider this?
  
  ```suggestion
  	ParentWorkloadAnnotation = "kueue.x-k8s.io/parent-workload"
  ```
  
  And the value would be the name of the workload. Otherwise a user could abuse the `ignore` annotation and add it to any job to run immediately.
  
  This gives us the chance to validate that the Workload is indeed admitted and even suspend the job if it's not. It could even simplify a custom job implementation that only uses Jobs: the CRD itself wouldn't even need a suspend field. Kueue suspends the underlying jobs.
  
  And we probably need a `podset-name` annotation.
  
  Note that for now we still need the suspend field in MPIJob, because the worker pods are not under a job.
  
  cc @mwielgus
