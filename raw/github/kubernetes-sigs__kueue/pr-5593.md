#### What this PR does / why we need it:
We decided to drop the TAS e2e tests for now (see https://github.com/kubernetes-sigs/kueue/issues/5558#issuecomment-2955767928). The rationale is that the integration tests provide the coverage of the same functionality and that the e2e tests in the current form cause flakiness in other e2e tests (this is because e.g. the test deletes some nodes of the kind cluster which affects other tests).

In the future we will add e2e tests covering the whole TAS node failures handling functionality.
