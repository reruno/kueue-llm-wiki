<summary>Release notes</summary>
<p><em>Sourced from <a href="https://github.com/cert-manager/cert-manager/releases">github.com/cert-manager/cert-manager's releases</a>.</em></p>
<blockquote>
<h2>v1.18.0</h2>
<p>cert-manager is the easiest way to automatically manage certificates in Kubernetes and OpenShift clusters.</p>
<p>cert-manager 1.18 introduces several new features and breaking changes. Highlights include support for ACME certificate profiles, a new default for <code>Certificate.Spec.PrivateKey.RotationPolicy</code> now set to <code>Always</code> (breaking change), and the default <code>Certificate.Spec.RevisionHistoryLimit</code> now set to <code>1</code> (potentially breaking).</p>
<blockquote>
<p>ℹ️ Be sure to review all new features and changes below, and read the <a href="https://cert-manager.io/docs/releases/release-notes/release-notes-1.18">full release notes</a> carefully before upgrading.</p>
</blockquote>
<h3>Known Issues</h3>
<ul>
<li>ACME HTTP01 challenge paths are rejected by the ingress-nginx validating webhook (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7791">#7791</a>)</li>
</ul>
<p>Changes since <code>v1.17.2</code>:</p>
<h3>Feature</h3>
<ul>
<li>Add config to the Vault issuer to allow the server-name to be specified when validating the certificates the Vault server presents. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7663">#7663</a>, <a href="https://github.com/ThatsMrTalbot"><code>@​ThatsMrTalbot</code></a>)</li>
<li>Added <code>app.kubernetes.io/managed-by: cert-manager</code> label to the created Let's Encrypt account keys (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7577">#7577</a>, <a href="https://github.com/terinjokes"><code>@​terinjokes</code></a>)</li>
<li>Added certificate issuance and expiration time metrics (<code>certmanager_certificate_not_before_timestamp_seconds</code>, <code>certmanager_certificate_not_after_timestamp_seconds</code>). (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7612">#7612</a>, <a href="https://github.com/solidDoWant"><code>@​solidDoWant</code></a>)</li>
<li>Added ingress-shim option: <code>--extra-certificate-annotations</code>,  which sets a list of annotation keys to be copied from Ingress-like to resulting Certificate object (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7083">#7083</a>, <a href="https://github.com/k0da"><code>@​k0da</code></a>)</li>
<li>Added the <code>iss</code> short name for the cert-manager <code>Issuer</code> resource. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7373">#7373</a>, <a href="https://github.com/SgtCoDFish"><code>@​SgtCoDFish</code></a>)</li>
<li>Added the <code>ciss</code> short name for the cert-manager <code>ClusterIssuer</code> resource (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7373">#7373</a>, <a href="https://github.com/SgtCoDFish"><code>@​SgtCoDFish</code></a>)</li>
<li>Adds the <code>global.rbac.disableHTTPChallengesRole</code> helm value to disable HTTP-01 ACME challenges. This allows cert-manager to drop its permission to create pods, improving security when HTTP-01 challenges are not required. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7666">#7666</a>, <a href="https://github.com/ali-hamza-noor"><code>@​ali-hamza-noor</code></a>)</li>
<li>Allow customizing signature algorithm (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7591">#7591</a>, <a href="https://github.com/tareksha"><code>@​tareksha</code></a>)</li>
<li>Cache the full DNS response and handle TTL expiration in <code>FindZoneByFqdn</code> (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7596">#7596</a>, <a href="https://github.com/ThatsIvan"><code>@​ThatsIvan</code></a>)</li>
<li>Cert-manager now uses a local fork of the golang.org/x/crypto/acme package (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7752">#7752</a>, <a href="https://github.com/wallrj"><code>@​wallrj</code></a>)</li>
<li>Add support for <a href="https://datatracker.ietf.org/doc/draft-aaron-acme-profiles/">ACME profiles extension</a>. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7777">#7777</a>, <a href="https://github.com/wallrj"><code>@​wallrj</code></a>)</li>
<li>Promote the <code>UseDomainQualifiedFinalizer</code> feature to GA. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7735">#7735</a>, <a href="https://github.com/jsoref"><code>@​jsoref</code></a>)</li>
<li>Switched service/servicemon definitions to use port names instead of numbers. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7727">#7727</a>, <a href="https://github.com/jcpunk"><code>@​jcpunk</code></a>)</li>
<li>The default value of <code>Certificate.Spec.PrivateKey.RotationPolicy</code> changed from <code>Never</code> to <code>Always</code>. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7723">#7723</a>, <a href="https://github.com/wallrj"><code>@​wallrj</code></a>)</li>
<li>Potentially breaking: Set the default revisionHistoryLimit to 1 for the CertificateRequest revisions (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7758">#7758</a>, <a href="https://github.com/ali-hamza-noor"><code>@​ali-hamza-noor</code></a>)</li>
</ul>
