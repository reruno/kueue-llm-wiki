<h2>v1.4.0</h2>
<h2>Highlights</h2>
<h3>Enhanced Kubectl Plugin</h3>
<p>KubeRay v1.4.0 introduces major improvements to the Kubectl Plugin:</p>
<ul>
<li>Added a new <code>scale</code> command to scale worker groups in a <code>RayCluster</code>.</li>
<li>Extended the <code>get</code> command to support listing Ray nodes and worker groups.</li>
<li>Improved the <code>create</code> command:
<ul>
<li>Allows overriding default values in config files.</li>
<li>Supports additional fields such as Kubernetes labels and annotations, node selectors, ephemeral storage, <code>ray start</code> parameters, TPUs, autoscaler version, and more.</li>
</ul>
</li>
</ul>
<p>See <a href="https://docs.ray.io/en/master/cluster/kubernetes/user-guides/kubectl-plugin.html">Using the Kubectl Plugin (beta)</a> and <a href="https://redirect.github.com/ray-project/ray/issues/53886">ray-project/ray#53886</a> (link will be updated to the docs site after merging) for more details.</p>
<h3>KubeRay Dashboard (alpha)</h3>
<p>Starting from v1.4.0, you can use the open source dashboard UI for KubeRay. This component is still experimental and not considered ready for production, but feedback is welcome.</p>
<p>KubeRay dashboard is a web-based UI that allows you to view and manage KubeRay resources running on your Kubernetes cluster. It's different from the Ray dashboard, which is a part of the Ray cluster itself. The KubeRay dashboard provides a centralized view of all KubeRay resources.</p>
<p>See <a href="https://redirect.github.com/ray-project/ray/issues/53830">ray-project/ray#53830</a> for more information. (The link will be replaced to doc website after the PR being merged)</p>
<h3>Integration with <code>kubernetes-sigs/scheduler-plugins</code></h3>
<p>Starting with v1.4.0, KubeRay integrates one more scheduler <a href="https://github.com/kubernetes-sigs/scheduler-plugins"><code>kubernetes-sigs/scheduler-plugins</code></a> to support gang scheduling for <code>RayCluster</code> resources. Currently, only single scheduler mode is supported.</p>
<p>See <a href="https://docs.ray.io/en/master/cluster/kubernetes/k8s-ecosystem/scheduler-plugins.html">KubeRay integration with scheduler plugins</a> for details.</p>
<h3>KubeRay APIServer V2 (alpha)</h3>
<p>The new APIServer v2 provides an HTTP proxy interface compatible with the Kubernetes API. It enables users to manage Ray resources using standard Kubernetes clients.</p>
<p>Key features:</p>
<ul>
<li>Full compatibility with Kubernetes OpenAPI Spec and CRDs.</li>
<li>Available as a Go library for building custom proxies with pluggable HTTP middleware.</li>
</ul>
<p>APIServer v1 is now in maintenance mode and will no longer receive new features. v2 is still in alpha. Contributions and feedback are encouraged.</p>
<h3>Service Level Indicator (SLI) Metrics</h3>
<p>KubeRay now includes SLI metrics to help monitor the state and performance of KubeRay resources.</p>
<p>See <a href="https://docs.ray.io/en/master/cluster/kubernetes/k8s-ecosystem/metrics-references.html">KubeRay Metrics Reference</a> for details.</p>
