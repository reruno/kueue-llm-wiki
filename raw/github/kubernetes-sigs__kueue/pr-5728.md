## Description

This change defines dedicated functions to generate the workload name prefix and suffix consistently:

* Prefix is derived from the Job object kind and name.
* Suffix is a pseudorandom string generated from a hash seeded with the Job owner's Name, UID, and GroupVersionKind.

Additional changes:

* Refactored `maxPrefixLength` to use `validation.DNS1123SubdomainMaxLength` for alignment with Kubernetes naming constraints.
* Introduced `nameDelimiter` constant for consistent use between production and test code.
