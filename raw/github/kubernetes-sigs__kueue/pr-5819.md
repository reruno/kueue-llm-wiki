Indeed, Prometheus metrics label requires `[a-zA-Z_:][a-zA-Z0-9_:]*` as you can see in https://prometheus.io/docs/concepts/data_model/#metric-names-and-labels

> Label names SHOULD match the regex [a-zA-Z_][a-zA-Z0-9_]* for the best experience and compatibility (see the warning below). Label names outside of that regex will require quoting e.g. when used in PromQL (see the [UTF-8 guide](https://prometheus.io/docs/guides/utf8/#querying)).

Although this criterion is not "MUST", it is "SHOULD: requirement.

### Comment by [@ichekrygin](https://github.com/ichekrygin) — 2025-07-01T06:27:58Z

> Indeed, Prometheus metrics label requires `[a-zA-Z_:][a-zA-Z0-9_:]*` as you can see in

My emphasis here is on the distinction between Prometheus metric label keys and values.
As far as I can tell, the controller name is used as a label value in Prometheus metrics, not as a key. Since Prometheus only imposes naming restrictions on label keys (not values), any valid UTF-8 string should be acceptable as a label value. Therefore, constraints intended for label keys shouldn't apply in this specific case (controller.name).
