#### Special notes for your reviewer:
Kueue currently doesn't support RayService, and a https://github.com/kubernetes-sigs/kueue/pull/5664 tries to implement RayService controller, webhook and multikueue-adapter. However, this pr won't work as there is asynchronization between RayService's embedded RayCluster object and the RayCluster object created for the target RayService, and this issue cannot be fixed by Kueue or current RayService controller on KubeRay side. On the other hand, RayService's label is propagated to the RayCluster for it, so it's able to let Kueue manage a RayService through its RayCluster. Therefore, a doc is needed to explain the process.

add_quota_to_clusterqueue() {
   log_info "Adding sufficient quota to ClusterQueue..."

   kubectl patch clusterqueue cluster-queue --type='json' -p='[{"op": "replace", "path": "/spec/resourceGroups/0/flavors/0/resources/0/nominalQuota", "value": "10"}, {"op": "replace", "path": "/spec/resourceGroups/0/flavors/0/resources/1/nominalQuota", "value": "20Gi"}]'

   # Verify ClusterQueue nominalQuota after patch

  Hi [mbobrovskyi](https://github.com/mbobrovskyi), thanks for your comments. Yes, Kueue currently doesn't support RayService, but it's able to make Kueue manage RayService through RayCluster. Here is some context:
  A [pr](https://github.com/kubernetes-sigs/kueue/pull/5664) tried to implement RayService controller, webhook and multikueue-adapter. However, this pr won't work as there is asynchronization between RayService's embedded RayCluster object and the RayCluster object created for the target RayService, and this issue cannot be fixed by Kueue or current RayService controller on KubeRay. On the other hand, RayService's label is propagated to the RayCluster for it, so it's able to let Kueue manage a RayService through its RayCluster. Therefore, a doc is needed to explain the process.
  That's why I think if a Kueue version supports RayCluster, it supports RayService.
