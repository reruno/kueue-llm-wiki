<h2>v2.8.2</h2>
<h2>2.8.2</h2>
<p>Ginkgo now includes a <code>tools.go</code> file in the root directory of the <code>ginkgo</code> package.  This should allow modules that simply <code>go get github.com/onsi/ginkgo/v2</code> to also pull in the CLI dependencies.  This obviates the need for consumers of Ginkgo to have their own <code>tools.go</code> file and makes it simpler to ensure that the version of the <code>ginkgo</code> CLI being used matches the version of the library.  You can simply run <code>go run github.com/onsi/ginkgo/v2/ginkgo</code> to run the version of the cli associated with your package go.mod.</p>
<h3>Maintenance</h3>
<ul>
