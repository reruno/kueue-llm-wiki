<h2>v1.18.2</h2>
<p>cert-manager is the easiest way to automatically manage certificates in Kubernetes and OpenShift clusters.</p>
<p>We fixed a bug in the CSR's name constraints construction (only applies if you have enabled the <code>NameConstraints</code> feature gate).
We dropped the new <code>global.rbac.disableHTTPChallengesRole</code> Helm option due to a bug we found, this feature will be released in <code>v1.19</code> instead.</p>
<p>Changes since <code>v1.18.1</code>:</p>
<h3>Bug or Regression</h3>
<ul>
<li>BUGFIX: permitted URI domains were incorrectly used to set the excluded URI domains in the CSR's name constraints (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7833">#7833</a>, <a href="https://github.com/cert-manager-bot"><code>@​cert-manager-bot</code></a>)</li>
<li>Reverted adding the <code>global.rbac.disableHTTPChallengesRole</code> Helm option. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7837">#7837</a>, <a href="https://github.com/cert-manager-bot"><code>@​cert-manager-bot</code></a>)</li>
</ul>
