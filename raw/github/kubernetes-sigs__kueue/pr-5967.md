### Comment by [@mykysha](https://github.com/mykysha) — 2025-07-18T14:20:48Z

The issue with the failing tests was related to the fact that the PodSets for the pods always set their count to 1, making all the slice configurations with size larger than 1 to fail, as was the case with the tests, so in the current version the upper slice bound is disabled for pods.

Another notable finding is that the same behavior happens for LeaderWorkerSet (as well as any job with leader and workers), where there is a mandatory PodSet with a count set to 1 existing, and it is unclear whether this behavior is expected and the user should be unable to create slices with a size larger than 1.
