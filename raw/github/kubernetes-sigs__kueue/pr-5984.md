### Comment by [@mimowo](https://github.com/mimowo) — 2025-07-16T06:49:45Z

/release-note-edit
```release-note
Emit the Workload event indicating eviction when LocalQueue is stopped
```
I call it a bug, because we emit the analogous event in all other situations of Workload eviction. In particular when ClusterQueue is stopped (stopPolicy set to HoldAndDrain) I tested with `k describe wl/wl-name>` and got events:
```
Events:
  Type    Reason                           Age   From                       Message
  ----    ------                           ----  ----                       -------
  Normal  QuotaReserved                    103s  kueue-admission            Quota reserved in ClusterQueue cluster-queue, wait time since queued was 1s
  Normal  Admitted                         103s  kueue-admission            Admitted by ClusterQueue cluster-queue, wait time since reservation was 0s
  Normal  EvictedDueToClusterQueueStopped  25s   kueue-workload-controller  The ClusterQueue is stopped
```
