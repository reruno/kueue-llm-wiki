### Review by [@mimowo](https://github.com/mimowo) (commented) — 2025-07-16T10:25:50Z

- **pkg/config/validation.go:1** by [@mimowo](https://github.com/mimowo) — 2025-07-16T10:25:50Z
  TLDR: I'm ok removing the IFs as we graduate to GA.
  
  Actually, before the [Emulated versions in k8s](https://github.com/kubernetes/enhancements/blob/master/keps/sig-architecture/4330-compatibility-versions/README.md), we would also drop the IFs when graduating to GA. 
  
  IIUC the emulated versions KEP was introduced in core k8s to support safe upgrades across multiple versions in HA mode. TBH, it seems like a lot of work, and testing needed to claim we support it well in Kueue.
  
  So, I would suggest not making that claim, and just delete the IFs as we graduate to GA. I'm open for adoption of the support of safe upgrades via Emulated Versions in Kueue, but I think this should be a dedicated Issue in Kueue, then we may start requiring keeping the IFs longer.
