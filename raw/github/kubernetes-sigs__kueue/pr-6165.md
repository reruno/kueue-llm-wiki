## Description

I'm submitting this PR just to provide an example where the scheduler can't preempt for more than one cluster queue at a time even if their are available non-preempted targets. The head of each cluster queue would get the same preemption target (assuming it's feasible) even when other preemption targets are available.

This isn't meant to be merged - I'll be closing this PR mostly to leave it as an example that can be linked to in issues/discussions
