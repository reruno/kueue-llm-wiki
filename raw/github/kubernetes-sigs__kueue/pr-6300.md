#### What this PR does / why we need it:

As I described in https://github.com/kubernetes-sigs/kueue/issues/6256#issuecomment-3136936818, the current Kueue Config clientConnection.qps does not support disabling the client side rate limitter while the controller-runtime supports it by a negative qps value. If we specify a negative value in the field, the Kueue manager will fail to start.

So, I fixed the bug, and support disabling that when they specify a negative value.
