#### Special notes for your reviewer:

The function `workload.ReportEvictedWorkload` was already capable of accounting for PodsReadyToEvictedTimeSeconds. However, it was not called from `applyPreemptionWithSSA` as an exception.

Now, I call the function from `applyPreemptionWithSSA`, and to avoid double counting remove it from workload.ReportPreemption.

We already have the integration test which demonstrates that the metric is accounted: https://github.com/kubernetes-sigs/kueue/blob/db2ccd903ca226be83e4da31879ce55b2a727bd0/test/integration/singlecluster/scheduler/podsready/scheduler_test.go#L198-L199
