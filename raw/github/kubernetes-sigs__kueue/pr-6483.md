## Description

#### What type of PR is this?

/kind cleanup

#### What this PR does / why we need it:

Reserving quota should only happen for active CQ. We are bypassing currently the prevention mechanism which makes the tests rely on backdoor behavior. I think the intention of the tests is to reflect business use as close as possible.

#### Which issue(s) this PR fixes:
<!--

-->
Fixes #6311
Part of #6331

#### Special notes for your reviewer:

I propose to drop the `readyAC` (which is actually pending) from tests, its intention is unclear, and it prevents making LQs and CQs active.

I can add a dedicated "When" test case if we figure out what was its intention, but I would prefer not to pollute most of the tests.

There was also another issue which prevented the CQ to be active - the mismatch in created RF name "on-demand" vs "rf1": https://github.com/kubernetes-sigs/kueue/pull/6483/files#diff-b524dd7e94e695c9007bf094b1ff9cd5a94952214c0db7de004c7ce5bea7ed1bR818-R822
