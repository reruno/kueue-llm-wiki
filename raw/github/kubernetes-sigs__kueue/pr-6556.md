### Comment by [@PBundyra](https://github.com/PBundyra) — 2025-08-12T12:08:29Z

Let's start from the last one as it's the simples one:

> Bonus question: why the UsageHalfLifeTime is changed to 1s?

This is so LQ's usage can actually reach threshold in asserts within the given time. Similar change has been in the past here: https://github.com/kubernetes-sigs/kueue/commit/2be7cbd7c1b8b4afcbd0aa6058d846cd9504ce4a#diff-03b8fcadabe2fe0e35d1e1d7d9a56d259b3673a457d895f94e2418525c2a1081R641

> Can you explain (PR description and potentially code comment) why specific values are chosen, it is quite unclear looking from outside.

It's a bit arbitrary but there's definitely some logic behind it. In the tests we want to  simulates what happens if the LQ's usage is at e.g. 4 or 8 CPU level. To get to these threshold we admit some initial workloads. However it take many cycles to get to exactly 4 or 8 CPUs, so instead I set thresholds at a value that is close enough to the final threshold but also doesn't require that many cycles so tests can proceed quicker. That's also the reason behind why I shortened the HalfLifeTime
