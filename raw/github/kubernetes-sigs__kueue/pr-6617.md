#### What this PR does / why we need it:
When there are large values of quotas and (especially) fair weights, the rounding in DRS calculation causes the returned value to sometimes be 0 for borrowing CQs.  

With this PR I would like to make sure that any borrowing CQ gets DRS of at least 1. As expressed in the comment https://github.com/kubernetes-sigs/kueue/blob/6c61dda233c15975247f6338c5c745c976716541/pkg/cache/fair_sharing.go#L41-L48 `If zero, it means that the usage of the ClusterQueue is below the nominal` -- this PR is enforcing this rule. It was previously violated as all of the testcases added to this PR are failing without the fix.

This PR in my opinion should fix #6577
