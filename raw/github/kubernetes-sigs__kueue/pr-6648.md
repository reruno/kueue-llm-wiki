#### What this PR does / why we need it:
The annotation was planned to be only a temporary measure. With each new feature in this area it becomes harder to remove. 

Currently, when node-failure-controller has to evict workload, it needs to make a second update to clear the nodeToReplace. With this PR, the eviction of a workload will cause the new field to be cleared (in a single patch, together with admission fields).
