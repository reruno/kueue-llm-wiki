#### What this PR does / why we need it:

It allows conveniently restricting a unit test run (with `make test`) to a specific folder, in a way analogous to [what's described](https://github.com/kubernetes/community/blob/master/contributors/devel/sig-testing/testing.md#run-unit-tests-from-certain-packages) in the Kubernetes Contributor's Guide.

The argument is optional; plain `make test` still behaves in the same way.
