## Description

#### What type of PR is this?

/kind cleanup

#### What this PR does / why we need it:

In TAS unit tests, the test data includes nodes with hostnames arranged in lexicographical. This might potentially hide some bugs, as we're inadvertently relying on the initial sort order.

This PR introduces some randomness to the ordering of the test node hostnames.

#### Which issue(s) this PR fixes:
Part of #5465

#### Special notes for your reviewer:
I've rearranged the hostnames in the following manner: x1->x3, x2->x5, x3->x1, x4->x6, x5->x2, x6->x4.
The first commit drops the sorting `wantAssignment`, as we can't rely on sorting by the names anymore. Instead, we ensure the order in `wantAssignment` is provided correctly.
The second commit rearranges the hostnames. This did have impact on the topology assignments in some tests:

- in 11 cases, the _ordering_ of domains changed
- in 3 cases, different nodes ended up in the domains:
  - `find topology assignment for two podsets with the same group - multiple hosts`
  - `block required for podset; rack required for slices; BestFit`
  - `block preferred for podset; host required for slices; 2 blocks with unbalanced subdomains; BestFit`

I simply changed the tests' expected values to match the impact, but I'm not knowledgeable enough (yet) to figure out if these changes are WAI, or if we actually discovered some issues. Let's discuss this further.
