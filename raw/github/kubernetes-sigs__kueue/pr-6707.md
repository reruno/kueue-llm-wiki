#### Special notes for your reviewer:
For scenarios with 4 nodes, I reordered: 1->3 2->1 3->4 4->2; for scenarios with 2 nodes I swapped them.

The change produced _reordering_ of expected results within `should evict the Workload if no replacement is possible and place it on another rack`: on line 1470, and on line 1506. I assume this is WAI, as in cases where 2 nodes are topologically the same, TAS algorithm picks them in lexicographical order. Hence, I swapped the expected results to match the new test results.

I've set the description to say that this PR fixes #5465, as the unit tests have been covered in my previous pull request, and I didn't find any hostnames to be swapped for e2e tests - though please correct me if I'm wrong.
