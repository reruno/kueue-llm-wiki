## Description

We've observed that inactive workloads (.spec.active=false) occasionally get admitted and are then immediately evicted.

After investigation, we found this is triggered by runtime class
 reconciliation, which pushes all pending workloads back onto the
 internal heap, regardless of their active status. This causes
 the scheduler to consider them, even though they're inactive.

Added a fix to check for if the workload is active.
