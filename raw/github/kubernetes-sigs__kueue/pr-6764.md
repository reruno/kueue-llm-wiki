#### What this PR does / why we need it:
- Moves `CandidatesOrdering` function to different package to avoid cyclic dependency 
- Uses `CandidatesOrdering` function in FS preemption algorithm in case of two ClusterQueues draw
- Adds logger to `TargetClusterQueueOrdering` so it can call `CandidatesOrdering`
- Adds integration test that tests the scenario described in the issue. Before the change the test failed
