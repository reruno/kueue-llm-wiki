#### What this PR does / why we need it:
A small cleanup of TAS -- it looks like we have a lot of duplicated code that do the same logic but for different fields (slice/noSlice, leader/noLeader). In this PR I would like to have the logic in one place and select the appropriate fields depending on the context. Also, it will make it easier to get rid of some of the extra fields (if we ever want to).

I'm planning to introduce the changes for balanced TAS but without this PR I'm afraid I would have to introduce the same changes in multiple places.
