#### What this PR does / why we need it:
As PatchStatus will be used as alternative to ApplyAdmissionStatus it requires to handle strict option.
However the final solution to approach some existing use cases may require changes in the function signature.
To avoid changing the signature we will move forward with variadic list of options.

  I'm thinking we could instead use it as an opportunity to introduce a variadic list of options.
  
  This would be WithNonStrict, and only passed if needed. In the future we will need it for:
  - passing the field manager which is optional for Patch, and only requires for Apply
  - maybe passing "currWorkload" along with "origWorkload" (but allow passing only one without modifying interface)
