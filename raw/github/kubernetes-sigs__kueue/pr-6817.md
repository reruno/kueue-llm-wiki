### Comment by [@mimowo](https://github.com/mimowo) — 2025-09-15T14:02:39Z

Unfortunately this is generally considered in K8s not a good practice, because it could leak important information between namespaces and tenants.  

EDIT: it is fine to log the full preemptor name, but not expose in the user-facing API.

For the same reason the full name of a preempted Pod is not logged in the core k8s: https://github.com/kubernetes/kubernetes/blob/a4981e1cf3c747b2224fc5d138e23f21f24cc0a5/pkg/scheduler/framework/preemption/preemption.go#L176-L199 (note how the name is logged but absent in the condition message).

That was the decision discussed in the past by sig-security and api-reviewers.
