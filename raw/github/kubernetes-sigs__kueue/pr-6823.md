### Comment by [@tenzen-y](https://github.com/tenzen-y) — 2025-09-15T05:16:15Z

I'm wondering if we should keep using v0.21.x since v0.22.0 has been accidentally released, even though the Apply function does not support SubResource yet, and then v0.22.1 has been released to revert something associated with Apply.

After the controller-runtime Apply function supports SubResource, I guess that we can remove `TreatSSAAsStrategicMerge` completely because the native controller-runtime Apply function provides the fake client as well.

### Comment by [@mbobrovskyi](https://github.com/mbobrovskyi) — 2025-09-15T06:38:41Z

> After the controller-runtime Apply function supports SubResource, I guess that we can remove TreatSSAAsStrategicMerge completely because the native controller-runtime Apply function provides the fake client as well.

I also tried removing the `TreatSSAAsStrategicMerge` wrapper, but the apply operation didn't work as expected. I suspect the issue is in this section of the code: https://github.com/kubernetes-sigs/controller-runtime/blob/157e6d282d89309e0a203be0324bdef2dd277263/pkg/client/fake/client.go#L1558-L1560. In statusPatch, we're calling sw.client.Patch(), which updates the spec as well. However, we're not sending the spec at all during a patch apply, leading to another error where PodSets are updated to nil.
