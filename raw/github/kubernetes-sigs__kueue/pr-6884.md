#### What this PR does / why we need it:
If a workload has an incomplete/invalid TopologyAssignment (with a mismatch between the number of pods and the total count in the assignment) then the topology ungater panics. This PR fixes the panic by validating the assignment before proceeding. 

A separate concern is why a workload might get such an invalid assignment.
