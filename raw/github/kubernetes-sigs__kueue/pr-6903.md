#### Special notes for your reviewer:
This PR adds active condition check during `workload.SetQuotaReservation` to explicitly deactivatie Evicted and Preempted when quota is reserved, and only annotating their previous message once, when they transition from True to False. 

Note: I am not 💯 % sure whether repeated calls to **SetQuotaReservation** occur in production, or how frequently. If they do, the only observable change would be a stable `LastTransitionTime` on the deactivated (False) Evicted and Preempted conditions. For now, I’m marking the user-facing change as **NONE**.
