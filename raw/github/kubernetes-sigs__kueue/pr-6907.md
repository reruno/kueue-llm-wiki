#### What this PR does / why we need it:
Ref: https://github.com/kubernetes-sigs/kueue/issues/6878

Transformation is an excellent feature that allows modification of the corresponding resources in a `ClusterQueue`. Currently, I have encountered a scenario where I need to modify the value of one resource, and this value depends on another resource. Therefore, I have added an `Operation` to address this requirement: it enables calculations on the input resource using common operations such as multiplication, addition, and subtraction, and the target of the calculation can be either a fixed value or another resource.

1. a fixed value:
```
    resources:
      excludeResourcePrefixes: []
      transformations:
      - input: nvidia.com/gpucores
        strategy: Retain
        operation:
          type: Mul
          target:
             quantity: 2
      - input: nvidia.com/gpumem
        strategy: Retain
        operation:
          type: Mul
          target:
             quantity: 2
```

2. depends on another resource:
```
    resources:
      excludeResourcePrefixes: []
      transformations:
      - input: nvidia.com/gpucores
        strategy: Retain
        operation:
          type: Mul
          target:
            targetResource: nvidia.com/gpu
      - input: nvidia.com/gpumem
        strategy: Retain
        operation:
          type: Mul
          target:
            targetResource: nvidia.com/gpu
```
