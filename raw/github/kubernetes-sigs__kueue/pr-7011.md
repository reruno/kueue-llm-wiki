#### What this PR does / why we need it:

When calling serve with just a port, it will only listen on localhost which does not work when using Kubernetes Ingress.

Before the fix the serve log:

```
 INFO  Accepting connections at http://localhost:8080
```

After the fix:

```
 INFO  Accepting connections at http://0.0.0.0:8080
```
