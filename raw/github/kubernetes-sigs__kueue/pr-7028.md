#### Special notes for your reviewer:
After updating the StatefulSet queue name, workloads are deleted but not consistently recreated, resulting in "workload not found" errors. This fix triggers pod reconciliation when workloads are deleted to ensure reliable workload recreation.

### Comment by [@IrvingMg](https://github.com/IrvingMg) — 2025-09-29T07:36:32Z

> Could you please provide the reasoning if it fixes the bug and if my understanding the the release not is correct.

As I understood from the logs, updating the StatefulSet causes the workload to be deleted, my assumption is that this is usually due to a rolling update. Then, it is expected that a new workload with the correct queue name would be created. However, this recreation wasn't always happening. So, in this way, we ensure the pod reconciler is triggered for workload recreation.
