## Description

#### What type of PR is this?
/kind feature

#### What this PR does / why we need it:
Since the `podset-group-name` annotation is used to co-locate the leader and workers, using slicing together with it seems to beat the purpose and suggests misconfiguration.

#### Which issue(s) this PR fixes:
This is the second of a series of PRs related to https://github.com/kubernetes-sigs/kueue/issues/6242.

#### Special notes for your reviewer:
The other validations requested in the issue won't fit inside the `ValidateTASPodSetRequest` method since they require checks between grouped `PodSet`s, so I'll send that in yet another separate PR.

I also added one unrelated test case to this change ("invalid slice topology request - slice size provided without slice topology") but it's part of the same boolean-check so I figured it would be good to have that covered as well.

#### Does this PR introduce a user-facing change?
```release-note
TAS: introduce validation against using PodSet grouping and PodSet slicing for the same PodSet, 
which is currently not supported. More precisely the `kueue.x-k8s.io/podset-group-name` annotation
cannot be set along with any of: `kueue.x-k8s.io/podset-slice-size`, `kueue.x-k8s.io/podset-slice-required-topology`.
```
