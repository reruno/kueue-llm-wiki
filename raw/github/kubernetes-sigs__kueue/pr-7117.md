  BTW the current code of `shouldTryNextFlavor` makes it hard - at least for me - to grasp how it behaves. \
  Hence, let me propose #7169 as an attempt to tidy this up.
  
  And when you look at [the new version](https://github.com/olekzabl/kueue/blob/79ff45ba794c69a48189a1922d3d77c21560eb1d/pkg/scheduler/flavorassigner/flavorassigner.go#L836-L853) (if I did it right), you'll see the "rules of the game" written quite clearly:
  
  * If it won't fit even with preemption -> try next.
  * Else:
     * if preemption mode is `Preempt` and preemption policy is `TryNext` -> try next.
     * if borrowing mode is `Borrow` and borrowing policy is `TryNext` -> try next.
  * Else -> stop search.
  
  So the interpretation "X not useful -> player X abstains" (in my comment above) is not something arbitrarily made up by me; it's a description of how Kueue (currently) behaves.
