<blockquote>
<h2>v1.19.0</h2>
<p>cert-manager is the easiest way to automatically manage certificates in Kubernetes and OpenShift clusters.</p>
<blockquote>
<p>⚠️ <strong>Known issues</strong>: We are working on a patch to fix the following issues:</p>
<ul>
<li><a href="https://redirect.github.com/cert-manager/cert-manager/issues/8158">Unexpected certificate renewal after upgrading to 1.19.0</a></li>
</ul>
</blockquote>
<p>This release focuses on expanding platform compatibility, improving deployment flexibility, enhancing observability, and addressing key reliability issues.</p>
<blockquote>
<p>📖  Read the full release notes at cert-manager.io: <a href="https://cert-manager.io/docs/releases/release-notes/release-notes-1.19">https://cert-manager.io/docs/releases/release-notes/release-notes-1.19</a></p>
</blockquote>
<p>Changes since <code>v1.18.0</code>:</p>
<h2>Feature</h2>
<ul>
<li>Add IPv6 rules to the default network policy (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7726">#7726</a>, <a href="https://github.com/jcpunk"><code>@​jcpunk</code></a>)</li>
<li>Add <code>global.nodeSelector</code> to helm chart to allow for a single <code>nodeSelector</code> to be set across all services. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7818">#7818</a>, <a href="https://github.com/StingRayZA"><code>@​StingRayZA</code></a>)</li>
<li>Add a feature gate to default to Ingress <code>pathType</code> <code>Exact</code> in ACME HTTP01 Ingress challenge solvers. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7795">#7795</a>, <a href="https://github.com/sspreitzer"><code>@​sspreitzer</code></a>)</li>
<li>Add generated <code>applyconfigurations</code> allowing clients to make type-safe server-side apply requests for cert-manager resources. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7866">#7866</a>, <a href="https://github.com/erikgb"><code>@​erikgb</code></a>)</li>
<li>Added API defaults to issuer references group (cert-manager.io) and kind (Issuer). (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7414">#7414</a>, <a href="https://github.com/erikgb"><code>@​erikgb</code></a>)</li>
<li>Added <code>certmanager_certificate_challenge_status</code> Prometheus metric. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7736">#7736</a>, <a href="https://github.com/hjoshi123"><code>@​hjoshi123</code></a>)</li>
<li>Added <code>protocol</code> field for <code>rfc2136</code> DNS01 provider (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7881">#7881</a>, <a href="https://github.com/hjoshi123"><code>@​hjoshi123</code></a>)</li>
<li>Added experimental field <code>hostUsers</code> flag to all pods. Not set by default. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7973">#7973</a>, <a href="https://github.com/hjoshi123"><code>@​hjoshi123</code></a>)</li>
<li>Support configurable resource requests and limits for ACME HTTP01 solver pods through ClusterIssuer and Issuer specifications, allowing granular resource management that overrides global <code>--acme-http01-solver-resource-*</code> settings. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7972">#7972</a>, <a href="https://github.com/lunarwhite"><code>@​lunarwhite</code></a>)</li>
<li>The <code>CAInjectorMerging</code> feature has been promoted to BETA and is now enabled by default (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8017">#8017</a>, <a href="https://github.com/ThatsMrTalbot"><code>@​ThatsMrTalbot</code></a>)</li>
<li>The controller, webhook and ca-injector now log their version and git commit on startup for easier debugging and support. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8072">#8072</a>, <a href="https://github.com/prasad89"><code>@​prasad89</code></a>)</li>
<li>Updated <code>certificate</code> metrics to the collector approach. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7856">#7856</a>, <a href="https://github.com/hjoshi123"><code>@​hjoshi123</code></a>)</li>
</ul>
<h2>Bug or Regression</h2>
<ul>
<li>ACME: Increased challenge authorization timeout to 2 minutes to fix <code>error waiting for authorization</code> (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7796">#7796</a>, <a href="https://github.com/hjoshi123"><code>@​hjoshi123</code></a>)</li>
<li>BUGFIX: permitted URI domains were incorrectly used to set the excluded URI domains in the CSR's name constraints (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7816">#7816</a>, <a href="https://github.com/kinolaev"><code>@​kinolaev</code></a>)</li>
<li>Enforced ACME HTTP-01 solver validation to properly reject configurations when multiple ingress options (<code>class</code>, <code>ingressClassName</code>, <code>name</code>) are specified simultaneously (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8021">#8021</a>, <a href="https://github.com/lunarwhite"><code>@​lunarwhite</code></a>)</li>
<li>Increase maximum sizes of PEM certificates and chains which can be parsed in cert-manager, to handle leaf certificates with large numbers of DNS names or other identities (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7961">#7961</a>, <a href="https://github.com/SgtCoDFish"><code>@​SgtCoDFish</code></a>)</li>
<li>Reverted adding the <code>global.rbac.disableHTTPChallengesRole</code> Helm option. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7836">#7836</a>, <a href="https://github.com/inteon"><code>@​inteon</code></a>)</li>
<li>This change removes the <code>path</code> label of core ACME client metrics and will require users to update their monitoring dashboards and alerting rules if using those metrics. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8109">#8109</a>, <a href="https://github.com/mladen-rusev-cyberark"><code>@​mladen-rusev-cyberark</code></a>)</li>
<li>Use the latest version of <code>ingress-nginx</code> in E2E tests to ensure compatibility (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7792">#7792</a>, <a href="https://github.com/wallrj"><code>@​wallrj</code></a>)</li>
</ul>
<h2>Other (Cleanup or Flake)</h2>
<ul>
<li>Helm: Fix naming template of <code>tokenrequest</code> RoleBinding resource to improve consistency (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7761">#7761</a>, <a href="https://github.com/lunarwhite"><code>@​lunarwhite</code></a>)</li>
<li>Improve error messages when certificates, CRLs or private keys fail admission due to malformed or missing PEM data (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7928">#7928</a>, <a href="https://github.com/SgtCoDFish"><code>@​SgtCoDFish</code></a>)</li>
<li>Major upgrade of Akamai SDK. NOTE: The new version has not been fully tested end-to-end due to the lack of cloud infrastructure. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8003">#8003</a>, <a href="https://github.com/hjoshi123"><code>@​hjoshi123</code></a>)</li>
<li>Update kind images to include the Kubernetes 1.33 node image (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7786">#7786</a>, <a href="https://github.com/wallrj"><code>@​wallrj</code></a>)</li>
<li>Use <code>maps.Copy</code> for cleaner map handling (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8092">#8092</a>, <a href="https://github.com/quantpoet"><code>@​quantpoet</code></a>)</li>
<li>Vault: Migrate Vault E2E add-on tests from deprecated <code>vault-client-go</code> to the new <code>vault/api</code> client. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8059">#8059</a>, <a href="https://github.com/armagankaratosun"><code>@​armagankaratosun</code></a>)</li>
</ul>
