#### Special notes for your reviewer:

All the relevant changes are to move the API dependent content of `pkg/util/testing/wrappers.go` and `pkg/util/testing/defaults.go` to 
`pkg/util/testing/v1beta1/wrappers.go` (dedicated package). 

All other changes are just using the new alias.

In the transition period we may introduce the v1beta2 API, but still use v1beta1 for testing, or for example migrate e2e testing independently of integration or performance testing. In that case we will need to duplicate the testing wrappers. 

This PRs allows to minimize the duplication in the transition period. 

It is also a useful refactoring on its own to separate helpers wrappers from other testing utils.

- **pkg/util/testing/v1beta1/wrappers.go:1** by [@mimowo](https://github.com/mimowo) — 2025-10-16T06:50:48Z
  First, you can think of the PR just as a cleanup, it does not introduce any new wrappers, just separates the ones offered by Kueue from those offered by Kubernetes. So, if we don't want to maintain v1beta1 and v1beta2 we will just rename the folder.
  
  Now, I think it might be beneficial in the transition period to have testing wrappers for v1beta1 and v1beta2 for the ease of reviews and transitioning the tests. Otherwise, we need to transition e2e, integration, unit, kubectl tests in one PR. I already experimented with that and it is massive, see PR: https://github.com/kubernetes-sigs/kueue/pull/7265. No one will be able to review that in one go. It also means conflicts with literally every other PR. 
  
  So with the temporarily added wrappers for v1beta2 we can separate migration of unit tests, kueuectl etc step by step.
  
  If you have a better idea how we can review the transition to beta2 I'm open to that.
