#### What this PR does / why we need it:
Workload Update validation should allow to set a ClusterName for workload replacement even though
it is not one of the nominatedClusterNames if the workload is part of the ElasticJob.

Remove the dispatcherName from the Workload Webhook code, that has unnecessary complexity as the validation was never required.
