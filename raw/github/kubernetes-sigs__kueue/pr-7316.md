  Sorry, I take that back. I would like to discuss this a bit more. 
  
  The `granularMode` has two "dimensions": preemptionMode and borrowingLevel. The preference should just choose which dimension is more important when comparing two granularModes. 
  
  I'm not sure if we need the categorization at all. The thing is that there are scenarios (in the hierarchical cohorts case) where quota is located only at cohorts (not at the CQs). So every CQ will be borrowing. And if we set `preference = Preemption` we would like to be able to find the most local quota (with respect to the `borrowingLevel`) regardless of the preemption we may have to make. 
  
  If we treat preemptionAndBorrowing as "the least desirable outcome in either mode" we might end up choosing a flavor with a more "distant" (in the hierarchy) quota.
