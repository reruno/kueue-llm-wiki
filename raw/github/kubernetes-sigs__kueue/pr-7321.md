<details>
<summary>Release notes</summary>
<p><em>Sourced from <a href="https://github.com/cert-manager/cert-manager/releases">github.com/cert-manager/cert-manager's releases</a>.</em></p>
<blockquote>
<h2>v1.19.1</h2>
<p>cert-manager is the easiest way to automatically manage certificates in Kubernetes and OpenShift clusters.</p>
<p>We reverted the CRD-based API defaults for <code>Certificate.Spec.IssuerRef</code> and <code>CertificateRequest.Spec.IssuerRef</code> after they were found to cause unexpected certificate renewals after upgrading to 1.19.0. We will try re-introducing these API defaults in cert-manager <code>1.20</code>.
We fixed a bug that caused certificates to be re-issued unexpectedly if the <code>issuerRef</code> kind or group was changed to one of the &quot;runtime&quot; default values.
We upgraded Go to <code>1.25.3</code> to address the following security vulnerabilities: <code>CVE-2025-61724</code>, <code>CVE-2025-58187</code>, <code>CVE-2025-47912</code>, <code>CVE-2025-58183</code>, <code>CVE-2025-61723</code>, <code>CVE-2025-58186</code>, <code>CVE-2025-58185</code>, <code>CVE-2025-58188</code>, and <code>CVE-2025-61725</code>.</p>
<blockquote>
<p>📖 Read the <a href="https://cert-manager.io/docs/releases/release-notes/release-notes-1.19">full 1.19 release notes</a> on the cert-manager.io website before upgrading.</p>
</blockquote>
<p>Changes since <code>v1.19.0</code>:</p>
<h3>Bug or Regression</h3>
<ul>
<li>BUGFIX: in case kind or group in the <code>issuerRef</code> of a Certificate was omitted, upgrading to <code>1.19.x</code> incorrectly caused the certificate to be renewed (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8175">#8175</a>, <a href="https://github.com/cert-manager-bot"><code>@​cert-manager-bot</code></a>)</li>
<li>Bump Go to 1.25.3 to fix a backwards incompatible change to the validation of DNS names in X.509 SAN fields which prevented the use of DNS names with a trailing dot (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8177">#8177</a>, <a href="https://github.com/wallrj-cyberark"><code>@​wallrj-cyberark</code></a>)</li>
<li>Revert API defaults for issuer reference kind and group introduced in 0.19.0 (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8178">#8178</a>, <a href="https://github.com/cert-manager-bot"><code>@​cert-manager-bot</code></a>)</li>
</ul>
</blockquote>
