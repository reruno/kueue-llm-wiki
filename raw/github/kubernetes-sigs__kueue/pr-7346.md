### Comment by [@kannon92](https://github.com/kannon92) — 2025-10-22T21:02:33Z

For some reason, the code that is panicking is:

```golang
func (c *Cache) AdmissionChecksForClusterQueue(cqName kueue.ClusterQueueReference) []AdmissionCheck {
	c.RLock()
	defer c.RUnlock()
	cq := c.hm.ClusterQueue(cqName)
	if cq == nil || len(cq.AdmissionChecks) == 0 {
		return nil
	}
	acs := make([]AdmissionCheck, 0, len(cq.AdmissionChecks))
	for acName := range cq.AdmissionChecks {
		if ac, ok := c.admissionChecks[acName]; ok {
			acs = append(acs, ac)
		}
	}
	return acs
}
```

At least the stacktrack is saying that the c.RLock() is causing a nil pointer exception.

The feature gate is causing this code to be called [here](https://github.com/kubernetes-sigs/kueue/blob/main/pkg/controller/jobframework/defaults.go#L99).

I am not sure of the fix though. I don't really get why the cache would be nil in this case.
