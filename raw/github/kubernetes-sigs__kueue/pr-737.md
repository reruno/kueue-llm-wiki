#### What this PR does / why we need it:
I added a usage field to the LocalQueue so that the controller can report the resource usage per the LocalQueue.

1. Added a new API FlavorUsage to LocalQueueStatus similar to FlavorUsage in ClusterQueueStatus.
2. Modified the ClusterQueue in the cache so that ClusterQueues can store resource usages per the LocalQueue. The usage per LocalQueues is mainly updated in the below cases:

    1.  ClusterQueues are created or updated.
    2. LocalQueues are created or deleted.
    3. Workloads are created or updated.

4. Updated an integration test for the local-queue-controller and a unit test for the cache.
