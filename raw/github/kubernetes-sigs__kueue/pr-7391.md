#### What this PR does / why we need it:

This PR removes long-deprecated APIs from Kueue's v1beta2 configuration, completing the deprecation lifecycle that began
in earlier versions. The following deprecated APIs are removed:

1. **QueueVisibility API** - Configuration field and type definitions
2. **retryDelayMinutes** - Field from AdmissionCheckSpec
3. **PodIntegrationOptions** - Configuration field (`podOptions`) and type definition
4. **Queue-name annotation** - Already removed in PR #7271

**Technical changes:**
- Updated core API types in `apis/config/v1beta2/` and `apis/kueue/v1beta2/`
- Regenerated deepcopy and conversion code via controller-gen
- Updated validation logic to use `managedJobsNamespaceSelector` instead of removed `podOptions`
- Fixed application code in `cmd/kueue/main.go` and `pkg/config/`
- Updated tests to remove deprecated API usage and fix validation expectations
- Regenerated CRD manifests

#### Which issue(s) this PR fixes:

Fixes #7247


#### Does this PR introduce a user-facing change?

```release-note
ACTION REQUIRED: Remove deprecated APIs from v1beta2 Configuration: QueueVisibility, PodIntegrationOptions (podOptions field), and retryDelayMinutes from AdmissionCheckSpec. Users must migrate to v1beta2 equivalents or continue using v1beta1 for these features. Pod integration now uses the top-level managedJobsNamespaceSelector field.
```
