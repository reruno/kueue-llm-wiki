#### What type of PR is this?

#### What this PR does / why we need it:

This PR removes the long-deprecated PodIntegrationOptions API from Kueue's v1beta2 Configuration. The PodIntegrationOptions API has been deprecated since v0.11, and users should migrate to using `managedJobsNamespaceSelector` for namespace-level pod filtering.

**Technical changes:**

- Remove `PodIntegrationOptions` type and `podOptions` field from v1beta2 Integrations
- Update conversion functions to ignore PodOptions field during v1beta1 to v1beta2 conversion
- Update main.go to remove PodOptions integration setup
- Update validation to use only managedJobsNamespaceSelector for pod integration
- Update pod webhook to remove PodOptions handling
- Regenerate deepcopy and conversion code

**Note:** Some tests still need updates and will be addressed in follow-up commits.

**Migration note:** The API remains available in v1beta1 for backward compatibility. Users should migrate to using the top-level `managedJobsNamespaceSelector` field for pod namespace filtering.

#### Which issue(s) this PR fixes:

Part of #7247

#### Does this PR introduce a user-facing change?

```release-note
v1beta2: Remove deprecated PodIntegrationOptions (podOptions field) from v1beta2 Configuration.

ACTION REQUIRED:  If you are using the podOptions in the configMap, you need to migrate to using  managedJobsNamespaceSelector (https://kueue.sigs.k8s.io/docs/tasks/run/plain_pods/) before
the upgrade.
```
