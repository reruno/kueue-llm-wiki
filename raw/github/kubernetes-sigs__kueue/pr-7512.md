+	} else if isSuspended && queueNameChanged && oldQueueName != "" {

  This is not correct — the workload should be here. A StatefulSet creates one Pod, and even if we have only one Pod, Kueue still creates a workload. If QuotaReserved = false, we should allow changing the queue name.
  
  Also, this won’t help fix the issue. The problem isn’t with changing the queue name — it’s a race condition between deleting the old workload and creating a new one.
  
  We can’t fix this in the webhook. We probably need to move to PrebuiltWorkload, like we did in LeaderWorkerSet, or add the StatefulSet as the owner of the workload.
  Most likely, https://github.com/kubernetes-sigs/kueue/pull/4799 will fix this issue.
