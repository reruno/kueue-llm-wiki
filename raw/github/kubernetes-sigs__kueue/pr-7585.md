#### What this PR does / why we need it:

This PR introduces the refactor of the delete path for the workload's lifetime, specifically the handling of the workload deletion by the workload controller.

All reconciliation logic (apart form logging) has been removed from the predicate "Delete" method inside the proper "Reconcile" method. This is to better adhere to the design principles behind the controllers and to avoid inconsistencies caused by errors being thrown inside of the predicates and silenced there, rather than being properly handled and returned through the reconciler.

The PR also introduces additional tests to account for the logic relocation.
