<h2>v1.5.0</h2>
<h1>Highlights</h1>
<h2>Ray Label Selector API</h2>
<p>Ray v2.49 introduced a <a href="https://docs.ray.io/en/latest/ray-core/scheduling/labels.html">label selector API</a>. Correspondingly, KubeRay v1.5 now features a top-level API for defining Ray labels and resources. This new top-level API is the preferred method going forward, replacing the previous practice of setting labels and custom resources within rayStartParams.</p>
<p>The new API will be consumed by the Ray autoscaler, improving autoscaling decisions based on task and actor label selectors. Furthermore, labels configured through this API are mirrored directly into the Pods. This mirroring allows users to more seamlessly combine Ray label selectors with standard Kubernetes label selectors when managing and interacting with their Ray clusters.</p>
<p>You can use the new API in the following way:</p>
<pre lang="yaml"><code>apiVersion: ray.io/v1
kind: RayCluster
spec:
  ...
  headGroupSpec:
    rayStartParams: {}
    resources:
      Custom1: &quot;1&quot;
    labels:
      ray.io/zone: us-west-2a
      ray.io/region: us-west-2
  workerGroupSpec:
  - replicas: 1
    rayStartParams: {}
    resources:
      Custom1: &quot;1&quot;
    labels:
      ray.io/zone: us-west-2a
      ray.io/region: us-west-2
</code></pre>
<h2>RayJob Sidecar submission mode</h2>
<p>The RayJob resource now supports a new value for <code>spec.submissionMode</code> called <code>SidecarMode</code>.
Sidecar mode directly addresses a key limitation in both <code>K8sJobMode</code> and <code>HttpMode</code>: the network connectivity requirement from an external Pod or the KubeRay operator for job submission. With Sidecar mode, job submission is orchestrated by injecting a sidecar container into the Head Pod. This solution eliminates the need for an external client to handle the submission process and reduces job submission failure due to network failures.</p>
<p>To use this feature, set spec.submissionMode to SidecarMode in your RayJob:</p>
<pre lang="yaml"><code>apiVersion: ray.io/v1
kind: RayJob
metadata:
  name: my-rayjob
spec:
  submissionMode: &quot;SidecarMode&quot;
  ...
</code></pre>
