#### What this PR does / why we need it:
The flakiness appears to be due to a timing issue. After the worker disconnects, the Kueue pods crash and restart multiple times. We need to harden the assertions so that the test continues only after verifying that all replicas are ready and stable, that is, there have been no new restarts for a certain period of time.
