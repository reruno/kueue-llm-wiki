### Review by [@mimowo](https://github.com/mimowo) (commented) — 2025-11-12T11:14:21Z

```diff
@@ -357,6 +357,8 @@ func (c *ClusterQueue) PendingActive() int {
 // workloads that were already tried and are waiting for cluster conditions
 // to change to potentially become admissible.

```

  Ok, I looked deeper and it seems we don't need `PendingActive` and `PendingInadmissible` at all since they are only called from manager, one after another, without guarantee of consistency. So it would be better to have `Pending()` which returns two numbers, because we anyway need both numbers always.
  
  Then instead two RLocks for PendingActive and PendingInadmissible we would have one lock to compute both and guarantee they are in-sync.
