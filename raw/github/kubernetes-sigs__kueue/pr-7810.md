#### What this PR does / why we need it:
Why remove the custom retry logic?
Kubernetes controllers automatically retry reconciliation indefinitely when an error is returned, so custom retry logic is unnecessary.
The custom logic retried on all errors except NotFound (e.g., Invalid, Forbidden, Connection refused), which is redundant since Kubernetes already handles these retries.

There is no good reason to keep this custom logic anyways.
