<blockquote>
<h1>v1.5.1</h1>
<h2>Highlights</h2>
<p>This release adds support for Ray token authentication using RayCluster, RayJob and RayService.</p>
<p>You can enable Ray token authentication using the following API:</p>
<pre><code>apiVersion: ray.io/v1
kind: RayCluster
metadata:
  name: ray-cluster-with-auth
spec:
  rayVersion: '2.52.0'
  authOptions:
    mode: token
</code></pre>
<p>You must specify <code>spec.rayVersion</code> to <code>2.52.0</code> or newer. See full example at <a href="https://github.com/ray-project/kuberay/blob/master/ray-operator/config/samples/ray-cluster.auth.yaml">ray-cluster.auth.yaml</a>.</p>
<h2>Bug fixes</h2>
<ul>
<li>Fix a bug in the NewClusterWithIncrementalUpgrade strategy where the Active (old) cluster's Serve configuration cache was incorrectly updated to the new ServeConfigV2 during an upgrade. <a href="https://redirect.github.com/ray-project/kuberay/issues/4212">#4212</a> <a href="https://github.com/ryanaoleary"><code>@​ryanaoleary</code></a></li>
<li>Surface pod-level container failures to RayCluster status <a href="https://redirect.github.com/ray-project/kuberay/issues/4196">#4196</a> <a href="https://github.com/spencer-p"><code>@​spencer-p</code></a></li>
<li>Fix a bug where RayJob status is not updated if failure happens in Initializing phase <a href="https://redirect.github.com/ray-project/kuberay/issues/4191">#4191</a> <a href="https://github.com/spencer-p"><code>@​spencer-p</code></a></li>
<li>Fix a bug where RayCluster status was not always propagated to RayJob status <a href="https://redirect.github.com/ray-project/kuberay/issues/4192">#4192</a> <a href="https://github.com/machichima"><code>@​machichima</code></a></li>
</ul>
</blockquote>
