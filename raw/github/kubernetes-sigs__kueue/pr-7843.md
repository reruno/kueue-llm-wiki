### Comment by [@sbgla-sas](https://github.com/sbgla-sas) — 2025-11-26T10:05:43Z

It looks like you're correct on the sequence of events, @mimowo:

https://github.com/kubernetes-sigs/kueue/blob/d38e2d939a1ca9fdb7b002c69ee8e3c2afb5a0c4/cmd/kueue/main.go#L135-L158

The call to `apply` is (via `config.Load`) where the delegated ClusterProfile cache is configured, but feature gates aren't set until later. Swapping the order won't work as-is, since the loaded configuration is used to validate/set feature gates if not set by CLI flags.

One option would be to push the FG logic into `apply` but that would require a signature change, as well as adding a responsibility that might not belong there. Another would be to handle controller-runtime configuration that depends on things that aren't derived solely from the Kueue config itself - feature gates that might be set by CLI flags in this case - separately and trigger it later in `main`.

A similar approach to the latter already exists with the controller-runtime metrics server options being set here:

https://github.com/kubernetes-sigs/kueue/blob/d38e2d939a1ca9fdb7b002c69ee8e3c2afb5a0c4/cmd/kueue/main.go#L191
