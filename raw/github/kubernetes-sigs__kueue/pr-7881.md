<blockquote>
<h2>kustomize/v5.8.0</h2>
<h1>Highlights</h1>
<h3>implements to replacements value in the structured data</h3>
<p>Now, We can edit yaml/json in yaml manifests with replacements transformer.
See <a href="https://redirect.github.com/kubernetes-sigs/kustomize/issues/5679">#5679</a></p>
<h4>For example</h4>
<pre lang="yaml"><code>## source
apiVersion: v1
kind: ConfigMap
metadata:
  name: source-configmap
data:
  HOSTNAME: www.example.com
---
apiVersion: v1
kind: ConfigMap
metadata:
  name: target-configmap
data:
  config.json: |-
    {&quot;config&quot;: {
      &quot;id&quot;: &quot;42&quot;,
      &quot;hostname&quot;: &quot;REPLACE_TARGET_HOSTNAME&quot;
    }}
</code></pre>
<pre lang="yaml"><code>## replacement
replacements:
- source:
    kind: ConfigMap
    name: source-configmap
    fieldPath: data.HOSTNAME
  targets:
  - select:
      kind: ConfigMap
      name: target-configmap
    fieldPaths:
    - data.config\.json.config.hostname
</code></pre>
<h3>fix: Propagate Namespace correctly to Helm</h3>
<p>The long-standing bug where kustomize's namespace transformer did not pass namespaces to helmCharts has been fixed.
See <a href="https://redirect.github.com/kubernetes-sigs/kustomize/issues/5940">#5940</a></p>
