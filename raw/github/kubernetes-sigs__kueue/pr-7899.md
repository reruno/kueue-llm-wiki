## Description

#### What type of PR is this?
/kind bug

#### What this PR does / why we need it:
This PR adds support for the `requiredDuringSchedulingIgnoredDuringExecution` affinity setting to TAS. This field is currently being ignored during scheduling.

#### Which issue(s) this PR fixes:
Fixes #7728

#### Special notes for your reviewer:
I marked this PR with `kind/bug` because the affinity being ignored seems like a bug/missing part of how scheduling is supposed to work rather than a standalone feature.

#### Does this PR introduce a user-facing change?
```release-note
TAS: Fix the `requiredDuringSchedulingIgnoredDuringExecution` node affinity setting being ignored in topology-aware scheduling.
```
