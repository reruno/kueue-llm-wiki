### Comment by [@mimowo](https://github.com/mimowo) — 2025-12-01T06:59:38Z

> The statefulset test was using static names for cluster-scoped resources, causing race conditions when tests ran in parallel. Switched to using GeneratedName to ensure isolation.

I don't think the reason for the failure is running the tests in parallel. I don't see any indication of that. While I agree we could consider generated names for CQs and RFs I wouldn't do it just for this test due to flake. We may consider doing that for allowing to run all test suites in parallel, but this is a much bigger change.

EDIT: I think after certs_test.go the Reconcile (per CRD) may get stuck here for 15s: https://github.com/kubernetes-sigs/kueue/blob/696c6770f86d864cb64ebb39d672d163ce906fa1/pkg/controller/core/leader_aware_reconciler.go#L73-L89 which seems what happened looking at the Kueue logs - we got the event about LQ creation, but no updates later on.
