## Pull request overview

This PR enables Pod-based integrations (`pod`, `deployment`, `statefulset`, and `leaderworkerset.x-k8s.io/leaderworkerset`) by default in Kueue, along with a default `managedJobsNamespaceSelector` configuration that excludes system namespaces. This change improves the out-of-the-box experience for users while maintaining safety by preventing Kueue from interfering with system pods.

Key changes:
- Enabled pod-based integrations by default in configuration files
- Added default `managedJobsNamespaceSelector` excluding `kube-system` and `kueue-system` namespaces
- Updated documentation to reflect new defaults while maintaining backward compatibility guidance for v0.15 and earlier
