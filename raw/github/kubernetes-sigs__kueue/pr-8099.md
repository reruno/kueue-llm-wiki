#### What this PR does / why we need it:

Following the issue #7066, this PR proposes an implementation to inject some labels in kueue metrics (in this PR, it only shows the clusterQueue one but can be expected to other components such as localQueue) from k8s label/annotations. 

It relies on : 
```
 metrics:
   bindAddress: :8080
   enableClusterQueueResources: true
   customMetricTags:
      clusterQueue:
        - resourceTag: team
        - resourceTag: user
        - resourceTag: gpu_type
          overrideMetricTag: my_gpu_type
``` 

There are different know issues of this PR :
* not properly managing the metrics doc 
* not handling the validation that the prometheus tag is a valid one for prometheus. (by default , the recent prometheus client accepts any UTF-8  but the legacy label name validator was more strict) 

#### Which issue(s) this PR fixes:
Fixes #7066
