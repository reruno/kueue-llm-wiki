## Description

This change adds a `WaitForLeaderElection` helper that waits for the leader lease to be renewed after the restart start time. This ensures tests don't proceed until the new controller has actually acquired leadership.

After `UpdateKueueConfiguration` restarts the controller, `RestartKueueController` only waits for deployment availability (pods running) but not for leader election to complete.

## Pull request overview

This PR adds a `WaitForLeaderElection` helper function that ensures tests don't proceed until the Kueue controller has actually acquired the leader lease after a restart. This addresses a flake where `RestartKueueController` would only wait for pod availability but not for the new controller instance to complete leader election.

Key changes:
- Added `WaitForLeaderElection` function to verify leader lease renewal after restart
- Captures restart timestamp and validates lease `RenewTime` is after that timestamp
- Integrated into `RestartKueueController` workflow after deployment availability checks
