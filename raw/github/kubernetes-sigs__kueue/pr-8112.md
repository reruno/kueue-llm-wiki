## Pull request overview

This PR addresses an issue with loading multi-architecture images into kind clusters running in Docker-in-Docker (DinD) environments. The change bypasses the `kind load docker-image --all-platforms` command, which has known issues in DinD, by using `docker save` piped to `ctr import` directly on each worker node.

**Key Changes:**
- Modified `cluster_kind_load_image` function to load images node-by-node using `docker save | docker exec ctr import` instead of the `kind load docker-image` command
- Removed the comma-separated node list creation (`paste -sd "," -`) since nodes are now processed individually
- Added explanatory comments linking to the upstream kind issue #3795
