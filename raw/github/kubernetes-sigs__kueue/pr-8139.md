## Pull request overview

This pull request implements an optimization to reduce API server load during high-contention scheduling scenarios by skipping redundant status updates when a workload remains inadmissible across multiple scheduling cycles and only the QuotaReserved condition message changes (e.g., "5 more needed" → "3 more needed").

**Key Changes:**
- Added early-return optimization in `UnsetQuotaReservationWithCondition` to skip updates when only the message differs
- Added comprehensive test coverage for the optimization logic in `TestUnsetQuotaReservationWithCondition`

+	// We intentionally ignore the message comparison because the message often contains
+	// dynamic information (e.g., "X more needed") which fluctuates between cycles.
+	// Skipping updates when only the message changes significantly reduces API server load
+	// during high-contention scheduling cycles. The user can still see the latest
+	// message in the Events.

  The early return optimization skips the `SyncAdmittedCondition` call on line 638, which could prevent the Admitted condition from being properly synchronized.
  
  When the early return is triggered, the workload has no quota reservation (QuotaReserved=False, Admission=nil). However, `SyncAdmittedCondition` is responsible for ensuring the Admitted condition is set to False when there's no reservation. By skipping this call, the Admitted condition might remain in an incorrect state (e.g., True from a previous admission) even though the workload is no longer admitted.
  
  **Recommendation**: Move the `SyncAdmittedCondition` call before the early return check, or ensure it's always called regardless of the optimization. For example:
  
  ```go
  existingCond := apimeta.FindStatusCondition(wl.Status.Conditions, kueue.WorkloadQuotaReserved)
  admittedChanged := SyncAdmittedCondition(wl, now)
  
  if existingCond != nil &&
      existingCond.Status == metav1.ConditionFalse &&
      existingCond.Reason == reason &&
      existingCond.ObservedGeneration == wl.Generation &&
      wl.Status.Admission == nil {
      // Only the message changed, but Admitted condition may have changed
      return admittedChanged
  }
  ```
