## Pull request overview

This PR addresses flaky tests that occur after controller restarts by adding a `WaitForWebhookReady` utility function. The function verifies webhook server readiness by attempting to create and delete a ResourceFlavor object, ensuring the webhook endpoints are fully propagated before tests proceed.

- Adds `WaitForWebhookReady` function that polls webhook availability using a test ResourceFlavor object
- Integrates the webhook readiness check into the `RestartKueueController` sequence after leader election
- Uses standard test timeout/interval constants and follows existing patterns for wait functions
