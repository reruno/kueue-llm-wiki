## Pull request overview

This PR is a manual cherry-pick of #8220 to the release-0.14 branch, adding functionality to wait for webhook service endpoints to be ready after controller restarts. This addresses flaky tests that occur when webhook requests are made before the endpoint controller has updated the EndpointSlice to include the new controller pod IPs.

- Adds `waitForWebhookEndpointsReady` function to ensure webhook endpoints are synchronized with ready controller pods before proceeding
- Introduces `isPodReady` helper to check pod readiness status
- Integrates the webhook readiness check into the `RestartKueueController` workflow
