### Comment by [@IrvingMg](https://github.com/IrvingMg) — 2025-12-16T15:35:17Z

> This looks great, but I'm wondering how confident we are this will work for preemption and borrowing?

Good question. I added integration tests covering preemption and borrowing scenarios with workloads that have zero resource requests, and in doing so found and fixed a potential issue: `TotalRequestsFor` could panic during preemption calculations when attempting to access a nil flavor, so I added a nil check to prevent this.

I also made an additional fix where toAPI() was including all resources in ResourceUsage, even those without assigned flavors. These are now correctly filtered out.
