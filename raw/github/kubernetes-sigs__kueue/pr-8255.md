  the only scenarios I can see:
  
  1. `Update` fail
    a. the API server can successfully commit the update (persisting the overrides) but fail to return a response to the client (e.g., due to a context deadline or network timeout).
    b. The controller receives an error and triggers a retry
    c. The Reconcile loop fetches the trainjob again. Since the server did process the update, the fetched object already contains the Kueue overrides.
    d. Without filtering, RunWithPodSetsInfo would append a second set of duplicate overrides.
  
  2. Double update conflict
    a. `Update` inside `RunWithPodSetsInfo` succeeds
    b. [clientutil.Patch](https://github.com/kubernetes-sigs/kueue/blob/d24291fa6146c1ae11b401b60c1c11a78291eb53/pkg/controller/jobframework/reconciler.go#L1168) fails
    c. reconciler retries
    d. trainjob is fetched - with the overrides
    e. the bug happens
  
  Maybe it's kind of highly unlikely but still could happen if I'm not missing something.
  wdyt?
