#### What this PR does / why we need it:

- without this check, the specific LWS reconciler ignores terminating pods, relying on the generic Pod controller (or timeouts) to eventually clean them up. This caused significant delays in pod deletion, particularly observed during parallel E2E tests, where pods would remain in the `Terminating` state for extended periods, causing test failures or timeouts.
