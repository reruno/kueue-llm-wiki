#### What this PR does / why we need it:

- Explicitly removes the `kueue.x-k8s.io/managed` finalizer when StatefulSet pod is in a `Terminating` state (`DeletionTimestamp` != nil).
Previously, the reconciler only checked for "Terminated" phases (Succeeded/Failed), ignoring deleting pods. This caused reliance on the generic Pod controller or timeouts to clean up finalizers, leading to significant delays in pod deletion (observed in parallel E2E tests) and potential deadlocks.


  Ah I see, so let's consider for simplicity StatefulSet with one Pod we may have the flow:
  1. The pods starts to terminate
  2. StatefulSet transitions status.ReadyReplicas 1 -> 0, triggering the reconcile
  3. The reconcile is triggered, but we still see the pod.phase=Running so we don't remove the finalizer
  4. The Pod transitions to terminal phase (Succeeded or Failed), but it is too late
  
  This PR eliminate most of the race condition, but still leaves a small window open in case the StatefulSet reconciler would see status.ReadyReplicas=0, but still see the Pod without the deletionTimestamp. This is technically possible if the k8s core StatefulSet controller sees the Pod update to set deletionTimestamp first, and sets status.ReadyReplicas=0. However, it would be very unlikely at this point.
  
  Let me merge this PR as it will already remove most of the race condition.
