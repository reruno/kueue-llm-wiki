#### What this PR does / why we need it:

Make the workload `Status.Admission.PodSetAssignment.Count` optional.

Since SSA will trigger a full validation of  of the updated resource/sub-resource regardless of the updated fields, we can end up not being able to finish the workloads that ware admitted when the CRD was changed.
