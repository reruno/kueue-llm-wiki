  Since this is not :
  
  Running a Job in Kueue is similar to [running a Job in a Kubernetes cluster](https://kubernetes.io/docs/tasks/job/) without Kueue.  
  
  However, you must set the  `kueue.x-k8s.io/queue-name` label selecting the LocalQueue
  you want to submit the Job to. You can also skip setting the "queue name" label if you use the LocalQueue defaulting (link here).
  
  Note that you do not need to create the Job in a [suspended state](https://kubernetes.io/docs/concepts/workloads/controllers/job/#suspending-a-job). Kueue automatically manages the Job's suspension via webhook and decides when it's the best time to start the Job.
