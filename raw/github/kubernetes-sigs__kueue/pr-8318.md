  I don't think the `nonroot` user with id `65532` exists in the `alpine` image by default (it does in the distroless images).
  I ran `docker run -it alpine:3.23.2 /bin/sh -c "cut -d: -f1,3 /etc/passwd"` to confirm and this user does not exist.
  
  This works because we don't depend on anything user specific, but this is still misleading in my opinion and could cause problems in other scenarios (even something like someone else using this Dockerfile as a template for some other Dockerfile in Kueue).
  
  Alpine does have the standard Linux `nobody` user though (id `65534`). Given the fact that this container does nothing, using `USER 65534:65534` could work here.
