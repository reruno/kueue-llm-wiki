## Description

#### What type of PR is this?

/kind bug

#### What this PR does / why we need it:

It symmetrizes the rules of what changes of a `WorkloadPriorityClass` attached to a `Workload` are allowed.
Specifically, it unlocks removal of WPC from suspended workloads, which seems to have been blocked by mistake (see #8321).

#### Which issue(s) this PR fixes:

Fixes #8321

#### Special notes for your reviewer:

I added a unit test for the method - previously missing - mostly AI-generated.

On the way, I realized some ugliness of the contract: we may return a nil slice (the standard of the modified file) or an empty slice (the standard of the underlying library).

For now, I just added `cmpopts.EquateEmpty()` to the unit test, so that it doesn't care.

#### Does this PR introduce a user-facing change?

```release-note
Fix a bug where suspended workloads didn't allow to remove a `WorkloadPriorityClass`.
```
