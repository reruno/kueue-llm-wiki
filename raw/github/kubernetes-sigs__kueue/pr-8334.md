## Description

#### What type of PR is this?

/kind cleanup

#### What this PR does / why we need it:

This adjusts the validation applied when a `WorkloadPriorityClass` reference is changed for a workload - in 2 ways:

* It changes the error messages coming from validation to be less misleading.
  The old error message said `field is immutable`, which is no longer true - it can be changed from non-empty value to another non-empty value, it can be also changed empty -> non-empty if the workload is suspended.
* BTW, it gets rid of the confusing construct `!isSuspended && oldEmpty || newEmpty` (where `newEmpty` stood outside the `&&` for a non-obvious reason) - making the intent clearer.

#### Which issue(s) this PR fixes:

Fixes the relicts of #8321  ;)
