### Comment by [@mykysha](https://github.com/mykysha) — 2025-12-19T14:56:30Z

The function cannot be removed as it is used mainly in the preemption and fairsharing tests. In the metrics e2e test it is used in the case that tests preemption metrics, so it is not strictly required but using it makes sense.

Using >= instead of strict equal makes for even more false positives, as all the preemption tests would succeed both when only one workload is admitted and the other lacks resources (expected way), and when both are admitted (something went wrong in the test).

Improving the message via including both the admitted list and the pool of the workloads that are of interest for the specific case should help with the future debugging.

Potential cleanup follow-ups for the package may include using more informative `klog.ObjectRef` and passing wlKeys instead of full workload objects, as the tests often operate using just the workload keys and get full workloads for these helper functions, only for the full workloads to be converted back to keys and disregarded afterwards.
