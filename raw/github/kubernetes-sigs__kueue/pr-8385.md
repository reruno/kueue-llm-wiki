<p><em>Sourced from <a href="https://github.com/kubernetes-sigs/kueue/releases">sigs.k8s.io/kueue's releases</a>.</em></p>
<blockquote>
<h2>v0.15.2</h2>
<p>Changes since <code>v0.15.1</code>:</p>
<h2>Changes by Kind</h2>
<h3>Feature</h3>
<ul>
<li>Ray: Support RayJob InTreeAutoscaling by using the ElasticJobsViaWorkloadSlices feature. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8284">#8284</a>, <a href="https://github.com/hiboyang"><code>@​hiboyang</code></a>)</li>
</ul>
<h3>Bug or Regression</h3>
<ul>
<li>
<p>Kubeflow TrainJob v2: fix the bug to prevent duplicate pod template overrides when starting the Job is retried. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8271">#8271</a>, <a href="https://github.com/j-skiba"><code>@​j-skiba</code></a>)</p>
</li>
<li>
<p>MultiKueue: Fixed status sync for CRD-based jobs (JobSet, Kubeflow, Ray, etc.) that was blocked while the local job was suspended. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8344">#8344</a>, <a href="https://github.com/IrvingMg"><code>@​IrvingMg</code></a>)</p>
</li>
<li>
<p>MultiKueue: fix the bug that for Pod integration the AdmissionCheck status would be kept Pending indefinitely,
even when the Pods are already running.</p>
<p>The analogous fix is also done for the batch/Job when the MultiKueueBatchJobWithManagedBy feature gate  is disabled. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8288">#8288</a>, <a href="https://github.com/IrvingMg"><code>@​IrvingMg</code></a>)</p>
</li>
<li>
<p>Scheduling: fix a bug that evictions submitted by scheduler (preemptions and eviction due to TAS NodeHotSwap failing)
could result in conflict in case of concurrent workload modification by another controller.
This could lead to indefinite failing requests sent by scheduler in some scenarios when eviction is initiated by
TAS NodeHotSwap. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8313">#8313</a>, <a href="https://github.com/mbobrovskyi"><code>@​mbobrovskyi</code></a>)</p>
</li>
<li>
<p>TAS NodeHotSwap: fixed the bug that allows workload to requeue by scheduler even if already deleted on TAS NodeHotSwap eviction. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8310">#8310</a>, <a href="https://github.com/mbobrovskyi"><code>@​mbobrovskyi</code></a>)</p>
</li>
<li>
<p>TAS: fix a performance bug that continues reconciles of TAS ResourceFlavor (and related ClusterQueues)
were triggered by updates to Nodes' heartbeat times. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8355">#8355</a>, <a href="https://github.com/PBundyra"><code>@​PBundyra</code></a>)</p>
</li>
<li>
<p>TAS: fixed performance issue due to unncessary (empty) request by TopologyUngater (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8333">#8333</a>, <a href="https://github.com/mbobrovskyi"><code>@​mbobrovskyi</code></a>)</p>
</li>
</ul>
<h3>Other (Cleanup or Flake)</h3>
<ul>
<li>Improve error messages for validation errors regarding WorkloadPriorityClass changes in workloads. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8352">#8352</a>, <a href="https://github.com/olekzabl"><code>@​olekzabl</code></a>)</li>
<li>MultiKueue: improve the MultiKueueCluster reconciler to skip attempting to reconcile and throw errors
when the corresponding Secret or ClusterProfile objects don't exist. The reconcile will be triggered on
creation of the objects. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8290">#8290</a>, <a href="https://github.com/mszadkow"><code>@​mszadkow</code></a>)</li>
</ul>
</blockquote>
