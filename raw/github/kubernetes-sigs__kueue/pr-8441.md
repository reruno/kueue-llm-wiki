## Description

The change tracks which cohort roots have already been processed within a single `QueueInadmissibleWorkloads` call and skip CQs whose root was already handled. This ensures each hierarchy is processed exactly once regardless of how many CQs from that hierarchy are in the input.

## Pull request overview

This PR optimizes the `QueueInadmissibleWorkloads` function by preventing redundant processing of ClusterQueues that share the same cohort hierarchy. When multiple ClusterQueues from the same cohort are included in a single call, the function now tracks which cohort roots have already been processed and skips subsequent ClusterQueues from that hierarchy.

### Key Changes
- Added `processedRoots` set to track cohort roots that have been processed
- Modified the loop to check if a ClusterQueue's root cohort has already been handled before processing
- Added a comprehensive test case to verify deduplication works correctly

### Reviewed changes

Copilot reviewed 2 out of 2 changed files in this pull request and generated no comments.
