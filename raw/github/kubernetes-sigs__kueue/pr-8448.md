#### What this PR does / why we need it:
Grouping functions accepting `*kueue.PodSetTopologyRequest` makes the file more streamlined, even though it is large and likely not traversed top-to-bottom. For example, the `slicesRequested` function was 500 lines under the closest function with that argument (and 1000 lines under the line where it's used). Grouping them closer together creates a clearer block of "what can be done with `*kueue.PodSetTopologyRequest`.

#### Which issue(s) this PR fixes:
Fixes #

#### Special notes for your reviewer:
First PR for #8036.

I decided to still have `resolveLevelIdx` and `levelKeyWithImpliedFallback` in the middle of the new block even though they do not have the `*kueue.PodSetTopologyRequest` argument, because it made sense to me for them to be close to `HasLevel`.
