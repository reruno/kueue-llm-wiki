#### What this PR does / why we need it:
This PR concludes the main changes to the queue cache. They are required to accommodate the changes coming to workload controller, allowing for delete (and in some cases: update) handling in workload controller to be decoupled form workload information tracking (i.e. the kubernetes workload object accessible via the client).
