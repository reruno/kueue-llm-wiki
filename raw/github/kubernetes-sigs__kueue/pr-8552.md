## Pull request overview

This PR is a manual cherry-pick of the fix for a flaky ClusterQueue deletion issue from the main branch (PR #8543). The bug occurred when pending workloads that had been assumed by the scheduler were deleted before being fully admitted, leaving stale entries in the cache that blocked ClusterQueue finalizer removal.

**Changes:**
- Simplified Delete event handler to unconditionally remove workloads from cache
- Removed conditional logic that only deleted workloads with quota reservation or unknown state
- Updated comments to reflect the new unconditional deletion approach
