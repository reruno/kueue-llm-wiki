#### What this PR does / why we need it:
The PR moves the logic previously set in the predicates to more appropriate places. 
Deletion now happens in the Reconcile method proper.
Notifying the watchers (other reconcilers, specifically queue reconcilers) has been moved to the Controller update handlers.
The two described changes aim to make Workload Controller and Reconciler compliant with general Kubernetes guidelines in terms of predicate methods only performing the check on whether the reconcile loop should be performed for a given event.
