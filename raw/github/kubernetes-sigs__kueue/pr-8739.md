<p><em>Sourced from <a href="https://github.com/kubernetes-sigs/kueue/releases">sigs.k8s.io/kueue's releases</a>.</em></p>
<blockquote>
<h2>v0.15.3</h2>
<p>Changes since <code>v0.15.2</code>:</p>
<h2>Changes by Kind</h2>
<h3>Feature</h3>
<ul>
<li>CLI: Support &quot;kwl&quot; and &quot;kueueworkload&quot; as a shortname for Kueue Workloads. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8469">#8469</a>, <a href="https://github.com/kannon92"><code>@​kannon92</code></a>)</li>
</ul>
<h3>Bug or Regression</h3>
<ul>
<li>
<p>Add lws editer and viewer roles to kustomize and helm (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8515">#8515</a>, <a href="https://github.com/kannon92"><code>@​kannon92</code></a>)</p>
</li>
<li>
<p>FailureRecovery: Fix Pod Termination Controller's MaxConcurrentReconciles (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8665">#8665</a>, <a href="https://github.com/gabesaba"><code>@​gabesaba</code></a>)</p>
</li>
<li>
<p>Fix ClusterQueue deletion getting stuck when pending workloads are deleted after being assumed by the scheduler. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8548">#8548</a>, <a href="https://github.com/sohankunkerkar"><code>@​sohankunkerkar</code></a>)</p>
</li>
<li>
<p>Fix a bug that WorkloadPriorityClass value changes do not trigger Workload priority updates. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8499">#8499</a>, <a href="https://github.com/ASverdlov"><code>@​ASverdlov</code></a>)</p>
</li>
<li>
<p>HC: Avoid redundant requeuing of inadmissible workloads when multiple ClusterQueues in the same cohort hierarchy are processed. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8510">#8510</a>, <a href="https://github.com/sohankunkerkar"><code>@​sohankunkerkar</code></a>)</p>
</li>
<li>
<p>Integrations based on Pods: skip using finalizers on the Pods created and managed by integrations.</p>
<p>In particular we skip setting finalizers for Pods managed by the built in Serving Workloads  Deployments,
StatefulSets, and LeaderWorkerSets.</p>
<p>This improves performance of suspending the workloads, and fixes occasional race conditions when a StatefulSet
could get stuck when deactivating and re-activating in a short interval. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8573">#8573</a>, <a href="https://github.com/mbobrovskyi"><code>@​mbobrovskyi</code></a>)</p>
</li>
<li>
<p>JobFramework: Fixed a bug that allowed a deactivated workload to be activated. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8438">#8438</a>, <a href="https://github.com/chengjoey"><code>@​chengjoey</code></a>)</p>
</li>
<li>
<p>LeaderWorkerSet: Fixed a bug that prevented deleting the workload when the LeaderWorkerSet was scaled down. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8673">#8673</a>, <a href="https://github.com/mbobrovskyi"><code>@​mbobrovskyi</code></a>)</p>
</li>
<li>
<p>MultiKueue now waits for WorkloadAdmitted (instead of QuotaReserved) before deleting workloads from non-selected worker clusters. To revert to the previous behavior, disable the <code>MultiKueueWaitForWorkloadAdmitted</code> feature gate. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8600">#8600</a>, <a href="https://github.com/IrvingMg"><code>@​IrvingMg</code></a>)</p>
</li>
<li>
<p>MultiKueue: Fix a bug that the priority change by mutating the <code>kueue.x-k8s.io/priority-class</code> label on the management cluster is not propagated to the worker clusters. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8574">#8574</a>, <a href="https://github.com/mbobrovskyi"><code>@​mbobrovskyi</code></a>)</p>
</li>
<li>
<p>MultiKueue: fix the eviction when initiated by the manager cluster (due to eg. Preemption or WairForPodsReady timeout). (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8402">#8402</a>, <a href="https://github.com/mbobrovskyi"><code>@​mbobrovskyi</code></a>)</p>
</li>
<li>
<p>ProvisioningRequest: Fixed a bug that prevented events from being updated when the AdmissionCheck state changed. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8404">#8404</a>, <a href="https://github.com/mbobrovskyi"><code>@​mbobrovskyi</code></a>)</p>
</li>
<li>
<p>Revert the changes in PR <a href="https://redirect.github.com/kubernetes-sigs/kueue/pull/8599">kubernetes-sigs/kueue#8599</a> for transitioning
the QuotaReserved, Admitted conditions to <code>False</code> for Finished workloads. This introduced a regression,
because users lost the useful information about the timestamp of the last transitioning of these
conditions to True, without an API replacement to serve the information. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8612">#8612</a>, <a href="https://github.com/mbobrovskyi"><code>@​mbobrovskyi</code></a>)</p>
</li>
<li>
<p>Scheduling: fix the bug that setting (none -&gt; some) a workload priority class label (kueue.x-k8s.io/priority-class) was ignored. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8584">#8584</a>, <a href="https://github.com/andrewseif"><code>@​andrewseif</code></a>)</p>
</li>
<li>
<p>TAS: Fix a bug that MPIJob with runLauncherAsWorker Pod indexes are not correctly evaluated during rank-based ordering assignments. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8663">#8663</a>, <a href="https://github.com/tenzen-y"><code>@​tenzen-y</code></a>)</p>
</li>
<li>
<p>TAS: Fixed an issue where workloads could remain in the second-pass scheduling queue (used for integration
or TAS with ProvisioningRequests, and for TAS Node Hot Swap) even if they no longer require to be in the queue. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8431">#8431</a>, <a href="https://github.com/skools-here"><code>@​skools-here</code></a>)</p>
</li>
<li>
<p>TAS: fix TAS resource flavor controller to extract only scheduling-relevant node updates to prevent unnecessary reconciliation. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8453">#8453</a>, <a href="https://github.com/Ladicle"><code>@​Ladicle</code></a>)</p>
</li>
<li>
<p>TAS: significantly improves scheduling performance by replacing Pod listing with an event-driven
cache for non-TAS Pods, thereby avoiding expensive DeepCopy operations during each scheduling cycle. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8484">#8484</a>, <a href="https://github.com/gabesaba"><code>@​gabesaba</code></a>)</p>
</li>
</ul>
