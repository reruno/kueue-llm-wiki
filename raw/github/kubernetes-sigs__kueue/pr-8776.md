## Description

The test expected WeightedShare=0 but the correct value is 1000. After preemption, the 2-CPU workload is admitted to cq1 which borrows all 2 CPU from cohort-a (lendable=2).

WeightedShare = borrowing * 1000 / lendable = 2 * 1000 / 2 = 1000

## Pull request overview

This PR fixes a flaky integration test related to fair sharing weighted share metric calculation. The test was failing because it expected a WeightedShare value of 0 but the correct value is 1000. After preemption, when a 2-CPU workload is admitted to cq1 which borrows all 2 CPU from cohort-a (with lendable=2), the weighted share calculation yields: WeightedShare = borrowing * 1000 / lendable = 2 * 1000 / 2 = 1000.

**Changes:**
- Updated the expected weighted share metric value from 0.0 to 1000 for cq1 in the "sticky workload becomes inadmissible. next workload admits" test case
