#### What this PR does / why we need it:
As we attempt to update to controller-runtime v0.23.0 we will have to rework webhooks.

The controller-runtime v0.23 introduces generic (templated) webhooks. These expect very specific types and initialization patterns, especially around the generic type T and how new objects are created via reflection - [here](https://github.com/kubernetes-sigs/controller-runtime/compare/v0.22.5...v0.23.0).

The idea is to stop using our custom webhook builder as it is patched version of the controller-runtime webhook builder. However fix has been already provided([issue](https://github.com/kubernetes-sigs/kueue/issues/3137.)): https://github.com/kubernetes-sigs/controller-runtime/commit/48ec3b71211f9fe1a313e34a9b44c39ca3adeec2

This will allow smoother transition to `controller-runtime v0.23.0`.
Prep PR for https://github.com/kubernetes-sigs/kueue/pull/8746
