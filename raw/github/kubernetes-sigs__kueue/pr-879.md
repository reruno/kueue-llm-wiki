  `foreach` is just a more generic version of  addprefix and addsufix,  it is used to "decorate" array (space delimited) elements and and join the decorated results.
  
  `$(MAKE)` is just  a macro containing the "make command" so every time used it will run make again.  
  
  The snippet proposed early on will essentially just  generate a string like
  ```make
  $(MAKE) run-test-e2e-v1; $(MAKE) run-test-e2e-v2; $(MAKE) run-test-e2e-v3; $(MAKE) run-test-e2e-v4;
  ```
  and execute it , so it will run make 4 times with other targets.
  
  One may suggest to only use `$(MAKE)` once:
  
  ```make
  test-e2e-all: kustomize manifests generate envtest ginkgo
  	$(MAKE)  $(foreach VERSION,$(E2E_K8S_VERSIONS), run-test-e2e-$(VERSION))
  ```
  which is equivalent to
  
  ```make
  test-e2e-all: kustomize manifests generate envtest ginkgo
  	$(MAKE)  $(addprefix run-test-e2e-,${E2E_K8S_VERSIONS})
  ```
  case in which we end up with something  like :
  
  ```make
  $(MAKE) run-test-e2e-v1 run-test-e2e-v2  run-test-e2e-v3 run-test-e2e-v4
  ```
  
  so run make once more with the generate set of targets.
  
  However since we are already in make the is no point of creating a child process , add just a the targets as dependencies. (what we have now).
  
  I  don't see any benefit in changing this.
