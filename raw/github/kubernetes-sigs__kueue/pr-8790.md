  This code is not being executed in the jobs controller tests because `WebhookPath: util.WebhookPath` is not set in the Framework. Enabling it also turns on webhook validations, which causes some tests to fail because they don’t account for these checks.
  
  For example, tests that directly set workload admission fail with:
  
  ```
  Workload is inadmissible because of missing LocalQueue
  localQueue: {"name":"test-queue","namespace":"core-7d9kt"}
  ```
  
  These tests set a queue name but never create the corresponding LocalQueue resource. This worked without webhooks, but with validation enabled, the workload controller rejects it.
  
  I suggest fixing the flaky test first and refactoring the remaining tests in a follow-up.
