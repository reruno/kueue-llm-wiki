## Description

#### What type of PR is this?
/kind cleanup

#### What this PR does / why we need it:
Adds a `--target` flag to `hack/releasing/prepare_pull.sh` that allows the release manager to create either the release-branch PR, the main-branch PR, or both.

This is needed because the main branch PR cannot be created until later in the release process (e.g., after security insights are generated). The script currently creates both PRs at once, which blocks the release workflow.

#### Which issue(s) this PR fixes:
Fixes #8796

#### Special notes for your reviewer:
Usage:

```
./hack/releasing/prepare_pull.sh [--target <all|release|main>]

  Create prepare-release PRs for the release branch and/or main

  Options:
    --target <all|release|main>  Which PR(s) to create (default: all)
      all     - Create both release branch and main PRs
      release - Create only the release branch PR
      main    - Create only the main branch PR

  Example:
    ./hack/releasing/prepare_pull.sh v0.13.2              # Create both PRs
    ./hack/releasing/prepare_pull.sh --target release v0.13.2  # Create only release branch PR
    ./hack/releasing/prepare_pull.sh --target main v0.13.2     # Create only main branch PR
```

**Testing:**
- Tested flag parsing with both `--target value` and `--target=value` syntax
- Tested validation rejects invalid target values
