## Pull request overview

This PR fixes a flaky test by correcting the expected WeightedShare metric value from 0.0 to 1000. The test was incorrectly expecting a WeightedShare of 0 when the cluster queue was actually borrowing resources from its cohort. This follows the same fix pattern as PR #8776.

**Changes:**
- Corrected the expected WeightedShare metric value for cluster queue cq1 from 0.0 to 1000 in a fairsharing test
