## Pull request overview

This PR fixes a bug where Topology-Aware Scheduling (TAS) was incorrectly using quota-filtered pod resource requests for node placement calculations instead of the actual resources specified in the PodSpec. This caused TAS to potentially over-schedule pods on nodes lacking sufficient resources that were excluded from quota via `excludeResourcePrefixes`.

**Changes:**
- Modified TAS to compute resource requests directly from PodSpec for placement decisions rather than using pre-filtered quota values
- Updated existing test expectations to reflect correct placement behavior with unfiltered resource requirements
- Added new test case validating that memory constraints are properly considered even when excluded from quota

### Reviewed changes

Copilot reviewed 2 out of 2 changed files in this pull request and generated no comments.

| File | Description |
| ---- | ----------- |
| pkg/scheduler/flavorassigner/tas_flavorassigner.go | Changed TAS to compute `singlePodRequests` directly from PodSpec using `resourcehelpers.PodRequests()` instead of using pre-filtered values from `wl.TotalRequests`, ensuring placement considers all actual pod resource needs |
| pkg/scheduler/scheduler_tas_test.go | Updated test expectations to reflect correct placement behavior (pods spread across nodes when actual resources exceed node capacity), added test case for memory constraints, and standardized CPU request format to use "m" suffix |
