## Description

This change replaces a klogr-style log statement with structured InfoS logging
in the fairSharingIterator path when a ClusterQueue has no cohort.

The previous log mixed free-form text with key-value fields, which results in
non–JSON-lines compliant output at higher verbosity levels. Converting this log
to structured logging ensures compatibility with JSON tooling, as discussed in
#8586.

Scope is intentionally minimal and limited to this code path.
