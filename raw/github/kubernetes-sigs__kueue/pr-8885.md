#### What this PR does / why we need it:
Fixes the generated OpenAPI for the VisibilityOnDemand APIs by assigning stable OpenAPI model names for visibility/v1beta1 and visibility/v1beta2 (via +k8s:openapi-model-package + generated zz_generated.model_name.go) and referencing types via OpenAPIModelName() in apis/visibility/openapi/zz_generated.openapi.go.
This prevents OpenAPI schema resolution failures like:
unknown model in reference: "…PendingWorkload"

VisibilityOnDemand: Fix Visibility API OpenAPI schema generation to prevent schema resolution errors when visibility v1beta1/v1beta2 APIServices are installed.

The visibility schema issues result in the following error when re-applying the manifest for Kueue 0.16.0:
