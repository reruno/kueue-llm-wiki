## Pull request overview

This PR adds a workaround for a yq installation issue where the Go proxy fails to handle the package due to an invalid '*' filename in the repository zip file. The workaround bypasses the Go proxy by fetching yq directly from version control.

**Changes:**
- Modified the `yq` target in Makefile-deps.mk to use GOPROXY=direct, bypassing the Go proxy
- Added comments explaining the workaround and referencing the upstream issue
