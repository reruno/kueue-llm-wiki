## Description

#### What type of PR is this?
/kind feature

#### What this PR does / why we need it:

This PR updates the KEP for Topology Aware Scheduling (TAS) to include a mechanism for evicting workloads running on nodes that become tainted.

Currently, when a node is tainted (e.g., for maintenance or to reserve it for high-priority workloads), pods may be deleted by the Kubernetes core TaintEviction controller, but Kueue’s TAS cache remains unaware of this change. This leads to "ghost" resource usage in the cache, preventing new workloads from being scheduled quickly.
This update introduces the `TASReplaceNodeOnNodeTaints` feature gate which:
- Treats nodes with an untolerated `NoExecute` taint as unhealthy, triggering replacement or eviction.
- Treats nodes with an untolerated `NoSchedule` taint as unhealthy if the pods are in a failed, terminating, or pending state.
- Ensures that TAS-managed workloads are evicted or moved to healthy nodes to maintain cluster agility and accurate resource accounting when taints are used.
