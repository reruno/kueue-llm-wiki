## Pull request overview

This PR is a manual cherry-pick of #8919 to the release-0.15 branch. It fixes a bug where finished or deactivated workloads prevented ClusterQueue deletion and finalizer removal. The fix introduces a new function `HasActiveQuotaReservation` that checks whether a workload has an active quota reservation (not finished and not deactivated), and updates the scheduler cache to exclude workloads that don't meet these criteria.

**Changes:**
- Added `HasActiveQuotaReservation` function to check if a workload has an active quota reservation that should be tracked
- Updated scheduler cache to remove finished/deactivated workloads, allowing ClusterQueue finalizers to be removed promptly
- Added comprehensive test coverage for the fix including unit and integration tests
