<details>
<summary>Release notes</summary>
<p><em>Sourced from <a href="https://github.com/cert-manager/cert-manager/releases">github.com/cert-manager/cert-manager's releases</a>.</em></p>
<blockquote>
<h2>v1.19.3</h2>
<p>cert-manager is the easiest way to automatically manage certificates in Kubernetes and OpenShift clusters.</p>
<p>This release is contains three bug fixes, including a fix for the MODERATE severity DoS issue in GHSA-gx3x-vq4p-mhhv. All users should upgrade to the latest release.</p>
<h2>Changes by Kind</h2>
<h3>Bug or Regression</h3>
<ul>
<li>Fixed an infinite re-issuance loop that could occur when an issuer returns a certificate with a public key that doesn't match the CSR. The issuing controller now validates the certificate before storing it and fails with backoff on mismatch. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8415">#8415</a>, <a href="https://github.com/cert-manager-bot"><code>@​cert-manager-bot</code></a>)</li>
<li>Fixed an issue where HTTP-01 challenges failed when the Host header containing an IPv6 address. This means that users can now issue IP address certificates for IPv6 address subjects. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8436">#8436</a>, <a href="https://github.com/cert-manager-bot"><code>@​cert-manager-bot</code></a>)</li>
<li>Security (MODERATE): Fix a potential panic in the cert-manager controller when a DNS response in an unexpected order was cached. If an attacker was able to modify DNS responses (or if they controlled the DNS server) it was possible to cause denial of service for the cert-manager controller. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8468">#8468</a>, <a href="https://github.com/SgtCoDFish"><code>@​SgtCoDFish</code></a>)</li>
</ul>
<h3>Other (Cleanup or Flake)</h3>
<ul>
<li>Bump go to 1.25.6 (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8459">#8459</a>, <a href="https://github.com/SgtCoDFish"><code>@​SgtCoDFish</code></a>)</li>
</ul>
<h2>v1.19.2</h2>
<p>cert-manager is the easiest way to automatically manage certificates in Kubernetes and OpenShift clusters.</p>
<p>We updated Go to fix some vulnerabilities in the standard library.</p>
<blockquote>
<p>📖 Read the <a href="https://cert-manager.io/docs/releases/release-notes/release-notes-1.19">full 1.19 release notes</a> on the cert-manager.io website before upgrading.</p>
</blockquote>
<h2>Changes since <code>v1.19.1</code></h2>
<h3>Bug or Regression</h3>
<ul>
<li>Address false positive vulnerabilities <code>CVE-2025-47914</code> and <code>CVE-2025-58181</code> which were reported by Trivy. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8283">#8283</a>, <a href="https://github.com/SgtCoDFish"><code>@​SgtCoDFish</code></a>)</li>
<li>Update Go to <code>v1.25.5</code> to fix <code>CVE-2025-61727</code> and <code>CVE-2025-61729</code> (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8294">#8294</a>, <a href="https://github.com/wallrj-cyberark"><code>@​wallrj-cyberark</code></a>)</li>
<li>Update <code>global.nodeSelector</code> to helm chart to perform a <code>merge</code> and allow for a single <code>nodeSelector</code> to be set across all services. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8233">#8233</a>, <a href="https://github.com/cert-manager-bot"><code>@​cert-manager-bot</code></a>)</li>
</ul>
<h3>Other (Cleanup or Flake)</h3>
<ul>
<li>Update cert-manager's ACME client, forked from <code>golang/x/crypto</code> (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8270">#8270</a>, <a href="https://github.com/SgtCoDFish"><code>@​SgtCoDFish</code></a>)</li>
<li>Updated Debian 12 distroless base images (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8326">#8326</a>, <a href="https://github.com/wallrj-cyberark"><code>@​wallrj-cyberark</code></a>)</li>
</ul>
</blockquote>
