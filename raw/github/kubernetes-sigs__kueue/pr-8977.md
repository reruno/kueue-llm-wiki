#### What this PR does / why we need it:
Fixes the Prometheus ServiceMonitor configuration so Kueue metrics can be scraped:

1. Updates the ServiceMonitor selector to match the metrics service label (`metrics-service` instead of `controller`)
2. Adds EndpointSlice RBAC permissions required by Prometheus for target discovery

#### Special notes for your reviewer:

- After #7371, the ServiceMonitor selector needs to be updated to match the metrics-service label.
- The EndpointSlice RBAC issue was identified during manual testing.


### Comment by [@IrvingMg](https://github.com/IrvingMg) — 2026-02-04T11:14:08Z

I verified that Helm also needs the fix for RBAC permissions. I installed with:

```bash
helm install kueue oci://registry.k8s.io/kueue/charts/kueue \
  --version=0.16.0 \
  --namespace kueue-system \
  --create-namespace \
  --set enablePrometheus=true \
  --set metrics.prometheusNamespace=monitoring \
  --wait --timeout 300s
```

Then I ran:

```bash
curl -s 'http://localhost:9090/api/v1/targets' | jq '.data.activeTargets[] | select(.labels.job | contains("kueue"))'
```
and got an empty result. After patching the role with the required permissions:

```bash
kubectl patch role kueue-prometheus-k8s -n kueue-system --type='json' -p='[
  {
    "op": "add",
    "path": "/rules/-",
    "value": {
      "apiGroups": ["discovery.k8s.io"],
      "resources": ["endpointslices"],
      "verbs": ["get", "list", "watch"]
    }
  }
]'
```

the Kueue metrics appeared as expected.

The same applies for manifests. I installed Kueue with:

```bash
kubectl apply --server-side -f https://github.com/kubernetes-sigs/kueue/releases/download/v0.16.0/manifests.yaml
```

then applied:

```bash
kubectl apply --server-side -f https://github.com/kubernetes-sigs/kueue/releases/download/v0.16.0/prometheus.yaml
```

When I ran the curl command to the targets endpoint, I also got an empty result. In this case, not only are the RBAC permissions required, but the ServiceMonitor selector also needs to be patched to match the metrics service:

```bash
kubectl patch servicemonitor kueue-controller-manager-metrics-monitor \
  -n kueue-system \
  --type='json' \
  -p='[{"op": "replace", "path": "/spec/selector/matchLabels/app.kubernetes.io~1component", "value": "metrics-service"}]'
```

After this patch, the Kueue metrics appeared as expected.

Both fixes are included already in this PR.
