## Description

#### What type of PR is this?

  /kind feature                                                                                                                                                                                                  
   
  #### What this PR does / why we need it:                                                                                                                                                                       
                                                                           
  Add `kueueViz.backend.ingress.enabled` and `kueueViz.frontend.ingress.enabled` Helm values to allow users to disable ingress creation when using alternative ingress solutions (e.g., Istio
  Gateway/VirtualService).

  Both options default to `true` for backwards compatibility.

  #### Which issue(s) this PR fixes:

  Fixes #7659

  #### Special notes for your reviewer:

  The change is implemented via the yaml-processor in `hack/processing-plan.yaml`. Helm unit tests have been added to verify the behavior.

  #### Does this PR introduce a user-facing change?

  ```release-note
Helm: Add kueueViz.backend.ingress.enabled and kueueViz.frontend.ingress.enabled Helm values to allow disabling KueueViz ingress resources.
```
