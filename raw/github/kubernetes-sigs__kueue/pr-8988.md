### Comment by [@vladikkuzn](https://github.com/vladikkuzn) — 2026-02-08T14:57:34Z

It doesn't flake(at least locally with 100 runs) with `.WAIT`, but requires GNU make 4.4, which is not currently available in CI. Another option is to define the body, which can work with any make
Here are the two approaches side-by-side for `ci-lint` / `verify-ci-lint`:

---

**1. `.WAIT` (GNU Make >= 4.4)**

```makefile
# Standalone — lightweight, for local use
.PHONY: ci-lint
ci-lint: golangci-lint
	find . \( -path ./site -o -path ./bin -o -path ./vendor \) -prune -false -o -name go.mod -exec dirname {} \; | xargs -I {} sh -c 'cd "{}" && $(GOLANGCI_LINT) run $(GOLANGCI_LINT_FIX) --timeout 15m0s --config "$(PROJECT_DIR)/.golangci.yaml"'

# Wrapper — reuses ci-lint directly, .WAIT enforces ordering
.PHONY: verify-ci-lint
verify-ci-lint: verify-tree-prereqs gomod-verify .WAIT ci-lint
```

---

**2. `define` (any GNU Make)**

```makefile
define _ci_lint_recipe
find . \( -path ./site -o -path ./bin -o -path ./vendor \) -prune -false -o -name go.mod -exec dirname {} \; | xargs -I {} sh -c 'cd "{}" && $(GOLANGCI_LINT) run $(GOLANGCI_LINT_FIX) --timeout 15m0s --config "$(PROJECT_DIR)/.golangci.yaml"'
endef

# Standalone — lightweight, for local use
.PHONY: ci-lint
ci-lint: golangci-lint
	$(_ci_lint_recipe)

# Wrapper — same recipe, heavier prerequisites
.PHONY: verify-ci-lint
verify-ci-lint: verify-tree-prereqs gomod-verify golangci-lint
	$(_ci_lint_recipe)
```
