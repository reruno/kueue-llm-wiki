## Pull request overview

This PR is a cherry-pick of #8865 to the release-0.15 branch, fixing a bug where the Topology Aware Scheduling (TAS) feature incorrectly used quota-filtered resource values for node placement instead of the actual PodSpec resources. This caused TAS to ignore resources excluded by `excludeResourcePrefixes`, leading to incorrect pod placement decisions.

**Changes:**
- Modified TAS to use PodSpec resources directly for physical placement calculations instead of quota-filtered admission values
- Added a new helper function `NewRequestsFromPodTemplate` to compute resources directly from PodTemplateSpec
- Added comprehensive unit and integration tests to verify the fix
