### Comment by [@IrvingMg](https://github.com/IrvingMg) — 2026-02-06T15:22:14Z

> will they run as part of regular e2e or dedicated CI job? I don't have a strong view here.

Regular e2e won’t run them because they’re not under the `singlecluster` directory and they require `test-e2e-prometheus`.

I think a dedicated CI job would be better, since these tests only require the Prometheus Operator, while the other tests do not. Likewise, the Prometheus tests don’t need all the components that the rest of the tests require. To avoid installing unused components, it’s probably better to keep them separate.
