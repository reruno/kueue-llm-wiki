#### What this PR does / why we need it:
Fixes the bug reported in #9043.

Keeps ElasticJob worker Jobs in sync with their local counterparts during scale‑up, so remote parallelism and workload labels stay correct while avoiding stale updates when using with or without `features.MultiKueueBatchJobWithManagedBy`
