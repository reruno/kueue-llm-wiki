#### Special notes for your reviewer:

Refactored the certmanager kustomize configuration to replace the deprecated vars feature with replacements.

I had to move the tlsConfig settings from a patch into the base monitor.yaml resource to allow replacements to target it correctly. I also created the stub for the CRD annotation injection to handle the replacement before the patch.

Verified locally using kustomize build to ensure no variables were left unreplaced.

**Verification Steps**:

**1. Validating all variables are replaced**
```bash
./bin/kustomize build ./test/e2e/config/certmanager | grep -E "METRICS_SERVICE_NAME|SERVICE_NAME|CERTIFICATE_NAMESPACE|VISIBILITY_SERVICE_NAME"
```

**Output:**
```bash
— no output returned
```

**2. Validating proper replacements**
```bash
./bin/kustomize build ./test/e2e/config/certmanager | grep "cert-manager.io/inject-ca-from"
```

**Output:**
