## Pull request overview

Adjusts the fair-sharing integration tests to avoid Prometheus metric series collisions across specs by ensuring the relevant ClusterQueue (and corresponding LocalQueue) names are unique per test namespace. This targets the reported flake where cumulative metrics from previous specs could cause assertions to intermittently fail.

**Changes:**
- Make the two “best-effort” ClusterQueue names unique by suffixing them with `ns.Name`.
- Update workload creation in the affected specs to enqueue into the renamed LocalQueues derived from `cqA.Name` / `cqB.Name`.
