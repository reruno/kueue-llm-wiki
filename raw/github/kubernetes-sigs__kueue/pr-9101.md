## Pull request overview

Fixes non-deterministic ordering returned by `ClusterQueue.Snapshot()` by ensuring stable tie-breaking and adding regression tests to validate deterministic snapshots across repeated calls.

**Changes:**
- Update `ClusterQueue.Snapshot()` sorting to treat `lessFunc` ties as equal and break ties deterministically by `Workload` UID.
- Add sorting for inadmissible workloads collected from a map (and update `totalElements` helper accordingly).
- Add unit tests that repeatedly call `Snapshot()` to ensure consistent ordering for tied workloads, including inadmissible workloads.
