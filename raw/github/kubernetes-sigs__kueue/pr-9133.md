## Pull request overview

Updates the test Makefile so the default `test-e2e` target no longer invokes the MultiKueue DRA e2e test target, aligning `make test-e2e` with a narrower singlecluster-only e2e run.

**Changes:**
- Remove `MULTIKUEUE-DRA-E2E_TARGETS` (and its K8s-version gating) from the `test-e2e` prerequisites.
- Simplify `test-e2e` prerequisites to call a single `run-test-e2e-singlecluster-…` target directly.

  `test-e2e` now depends on `run-test-e2e-singlecluster-$(E2E_KIND_VERSION:kindest/node:v%=%)` directly. If `E2E_KIND_VERSION` is ever set to multiple images/versions (space-separated), Make will only prefix the first word and treat the remaining words as separate prerequisites (e.g., `run-test-e2e-singlecluster-1.34.3` and `1.35.0`), which will break the build. The previous `$(addprefix ...)` approach handled lists correctly; consider restoring a list-safe expansion (e.g., `$(addprefix run-test-e2e-singlecluster-,$(...))`) or enforcing/documenting that `E2E_KIND_VERSION` must be a single value.
