## Description

#### What type of PR is this?

/kind cleanup

#### What this PR does / why we need it:

Adds two scheduler unit tests for hierarchical cohort lendingLimit behaviour:

1. **Rejects workloads exceeding lendingLimit** - A workload requesting more resources than the cohort can lend is marked inadmissible.

2. **Admits workloads within lendingLimit** - A workload requesting exactly the lendable amount is admitted.

Both tests use a parent-child cohort structure where one ClusterQueue sets a lendingLimit and another attempts to borrow.

#### Which issue(s) this PR fixes:

Fixes https://github.com/kubernetes-sigs/kueue/issues/6574

#### Special notes for your reviewer:

Tests follow existing patterns in `scheduler_test.go`.
