## Description

#### What type of PR is this?
/kind bug

#### What this PR does / why we need it:
Fix a potential lost-wakeup race in the waiting queue manager by ensuring CleanUpOnContext acquires the queue mutex before broadcasting on the condition variable. Previously, the broadcast could occur without holding the same lock used by cond.Wait(), allowing a waiter to miss the signal and block indefinitely during shutdown/cancellation. This change aligns the implementation with sync.Cond’s expected locking pattern and adds a regression test to catch the race and prevent future hangs.

#### Which issue(s) this PR fixes:
Fixes #8434
