<details>
<summary>Release notes</summary>
<p><em>Sourced from <a href="https://github.com/kubernetes-sigs/kueue/releases">sigs.k8s.io/kueue's releases</a>.</em></p>
<blockquote>
<h2>v0.16.1</h2>
<p>Changes since <code>v0.16.0</code>:</p>
<h2>Changes by Kind</h2>
<h3>Feature</h3>
<ul>
<li>KueueViz backend and frontend resource requests/limits are now configurable via Helm values (kueueViz.backend.resources and kueueViz.frontend.resources). (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8981">#8981</a>, <a href="https://github.com/david-gang"><code>@​david-gang</code></a>)</li>
</ul>
<h3>Bug or Regression</h3>
<ul>
<li>
<p>Fix Visibility API OpenAPI schema generation to prevent schema resolution errors when visibility v1beta1/v1beta2 APIServices are installed.</p>
<p>The visibility schema issues result in the following error when re-applying the manifest for Kueue 0.16.0:
<code>failed to load open api schema while syncing cluster cache: error getting openapi resources: SchemaError(sigs.k8s.io/kueue/apis/visibility/v1beta1.PendingWorkloadsSummary.items): unknown model in reference: &quot;sigs.k8s.io~1kueue~1apis~1visibility~1v1beta1.PendingWorkload&quot;</code> (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8901">#8901</a>, <a href="https://github.com/vladikkuzn"><code>@​vladikkuzn</code></a>)</p>
</li>
<li>
<p>Fix a bug where finished or deactivated workloads blocked ClusterQueue deletion and finalizer removal. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8936">#8936</a>, <a href="https://github.com/sohankunkerkar"><code>@​sohankunkerkar</code></a>)</p>
</li>
<li>
<p>LeaderWorkerSet: Fix the bug where rolling updates with maxSurge could get stuck. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8886">#8886</a>, <a href="https://github.com/PannagaRao"><code>@​PannagaRao</code></a>)</p>
</li>
<li>
<p>LeaderWorkerSet: Fixed bug that doesn't allow to delete Pod after LeaderWorkerSet delete (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8882">#8882</a>, <a href="https://github.com/mbobrovskyi"><code>@​mbobrovskyi</code></a>)</p>
</li>
<li>
<p>Metrics certificate is now reloaded when certificate data is updated. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9099">#9099</a>, <a href="https://github.com/MaysaMacedo"><code>@​MaysaMacedo</code></a>)</p>
</li>
<li>
<p>MultiKueue &amp; ElasticJobs: fix the bug that the new size of a Job was not reflected on the worker cluster. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9055">#9055</a>, <a href="https://github.com/ichekrygin"><code>@​ichekrygin</code></a>)</p>
</li>
<li>
<p>Observability: Fix Prometheus ServiceMonitor selector and RBAC to enable metrics scraping. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8980">#8980</a>, <a href="https://github.com/IrvingMg"><code>@​IrvingMg</code></a>)</p>
</li>
<li>
<p>Observability: Fixed a bug where workloads that finished before a Kueue restart were not tracked in the gauge metrics for finished workloads. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8827">#8827</a>, <a href="https://github.com/mbobrovskyi"><code>@​mbobrovskyi</code></a>)</p>
</li>
<li>
<p>Observability: fix the bug that the &quot;replica-role&quot; (leader / follower) log decorator was missing in the log lines output by
the  webhooks for LeaderWorkerSet and StatefulSet . (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8820">#8820</a>, <a href="https://github.com/mszadkow"><code>@​mszadkow</code></a>)</p>
</li>
<li>
<p>PodIntegration: Fix the bug that Kueue would occasionally remove the custom finalizers when
removing the <code>kueue.x-k8s.io/managed</code> finalizer. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8903">#8903</a>, <a href="https://github.com/mykysha"><code>@​mykysha</code></a>)</p>
</li>
<li>
<p>RayJob integration: Make RayJob top level workload managed by Kueue when autoscaling via
ElasticJobsViaWorkloadSlices is enabled.</p>
<p>If you are an alpha user of the ElasticJobsViaWorkloadSlices feature for RayJobs, then upgrading Kueue may impact running live jobs which have autoscaling / workload slicing enabled. For example, if you upgrade Kueue, before
scaling-up completes,  the new pods will be stuck in SchedulingGated state. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9039">#9039</a>, <a href="https://github.com/hiboyang"><code>@​hiboyang</code></a>)</p>
</li>
<li>
<p>TAS: Fix a bug that TAS ignored resources excluded by excludeResourcePrefixes for node placement. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8990">#8990</a>, <a href="https://github.com/sohankunkerkar"><code>@​sohankunkerkar</code></a>)</p>
</li>
<li>
<p>TAS: Fixed a bug that pending workloads could be stuck, not being considered by the Kueue's scheduler,
after the restart of Kueue. The workloads would be considered for scheduling again after any update to their
ClusterQueue. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9056">#9056</a>, <a href="https://github.com/sohankunkerkar"><code>@​sohankunkerkar</code></a>)</p>
</li>
</ul>
<h3>Other (Cleanup or Flake)</h3>
<ul>
<li>KueueViz: It switches to the v1beta2 API (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/8804">#8804</a>, <a href="https://github.com/mbobrovskyi"><code>@​mbobrovskyi</code></a>)</li>
</ul>
</blockquote>
