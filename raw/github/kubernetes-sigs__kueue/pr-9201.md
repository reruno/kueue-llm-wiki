## Description

When MultiKueueGC deletes the remote workload before the reconciler processes the deletion, RemoveRemoteObjects would early-return on `remWl == nil` without deleting the remote controller object (e.g., LWS).

Move `DeleteRemoteObject` before the nil check to ensure the remote controller object is always deleted regardless of remote workload state.

## Pull request overview

This PR fixes a race condition in MultiKueue workload deletion where remote controller objects (e.g., LeaderWorkerSet) could be orphaned when the garbage collector deletes the remote workload before the reconciler processes the deletion.

**Changes:**
- Move `DeleteRemoteObject` call before nil check in `RemoveRemoteObjects` to ensure controller object is always deleted
