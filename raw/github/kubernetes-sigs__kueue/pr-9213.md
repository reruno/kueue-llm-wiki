### Comment by [@mszadkow](https://github.com/mszadkow) — 2026-02-13T10:22:27Z

Seems that the problem is with `-mod=readonly`.
We ask for the directory of the package, but as we don't have it ,it can't be assumed.
And because we use `-mod=readonly` go will not download the package automatically as it would without the flag.
https://go.dev/ref/mod#build-commands
Now, we need to have the module present to run the `kube-codegen.sh` so either we download it before or we skip this flag.
