#### What this PR does / why we need it:
I upgraded the MPI Operator version to [v0.8.0](https://github.com/kubeflow/mpi-operator/releases/tag/v0.8.0), which fixes the https://github.com/kubernetes-sigs/kueue/issues/3400 issue.

Additionally, I enabled MPIJob TAS E2E tests for rank-ordering verifications.

Fixes #

#### Special notes for your reviewer:
The root cause is that the previous MPI Operator doesn't support [Mutable Scheduling Directives for suspended Jobs](https://kubernetes.io/docs/concepts/workloads/controllers/job/#mutable-scheduling-directives-for-suspended-jobs) on Launcher Job. So, when Kueue admits MPIJob, the assigned scheduling directives and Kueue annotations are not propagated to the underlying batch/v1 Job and Pods for Launcher.
