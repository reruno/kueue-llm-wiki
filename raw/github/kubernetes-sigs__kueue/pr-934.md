#### What this PR does / why we need it:

Ensure that the job counts are either equal to the workload spec or the counts in the workload admission, depending on the stage.

As a recap, the workload and job objects go through the following stages in an admission is followed by eviction:

1. Workload is not admitted, job is suspended
2. Workload is admitted (might get smaller counts due to partial admission), job is suspended
3. Job gets new counts and is unsuspended
4. Job is running, but then eviction/preemption happens: Workload gets and eviction condition, but it's still admitted.
5. Job is suspended, counts are reset (for `batch/Job`, this happens across 3 API calls, but I'll followup on this in another PR).
6. Workload is not admitted, job is suspended (same as step 1).

This all translates that if the job is running, the counts should match the counts in the workload admission.
Otherwise, counts should match the workload spec counts.

### Comment by [@trasc](https://github.com/trasc) — 2023-07-06T07:39:21Z

To summarize:

We have the following counts to take into account:

- **jobPsCount** - the count extracted from the job `job.Podsets()[*].Count`
- **wlSpecPsCount** - the count extracted from the workload spec `wl.Spec.Podsets.Count`
- **wlSpecPsMinCount** - the count extracted from the workload `*wl.Spec.Podsets.MinCount`, only present when the pod supports partial partial-admission
- **wlAdmissionPsCount** -the count extracted from the workload's status `wl.Status.Admission.PodSetAssignments[*].Count`  

If the job is suspended the counts in the job **jobPsCount** should match the ones in the workload **wlSpecPsCount**.

If the job is unsuspended then depending on the support for dynamic-reclaim and partial-admission we have:

|dyn-reclaim|partaial-ad.|       | |
| - | - | - | - |
|false| false | **jobPsCount** ==  **wlSpecPsCount** | **wlAdmissionPsCount** == **wlSpecPsCount** |
|false| true | **jobPsCount** == **wlAdmissionPsCount** (**wlSpecPsMinCount** >= **jobPsCount** <= **wlSpecPsCount**) |**wlAdmissionPsCount** <= **wlSpecPsCount** |
|true| false| **jobPsCount** ==  **wlSpecPsCount** | **wlAdmissionPsCount** == **wlSpecPsCount** |
|true| true | **jobPsCount** == **wlAdmissionPsCount** (**wlSpecPsMinCount** >= **jobPsCount** <= **wlSpecPsCount**) | **wlAdmissionPsCount** <= **wlSpecPsCount** |

So this should work, however I don't understand the need to change 

```diff
- if !workload.CanBePartiallyAdmitted(wl) || job.IsSuspended() {
+ if !workload.CanBePartiallyAdmitted(wl) || !workload.IsAdmitted(wl) {
```

and the addition of 

```go
	// If the workload is admitted but the job is suspended, ignore counts.
	// This might allow some violating jobs to pass equivalency checks, but their
	// workloads would be invalidated in the next sync after unsuspending.
	if job.IsSuspended() {
		return true
	}
```
