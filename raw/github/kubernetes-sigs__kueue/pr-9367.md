### Review by [@mimowo](https://github.com/mimowo) (commented) — 2026-02-19T13:04:35Z

cc @kshalot @mwielgus @j-skiba ptal

We need to use v1beta1 as the default, so that the script can effectively LIST all workloads just after upgrade of Kueue 0.15 -> 0.16. Without the change the API server may take too long (exceeding etcd timeout of 2.5min) to convert all Workload into v1beta2 to respond to the script's client. Note that just after migration almost all workloads are in v1beta1 in etcd.

  ```
  Use a specific version ('v1beta1' or 'v1beta2'), `v1beta1` is the default.
  The selected version is used to list all resources, like Workloads, from the API server before updating them. 
  It is preferred to use `v1beta1` when most Workloads are still in `v1beta1` (eg. just after upgrade to Kueue 0.16+), to avoid too many calls to conversion webhooks when the API server lists workloads from etcd. 
  On large deployments listing all workloads and going through conversion may cause timeouts for API
  server when communicating with etcd.
  ```
  wdyt?
