#### What this PR does / why we need it:

For v1beta2:
When running `kubectl get clusterqueue` command, the expected output is:
```
NAME                                        COHORT   PENDING WORKLOADS
clusterqueue.kueue.x-k8s.io/cluster-queue   cohort-1         0
```
But the actual output has cohort name missing:
```
NAME                                        COHORT   PENDING WORKLOADS
clusterqueue.kueue.x-k8s.io/cluster-queue            0
```
This PR fixes this bug.
