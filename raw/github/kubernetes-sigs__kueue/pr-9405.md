## Pull request overview

This PR is a cherry-pick to the release-0.15 branch that fixes a flaky workload retention test. The fix adds a controller-readiness barrier after manager restarts to ensure controllers are processing events before the test proceeds with assertions. Without this barrier, the test could pass based on stale Prometheus metric state, causing the workload deletion check to race ahead of controller startup.

**Changes:**
- Refactored manager lifecycle code into reusable helper functions (startManager, stopManager, restartManager)
- Added a readiness check after manager restart using a probe workload to verify controllers are active
- Ensured the test waits for the LocalQueue status to reflect the probe workload before proceeding
