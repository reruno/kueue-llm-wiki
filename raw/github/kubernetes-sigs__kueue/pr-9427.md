#### What this PR does / why we need it:

When reviewing [KEP-8902](https://github.com/kubernetes-sigs/kueue/pull/9251/changes#diff-af475a18949bdc489d3c2e61e817a3507f1c71f27cb65eb5daad2ff363782c71R257), @sohankunkerkar made a really good point that enabling TAS currently makes the entire ResourceFlavorSpec immutable. This is handled by a CEL validation rule: `!has(oldSelf.topologyName) || self == oldSelf` and was introduced as a conservative measure to protect the TAS topology tree. But it over-approximates: any new field added to the spec (e.g. `fairSharing.resourceWeights`) also becomes immutable, forcing users to delete and recreate the ResourceFlavor to update it.

This PR narrows the immutability to only the fields whose mutation would actually invalidate the TAS topology tree: `nodeLabels`, `tolerations`, and `topologyName`. Each gets its own CEL rule with `has()` guards for absent optional fields and a field-specific error message.

`nodeTaints` is excluded because it is not stored in the TAS cache and is not used to build the topology tree. It is only consumed by the flavor assigner at admission time. Please correct me if I'm wrong.
