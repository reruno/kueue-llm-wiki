#### What this PR does / why we need it:

The fair sharing iterator's admission ordering compares workloads by DRS, Priority and then FIFO. This means a workload that fits within its CQ's nominal quota can be deprioritized because the CQ's aggregate DRS is inflated by borrowing on a different flavor.

The classical iterator already handles this — it prefers workloads with lower borrowing levels before comparing other criteria. This PR adds a similar nominal-first ordering to the fair sharing iterator: workloads that don't require borrowing are always preferred over those that do, before falling through to DRS and other comparisons. 

Not adding a feature gate since it's a simple fix.
