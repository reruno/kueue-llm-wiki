#### What this PR does / why we need it:
We are seeing flake in https://github.com/kubernetes-sigs/kueue/pull/9232#issuecomment-3957576106, where the metric `kueue_admission_cycle_preemption_skips` is unavailable.  This is due to a race where only one workload is considered in the scheduling cycle following the deactivation of blocker workload.

This synchronization ensures that both are considered inadmissible before deactivating the blocker workload, so that they are both considered in the scheduling cycle _after_ the blocking workload is deactivated, and this metric is incremented.

Perhaps related is https://github.com/kubernetes-sigs/kueue/issues/4821, where we identified other parts of this suite being brittle
