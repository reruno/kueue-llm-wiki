#### What this PR does / why we need it:

<img width="1141" height="228" alt="image" src="https://github.com/user-attachments/assets/c86dd775-8d3f-4c90-9dd9-1f1fd88a9686" />

As shown in the image, several ClusterQueues have admission_wait_time_seconds quantiles plateauing at ~2.84 hours, the highest finite bucket boundary (10,240s). Beyond this, histogram_quantile() cannot resolve actual wait times.

This PR adds one additional bucket at 40,960 (~11.3h) to the AdmissionWaitTime histogram, improving observability for clusters with multi-hour queue times.

Since each histogram bucket creates a separate time series per unique label combination, the change is intentionally scoped to a single metric and a 2 additional buckets to minimize the cardinality impact on Prometheus memory, storage, and scrape size.
