#### What this PR does / why we need it:

In a multi-flavor setup with fair sharing enabled, borrowing resources on one flavor can prevent a ClusterQueue from using its nominal quota on a different flavor — both in admission ordering and preemption. The issue is described in detail in #9406.

This PR addresses the preemption case. When a CQ is within nominal quota for the contested flavor, it should be able to preempt workloads even if its aggregate DRS is inflated by borrowing on other flavors. Without this fix, the DRS strategy gates block the preemption despite the preemptor having a legitimate nominal entitlement.


Changes:

1. Extract preemptCandidate helper to avoid duplicating 4 lines across 3 functions.
2. In runFirstFsStrategy, check if the preemptor is within nominal quota for contested resources. If so, bypass the DRS strategy gate and preempt with InCohortReclamationReason. Several existing test cases updated accordingly, as they were actually reclaiming nominal quota.
3. Add FairSharingPreemptWithinNominal feature gate (Beta, enabled by default).
