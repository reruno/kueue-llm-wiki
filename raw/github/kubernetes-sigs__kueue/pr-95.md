## Description

#### What type of PR is this?

/kind cleanup
/kind api-change

#### What this PR does / why we need it:

Capacity not only defines usage limits for a set of tenants, but it is the level at which ordering will be done for workloads submitted to queues sharing a capacity.

"Assigning to a ClusterQueue" no longer makes sense. The correct interpretation is that the QueuedWorkload got admitted by the ClusterQueue. Thus, we group all the admission details in a struct within the QueuedWorkloadSpec.

#### Which issue(s) this PR fixes:

Fixes #85

#### Special notes for your reviewer:

I tried my best to split the 2 renames in different PRs, for reviewer convenience. But I might end up squashing them after review.

I might have missed some renames in some test variables. I think we can fix them in follow ups.
