#### What this PR does / why we need it:
 
The e2e test that ensures that Kueue doesn't suspend the pods created by a StatefulSet in the kube-system namespace when `manageJobsWithoutQueueName=true` is flaky. The test creates a StatefulSet and, once it verifies that its pods are running, it deletes them. However, in some situations the pods are not deleted in time, causing the test to fail. This happens because the pod deletion signal is sent before the agnhost service is ready, causing the actual deletion to take too long.

This PR fixes the issue by making the test wait for the Ready condition instead of Running, ensuring that the service is ready, and by reducing the `TerminationGracePeriod` so that the pod is terminated more quickly.
