## Pull request overview

Cherry-picks upstream fix (#9437) into `release-0.16` to prevent Kueue’s scheduler from issuing duplicate preemption requests/events for the same Workload across scheduling cycles by tracking in-flight preemptions via a shared expectations store between the scheduler and workload controller.

**Changes:**
- Introduce and plumb a shared `expectations.Store` for preemption across scheduler and core workload controller.
- Update scheduler/preemptor logic to skip re-issuing preemptions while an eviction is still “in flight”, and clear expectations when eviction/deletion is observed.
- Update unit/integration/performance test setups to pass the preemption expectations store into controllers and scheduler; add a unit test covering the duplicate-skip behavior.
