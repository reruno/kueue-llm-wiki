## Pull request overview

Cherry-picks upstream fix to prevent Kueue’s scheduler from issuing duplicate preemption requests/events for the same workload across scheduling cycles by introducing an in-flight preemption expectations store shared between scheduler and workload controller.

**Changes:**
- Add a shared `expectations.Store` used to track workloads with in-flight preemptions and skip duplicate preemption issuance.
- Wire the expectations store through `cmd/kueue`, core controller setup, scheduler construction, and test harnesses.
- Add/adjust unit + integration/perf tests to cover and exercise the new expectations behavior.
