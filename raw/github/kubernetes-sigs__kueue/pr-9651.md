### Comment by [@tenzen-y](https://github.com/tenzen-y) — 2026-03-03T17:38:35Z

As we discussed with @mimowo offline, I also think of replacing the current annotation name `kueue.x-k8s.io/safe-to-forcefully-terminate` with `kueue.x-k8s.io/safe-to-forcefully-delete`.

Because I think that the previous and the new recovery controller behaviors are not the same in the following:

- Previous: The controller relies on the kubelet, which means the kubelet tries to send SIGTERM to the container runtime. That is "termination" by kubelet.
- New: The controller does the same thing as "Previous" behavior, but it also issues deletion from etcd forcefully.

I think the "delete" operation is a stronger behavior than "terminate" in this step. 
Hence, I think that `kueue.x-k8s.io/safe-to-forcefully-delete` is more proper for the annotation name.
