## Description

When the scheduler evaluates a workload and gets NoFit from FlavorAssigner, bulk-move all heap workloads with the same scheduling hash to inadmissible. This reduces scheduling cost from O(workloads) to O(equivalence classes), allowing the scheduler to reach schedulable workloads deep in the queue.

## Pull request overview

This PR introduces a “scheduling equivalence class” mechanism to reduce repeated NoFit evaluations in the scheduler by hashing workload scheduling shape and bulk-moving same-hash workloads to inadmissible, plus adding observability and tests.

- When a workload gets `NoFit`, bulk-move same-hash workloads from the CQ heap to inadmissible and track a new metric.
- Add unit/integration tests covering hash behavior and the bulk-move path.

  recordInadmissibleHash moves heap workloads to inadmissible and records the hash unconditionally. To match the PR intent and avoid changing StrictFIFO semantics (where NoFit workloads are typically requeued immediately and should continue blocking), gate this behavior on the ClusterQueue queueingStrategy (e.g., no-op unless BestEffortFIFO).
