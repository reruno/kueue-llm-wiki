<p><em>Sourced from <a href="https://github.com/kubernetes-sigs/kueue/releases">sigs.k8s.io/kueue's releases</a>.</em></p>
<blockquote>
<h2>v0.16.2</h2>
<p>Changes since <code>v0.16.1</code>:</p>
<h2>Changes by Kind</h2>
<h3>Feature</h3>
<ul>
<li>KueueViz Helm: Add podSecurityContext and containerSecurityContext configuration options to KueueViz Helm chart for restricted pod security profile compliance (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9319">#9319</a>, <a href="https://github.com/ziadmoubayed"><code>@​ziadmoubayed</code></a>)</li>
<li>Observability: Increased the maximum finite bucket boundary for admission_wait_time_seconds histogram from ~2.84 hours to ~11.3 hours for better observability of long queue times. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9507">#9507</a>, <a href="https://github.com/mukund-wayve"><code>@​mukund-wayve</code></a>)</li>
</ul>
<h3>Bug or Regression</h3>
<ul>
<li>ElasticJobs: fix the temporary double-counting of quota during workload replacement.
In particular it was causing double-counting of quota requests for unchanged PodSets. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9364">#9364</a>, <a href="https://github.com/benkermani"><code>@​benkermani</code></a>)</li>
<li>FairSharing: workloads fitting within their ClusterQueue's nominal quota are now preferred over workloads that require borrowing, preventing heavy borrowing on one flavor from deprioritizing a CQ's nominal entitlement on another flavor. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9532">#9532</a>, <a href="https://github.com/mukund-wayve"><code>@​mukund-wayve</code></a>)</li>
<li>Fix non-deterministic workload ordering in ClusterQueue by adding UID tie-breaker to queue ordering function. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9140">#9140</a>, <a href="https://github.com/sohankunkerkar"><code>@​sohankunkerkar</code></a>)</li>
<li>Fix serverName substitution in kustomize prometheus ServiceMonitor TLS patch for cert-manager deployments. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9188">#9188</a>, <a href="https://github.com/IrvingMg"><code>@​IrvingMg</code></a>)</li>
<li>Fixed invalid field name in the <code>ClusterQueue</code> printer columns. The &quot;Cohort&quot; column will now correctly display the assigned cohort in kubectl, k9s, and other UI tools instead of being blank. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9422">#9422</a>, <a href="https://github.com/polinasand"><code>@​polinasand</code></a>)</li>
<li>Fixed the bug that prevented managing workloads with duplicated environment variable names in initContainers. This issue manifested when creating the Workload via the API. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9126">#9126</a>, <a href="https://github.com/monabil08"><code>@​monabil08</code></a>)</li>
<li>FlavorFungability: fix the bug that the semantics for the <code>flavorFungability.preference</code> enum values
(ie. PreemptionOverBorrowing and BorrowingOverPreemption) were swapped. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9486">#9486</a>, <a href="https://github.com/tenzen-y"><code>@​tenzen-y</code></a>)</li>
<li>LeaderWorkerSet: fix an occasional race condition resulting in workload deletion getting stuck during scale down. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9135">#9135</a>, <a href="https://github.com/PannagaRao"><code>@​PannagaRao</code></a>)</li>
<li>MultiKueue: Fix a bug that the remote Job object was occasionally left by MultiKueue GC,
even when the corresponding Job object on the management cluster was deleted.
This issue was observed for LeaderWorkerSet. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9310">#9310</a>, <a href="https://github.com/sohankunkerkar"><code>@​sohankunkerkar</code></a>)</li>
<li>MultiKueue: for the StatefulSet integration copy the entire StatefulSet onto the worker clusters. This allows
for proper management (and replacements) of Pods on the worker clusters. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9539">#9539</a>, <a href="https://github.com/IrvingMg"><code>@​IrvingMg</code></a>)</li>
<li>Observability: Fix missing &quot;replica-role&quot; in the logs from the NonTasUsageReconciler. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9456">#9456</a>, <a href="https://github.com/IrvingMg"><code>@​IrvingMg</code></a>)</li>
<li>Observability: Fix the stale &quot;replica-role&quot; value in scheduler logs after leader election. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9431">#9431</a>, <a href="https://github.com/IrvingMg"><code>@​IrvingMg</code></a>)</li>
<li>Scheduling: Fix the bug where inadmissible workloads would be re-queued too frequently at scale.
This resulted in excessive processing, lock contention, and starvation of workloads deeper in the queue.
The fix is to throttle the process with a batch period of 1s per CQ or Cohort. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9490">#9490</a>, <a href="https://github.com/gabesaba"><code>@​gabesaba</code></a>)</li>
<li>TAS: Fix a bug that LeaderWorkerSet with multiple PodTemplates (<code>.spec.leaderWorkerTemplate.leaderTemplate</code> and <code>.spec.leaderWorkerTemplate.workerTemplate</code>), Pod indexes are not correctly evaluated during rank-based ordering assignments. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9368">#9368</a>, <a href="https://github.com/tenzen-y"><code>@​tenzen-y</code></a>)</li>
<li>TAS: fix a bug where NodeHotSwap may assign a Pod, based on rank-ordering, to a node which is already
occupied by another running Pod. (<a href="https://redirect.github.com/kubernetes-sigs/kueue/issues/9282">#9282</a>, <a href="https://github.com/j-skiba"><code>@​j-skiba</code></a>)</li>
</ul>
</blockquote>
