## Pull request overview

This PR propagates a `logr.Logger` into `workload.Info.Update` so callers can emit contextual, high-verbosity debug logs when `Info` is refreshed, without storing any logger/runtime state inside `workload.Info`.

**Changes:**
- Updated `workload.Info.Update` signature to accept a `logr.Logger` and added a V(5) debug log on update.
- Updated queue manager requeue path to pass a context-derived logger into `Info.Update`.

### Reviewed changes

Copilot reviewed 2 out of 2 changed files in this pull request and generated no comments.

| File | Description |
| ---- | ----------- |
| pkg/workload/workload.go | Extends `Info.Update` to accept a logger and emits V(5) structured debug info when updating the workload reference. |
| pkg/cache/queue/manager.go | Passes `ctrl.LoggerFrom(ctx)` into `info.Update` during `RequeueWorkload` so the update log is contextual. |
