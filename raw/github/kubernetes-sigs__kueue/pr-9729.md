## Pull request overview

Cherry-picks upstream change (#9723) into `release-0.15` to allow callers to pass a contextual `logr.Logger` into `workload.Info.Update`, enabling debug-level logging without storing logger state inside `workload.Info`.

**Changes:**
- Extend `workload.Info.Update` to accept a `logr.Logger` and emit a V(5) debug log when updating.
- Update the queue manager requeue path to source the logger from `context.Context` and pass it into `Info.Update`.

### Reviewed changes

Copilot reviewed 2 out of 2 changed files in this pull request and generated no comments.

| File | Description |
| ---- | ----------- |
| pkg/workload/workload.go | Adds `logr.Logger` parameter to `Info.Update` and logs a V(5) debug message when updating the stored workload object. |
| pkg/cache/queue/manager.go | Propagates a contextual logger (`ctrl.LoggerFrom(ctx)`) into `info.Update(...)` during requeue. |
