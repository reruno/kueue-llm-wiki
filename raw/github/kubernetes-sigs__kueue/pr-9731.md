## Pull request overview

This PR extends the `workload.NewInfo` InfoOption pattern to accept a contextual `logr.Logger` via `WithLogger`, and wires that logger through key call sites so construction-time logging (and hash computation logging) uses the caller’s context.

**Changes:**
- Add `WithLogger(logr.Logger)` InfoOption and plumb it into `workload.NewInfo`.
- Switch `computeSchedulingHash` invocation in `NewInfo` to use the provided logger.
- Update scheduler cache and queue manager call sites to pass their contextual logger into `NewInfo`.

### Reviewed changes

| pkg/cache/queue/manager.go | Passes contextual logger (from ctx or argument) into `workload.NewInfo` across workload enqueue/update paths. |
