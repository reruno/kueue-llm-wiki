## Pull request overview

This PR cherry-picks a performance optimization from the main branch (#9698) to the release-0.16 branch. It introduces scheduling equivalence hashing for BestEffortFIFO ClusterQueues, enabling the scheduler to skip redundant evaluation of workloads that have identical scheduling-relevant shapes. When a workload receives a NoFit result from the FlavorAssigner, all other workloads in the same CQ heap with the same scheduling hash are bulk-moved to the inadmissible queue, reducing scheduling cost from O(workloads) to O(equivalence classes).

**Changes:**
- Adds a new `SchedulingEquivalenceHashing` feature gate (Beta, default: true in v0.16) and computes a scheduling hash (`computeSchedulingHash`) on workload `Info` creation/update based on pod spec shape, priority, counts, and topology requests.
- Implements `handleInadmissibleHash` on `ClusterQueue` to bulk-move matching workloads from the heap to inadmissible, with a corresponding `HandleInadmissibleHash` method on the `Manager` and integration into the scheduler's NoFit path.
- Adds comprehensive unit tests for the equivalence hashing behavior and an integration test verifying that a fitting workload behind identical no-fit workloads gets admitted.
