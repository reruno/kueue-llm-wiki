#### What this PR does / why we need it:

Adds `minAdmitDuration` to the FairSharing configuration, gated behind the `FairSharingMinAdmitDuration` feature gate. The feature is alpha and disabled by default. When enabled and configured, workloads admitted less than `minAdmitDuration` ago are excluded from fair sharing preemption.

This helps prevent preemption oscillation, where competing ClusterQueues repeatedly evict each other’s workloads before they make meaningful progress. The issue became more visible as I was working with the historical borrowing strategy KEP in #9737, but this configuration is generic and expected to provide value across Fair Sharing strategies. The behavior is analogous to Slurm’s `PreemptExemptTime`.

This change is conceptually similar to #8646, but operates at a different level. #8646 applies within a single ClusterQueue, whereas `minAdmitDuration` applies across ClusterQueues during fair sharing. The two mechanisms are complementary. This implementation reuses the minimum value of one minute for `minAdmitDuration`, as suggested in #8646.
