## Pull request overview

This PR introduces scheduling-equivalence hashing to optimize BestEffortFIFO scheduling by bulk-moving workloads with an equivalent “NoFit” shape to inadmissible, reducing redundant evaluations.

**Changes:**
- Add `SchedulingEquivalenceHashing` feature gate (default on, Beta in 0.15) and publish it in feature lists.
- Compute and store a per-workload `SchedulingHash` used to identify scheduling equivalence classes.
- Bulk-move same-hash workloads to inadmissible on `NoFit`, plus add integration coverage for the behavior.

  Truncating SHA-256 to 16 hex chars (64 bits) materially increases the chance of collisions.
