  `NewInfo` no longer initializes `SchedulingHash`, leaving it as the empty string. Downstream logic treats any value other than `SchedulingHashUnknown` as a valid hash (e.g., checks like `wInfo.SchedulingHash != workload.SchedulingHashUnknown`), so an empty string can be incorrectly considered “known” and could trigger hash-based behavior (e.g., HandleInadmissibleHash) if a call site forgets to invoke `UpdateSchedulingHash`.
