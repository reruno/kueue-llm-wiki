### Comment by [@VassilisVassiliadis](https://github.com/VassilisVassiliadis) — 2026-03-17T09:40:24Z

@mimowo I replaced most of the integration tests with unit tests to respect the pyramid of tests. 

I added unit tests to the reconcilers (workload_controller, job reconciler, leaderworkerset reconciler, and statefulset reconciler) to test the logic of propagating the AdmissionGatedBy annotation from the Job types to the Workload. These also test that the workload controller correctly sets the QuotaReserved condition and emits events when gates are initially added or removed.

Fir the QuotaReserved condition I went for the simple route of only setting it when the condition is not set and the gate is set. This implies that the QuotaReserved message will not update to reflect removal of gates. I opted for this because it was simpler and because it reduces the chances of 2 parts of Kueue setting the message of QuotaReserved back and forth causing update-spam.

For the integration tests, I kept only the essential ones in `scheduler_test.go` and `workload_controller_test.go` that verify how the code interprets the annnotation to gate/un-gate (including the scenario when the feature is off). I also deleted the helper functions that I was using before and inlined them in the now much fewer integration tests.

I kept the webhook unit tests for all job types to ensure that only valid AdmissionGatedBy annotations reach the reconcilers.

Finally, I added unit tests for parsing and serializing the annotation values for things like invalid values, spaces in different parts of the value, etc.
