## Description

Log the duration of each scheduling phase (heads wait, snapshot, nomination, workload processing) at V(2) to make it easy to identify which phase is slow. A deferred log at cycle end also makes inter-cycle  sleep time visible from log timestamps.

## Pull request overview

Adds verbose (V(2)) boundary logging around the scheduler’s main cycle phases to make it easy to identify where time is being spent (including time blocked waiting for queue heads and time between cycles via timestamps).

**Changes:**
- Log scheduling cycle start and (deferred) completion with total cycle duration.
- Log “heads obtained” including head count and time spent waiting for heads.
- Log per-phase durations for snapshotting, nomination, and workload processing.
