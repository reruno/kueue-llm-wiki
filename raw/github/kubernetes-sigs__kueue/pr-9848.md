<blockquote>
<h2>v1.20.0</h2>
<p>cert-manager is the easiest way to automatically manage certificates in Kubernetes and OpenShift clusters.</p>
<p>v1.20.0 adds alpha support for the new ListenerSet resource, adds support for Azure Private DNS; parentRefs are no longer required when using ACME with Gateway API, and OtherNames was promoted to Beta.</p>
<h2>Changes by Kind</h2>
<h3>Feature</h3>
<ul>
<li>Added a set of flags to permit setting NetworkPolicy across all deployed containers. Remove redundant global IP ranges from example policies. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8370">#8370</a>, <a href="https://github.com/jcpunk"><code>@​jcpunk</code></a>)</li>
<li>Added selectable fields to custom resource definitions for .spec.issuerRef.{group, kind, name} (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8256">#8256</a>, <a href="https://github.com/tareksha"><code>@​tareksha</code></a>)</li>
<li>Added support for specifying <code>imagePullSecrets</code> in the <code>startupapicheck-job</code> Helm template to enable pulling images from private registries. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8186">#8186</a>, <a href="https://github.com/mathieu-clnk"><code>@​mathieu-clnk</code></a>)</li>
<li>Added 'extraContainers' helm chart value, allowing the deployment of arbitrary sidecar containers within the cert-manager operator pod. This can be used to support, for e.g., AWS IAM Roles Anywhere for Route53 DNS01 verification. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8355">#8355</a>, <a href="https://github.com/dancmeyers"><code>@​dancmeyers</code></a>)</li>
<li>Added <code>parentRef</code> override annotations on the Certificate resource. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8518">#8518</a>, <a href="https://github.com/hjoshi123"><code>@​hjoshi123</code></a>)</li>
<li>Added support for azure private zones for dns01 issuer. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8494">#8494</a>, <a href="https://github.com/hjoshi123"><code>@​hjoshi123</code></a>)</li>
<li>Added support for configuring PEM decoding size limits, allowing operators to handle larger certificates and keys. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7642">#7642</a>, <a href="https://github.com/robertlestak"><code>@​robertlestak</code></a>)</li>
<li>Added support for unhealthyPodEvictionPolicy in PodDisruptionBudget (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/7728">#7728</a>, <a href="https://github.com/jcpunk"><code>@​jcpunk</code></a>)</li>
<li>For Venafi provider, read <code>venafi.cert-manager.io/custom-fields</code> annotation on Issuer/ClusterIssuer and use it as base with override/append capabilities on Certificate level. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8301">#8301</a>, <a href="https://github.com/k0da"><code>@​k0da</code></a>)</li>
<li>Improve error message when CA issuers are misconfigured to use a clashing secret name (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8374">#8374</a>, <a href="https://github.com/majiayu000"><code>@​majiayu000</code></a>)</li>
<li>Introduce a new Ingress annotation <code>acme.cert-manager.io/http01-ingress-ingressclassname</code> to override <code>http01.ingress.ingressClassName</code> field in HTTP-01 challenge solvers. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8244">#8244</a>, <a href="https://github.com/lunarwhite"><code>@​lunarwhite</code></a>)</li>
<li>Update <code>global.nodeSelector</code> to helm chart to perform a <code>merge</code> and allow for a single <code>nodeSelector</code> to be set across all services. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8195">#8195</a>, <a href="https://github.com/StingRayZA"><code>@​StingRayZA</code></a>)</li>
<li>Vault issuers will now include the Vault server address as one of the default audiences on generated service account tokens. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8228">#8228</a>, <a href="https://github.com/terinjokes"><code>@​terinjokes</code></a>)</li>
<li>Added experimental <code>XListenerSets</code> feature gate (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8394">#8394</a>, <a href="https://github.com/hjoshi123"><code>@​hjoshi123</code></a>)</li>
</ul>
<h3>Documentation</h3>
<ul>
<li>Add GWAPI documentation to NOTES.TXT in helm chart (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8353">#8353</a>, <a href="https://github.com/jaxels10"><code>@​jaxels10</code></a>)</li>
</ul>
<h3>Bug or Regression</h3>
<ul>
<li>Adds logs for cases when acme server returns us a fatal error in the order controller (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8199">#8199</a>, <a href="https://github.com/Peac36"><code>@​Peac36</code></a>)</li>
<li>Fixed an issue where kind or group in the issuerRef of a Certificate was omitted, upgrading to 1.19.x incorrectly caused the certificate to be renewed (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8160">#8160</a>, <a href="https://github.com/inteon"><code>@​inteon</code></a>)</li>
<li>Changes to the Duration and RenewBefore annotations on ingress and gateway-api resources will now trigger certificate updates. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8232">#8232</a>, <a href="https://github.com/eleanor-merry"><code>@​eleanor-merry</code></a>)</li>
<li>Fix an issue where ACME challenge TXT records are not cleaned up when there are many resource records in CloudDNS. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8456">#8456</a>, <a href="https://github.com/tkna"><code>@​tkna</code></a>)</li>
<li>Fix unregulated retries with the DigitalOcean DNS-01 solver
Add full detailed DNS-01 errors to the events attached to the Challenge, for easier debugging (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8221">#8221</a>, <a href="https://github.com/wallrj-cyberark"><code>@​wallrj-cyberark</code></a>)</li>
<li>Fixed an infinite re-issuance loop that could occur when an issuer returns a certificate with a public key that doesn't match the CSR. The issuing controller now validates the certificate before storing it and fails with backoff on mismatch. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8403">#8403</a>, <a href="https://github.com/calm329"><code>@​calm329</code></a>)</li>
<li>Fixed an issue where HTTP-01 challenges failed when the Host header contains an IPv6 address. This means that users can now issue IP address certificates for IPv6 address subjects. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8424">#8424</a>, <a href="https://github.com/SlashNephy"><code>@​SlashNephy</code></a>)</li>
<li>Fixed the HTTP-01 Gateway solver creating invalid HTTPRoutes by not setting spec.hostnames when the challenge DNSName is an IP address. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8443">#8443</a>, <a href="https://github.com/alviss7"><code>@​alviss7</code></a>)</li>
<li>Revert API defaults for issuer reference kind and group introduced in 0.19.0 (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8173">#8173</a>, <a href="https://github.com/erikgb"><code>@​erikgb</code></a>)</li>
<li>Security (MODERATE): Fix a potential panic in the cert-manager controller when a DNS response in an unexpected order was cached. If an attacker was able to modify DNS responses (or if they controlled the DNS server) it was possible to cause denial of service for the cert-manager controller. (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8469">#8469</a>, <a href="https://github.com/SgtCoDFish"><code>@​SgtCoDFish</code></a>)</li>
<li>Update Go to <code>v1.25.5</code> to fix <code>CVE-2025-61727</code> and <code>CVE-2025-61729</code> (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8290">#8290</a>, <a href="https://github.com/octo-sts"><code>@​octo-sts</code></a>[bot])</li>
<li>When Prometheus monitoring is enabled, the metrics label is now set to the intended value of <code>cert-manager</code>. Previously, it was set depending on various factors (namespace cert-manager is installed in and/or Helm release name). (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8162">#8162</a>, <a href="https://github.com/LiquidPL"><code>@​LiquidPL</code></a>)</li>
</ul>
<h3>Other (Cleanup or Flake)</h3>
<ul>
<li>Promoted the OtherNames feature to Beta and enabled it by default (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8288">#8288</a>, <a href="https://github.com/wallrj-cyberark"><code>@​wallrj-cyberark</code></a>)</li>
<li>Promoting <code>XListenerSets</code> feature gate to <code>ListenerSets</code> (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8501">#8501</a>, <a href="https://github.com/hjoshi123"><code>@​hjoshi123</code></a>)</li>
<li>Rebranding of the Venafi Issuer to CyberArk (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8215">#8215</a>, <a href="https://github.com/iossifbenbassat123"><code>@​iossifbenbassat123</code></a>)</li>
<li>Switched to SSA for challenge finalizer updates (<a href="https://redirect.github.com/cert-manager/cert-manager/issues/8519">#8519</a>, <a href="https://github.com/inteon"><code>@​inteon</code></a>)</li>
</ul>
