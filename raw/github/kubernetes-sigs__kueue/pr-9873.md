#### What this PR does / why we need it:
When CQ nominalQuota is set to "1E" (10^18) for CPU, MilliValue() overflows int64 (10^21 > MaxInt64) and silently returns 0, causing workloads to fail with "maximum capacity (0)".

Cap overflowed values at math.MaxInt64 instead of returning 0.
