#### What this PR does / why we need it:

Adds WithinClusterQueueConfig to ClusterQueuePreemption with two duration fields:

A workload is "opportunistic" if the preemptor was waiting in the queue before it got admitted. Otherwise it's "incumbent".

`minAdmitDuration` -- how long an incumbent workload (one that was queued before the preemptor) must run before it can be preempted.

`opportunisticMinAdmitDuration` -- how long an opportunistic workload (one that was queued after the preemptor but got admitted first, e.g. via BestEffortFIFO) must run before it can be preempted.

This is not a new preemption policy. It adds a time check on top of both `LowerPriority` and `LowerOrNewerEqualPriority`. Each policy first decides if a candidate can be preempted using its existing rules (priority, queue order). If yes, the time check runs as a final step: if the candidate hasn't run long enough, preemption is blocked. This works the same way for both policies, making it a **generic** addition.

Both durations must be at least 1 minute. The config can be used with any withinClusterQueue policy except Never.
