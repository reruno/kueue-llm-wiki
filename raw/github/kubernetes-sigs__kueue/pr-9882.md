## What this PR does / why we need it

This is a **preparatory PR** for adding **RayService MultiKueue support** (#9439, follow-up in #9449).

It introduces two main changes:

### 1. Refactor Ray MultiKueue adapters

Refactors the **RayCluster** and **RayJob** MultiKueue adapters to use a shared generic adapter:

```
ray.NewMKAdapter()
```

This eliminates duplicated code between the adapters.

* Each adapter is reduced from **~124 lines → ~51 lines**
* Logic is centralized in a reusable generic adapter

The shared adapter lives in:

```
pkg/controller/jobs/ray/ray_multikueue_adapter.go
```

It follows the same design pattern used by the Kubeflow adapter:

```
kubeflowjob.NewMKAdapter()
```

The implementation uses **Go generics** with callback functions for type-specific operations:

* `copySpec`
* `copyStatus`
* `emptyList`
* `gvk`
* `getManagedBy`
* `setManagedBy`

This refactoring was suggested by **@mimowo** during the review of #9449:

[https://github.com/kubernetes-sigs/kueue/pull/9449#discussion_r1977458832](https://github.com/kubernetes-sigs/kueue/pull/9449#discussion_r1977458832)

---

### 2. Bump kuberay dependency

Updates **kuberay** from:

```
v1.5.1 → v1.6.0-rc.0
```

This version introduces the `ManagedBy` field in `RayServiceSpec`, which is required for **RayService MultiKueue support**.

## Additional context

* **No behavioral changes** — this PR is purely a **refactoring** plus a **dependency bump**.
* All existing **MultiKueue adapter tests pass**.
* The kuberay **RC0 bump** is required because the full `v1.6.0` release is expected around **March 15**.

The RC0 bump was requested by **@mimowo**:

[https://github.com/kubernetes-sigs/kueue/pull/9449#issuecomment-2715668268](https://github.com/kubernetes-sigs/kueue/pull/9449#issuecomment-2715668268)

See also **#9816**, which validates that the RC0 bump works with all CI.

The follow-up PR (**#9449**) will add the **RayService MultiKueue adapter** using this shared infrastructure, resulting in a much **smaller and more focused diff**.

**E2E tests** for RayService MultiKueue will be added in a separate follow-up PR (agreed with **@hiboyang**).
