## What this PR does / why we need it

Bumps the kuberay dependency from `v1.5.1` to `v1.6.0-rc.0`.

This is needed because `v1.6.0-rc.0` adds the `ManagedBy` field to `RayServiceSpec`, which is a prerequisite for RayService MultiKueue support (#9439).

The RC0 bump was requested by @mimowo in https://github.com/kubernetes-sigs/kueue/pull/9449#issuecomment-2715668268. See also #9816 which validates the RC0 works with all CI.

Split from #9882 per @mimowo's feedback. The refactoring is in a separate PR.

### New transitive dependencies:
- `github.com/hashicorp/golang-lru/v2` v2.0.7
- `github.com/robfig/cron/v3` v3.0.1
- `github.com/smallnest/chanx` v1.2.0

### Upstream reference:
- https://github.com/ray-project/kuberay/issues/4486 (ManagedBy support for RayService — merged, RC0 released)
- Full v1.6.0 release expected ~March 15, can switch from RC to final when available
