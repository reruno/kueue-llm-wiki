## Pull request overview

This PR addresses non-deterministic ordering in `ClusterQueue.Snapshot()` when `UsageBasedAdmissionFairSharing` is enabled by ensuring fair-sharing comparisons don’t observe concurrently mutating AFS state mid-sort.

**Changes:**
- Refactors FS usage computation to a helper that iterates resources deterministically.
- Updates `Snapshot()` sorting to precompute per-LocalQueue FS usage from deep-copied AFS state before sorting.
- Adds a concurrency-focused regression test for snapshot ordering stability.
