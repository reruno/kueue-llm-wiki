#### What this PR does / why we need it:

Adds an e2e regression test covering the two-phase StatefulSet pattern
described in issue 9830. The test:

1. Creates a StatefulSet with `replicas=0` and a non-existent queue
2. Scales up to 1 replica (triggering webhook annotation injection and
   a controller revision mismatch)
3. Changes the queue label to a valid LocalQueue
4. Verifies the pod queue label is updated, the workload queue name is
   updated, the replica becomes ready, and the workload is admitted

#### Which issue(s) this PR fixes:

Fixes #9830

#### Special notes for your reviewer:

This test is expected to fail until the underlying bug is fixed. It
reproduces the exact scenario from the issue where the pod keeps the
old queue label after a queue change on a StatefulSet that was scaled
up from zero replicas.
