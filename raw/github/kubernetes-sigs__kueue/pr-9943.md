### Comment by [@mykysha](https://github.com/mykysha) — 2026-03-18T15:14:05Z

The failing tests uncovered an interesting case where we use kueue.WorkloadEvictedByPodsReadyTimeout reason with the underlyingCause, which means that as the reason is not Deactivated, the underlyingCause is lost before the controller Update function gets the Eviction condition.

So as it looks now, there is more work to be done to simplify eviction reporting.

I propose to use the current, non-optimized version of Evict in the local queue metrics configuration in #9371 , and come back to this PR afterwards.
@mbobrovskyi wdyt?
