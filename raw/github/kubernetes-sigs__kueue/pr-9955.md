## Pull request overview

Cherry-pick to release-0.16 that fixes non-deterministic `ClusterQueue.Snapshot()` ordering when `UsageBasedAdmissionFairSharing` is enabled by avoiding inconsistent comparisons during a single sort.

**Changes:**
- Make FS usage computation deterministic by iterating resources in sorted key order.
- Refactor queue ordering to separate a base comparator from fair-sharing logic, and introduce a dedicated Snapshot sort that pre-computes per-LocalQueue FS usage from point-in-time AFS state.
- Add a regression test that stresses Snapshot ordering under concurrent fair-sharing penalty updates.
