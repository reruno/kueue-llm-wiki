## Pull request overview

Cherry-pick to release-0.15 that fixes non-deterministic `ClusterQueue.Snapshot()` ordering when `UsageBasedAdmissionFairSharing` is enabled by ensuring sort comparisons are consistent/deterministic even under concurrent AFS state updates.

**Changes:**
- Precompute Admission FairSharing (AFS) usage per `LocalQueue` from a point-in-time copy of AFS state before sorting snapshots.
- Make FS usage computation deterministic by iterating resource keys in sorted order.
- Add a regression test that exercises `Snapshot()` stability while penalties are concurrently updated.

  `CalcFSUsageFromResources` divides by `lqWeight` unconditionally. When a LocalQueue has FairSharing weight set to 0 (which is allowed and documented as implying an infinite share/disadvantage) and the summed usage is also 0, this returns `NaN` (0/0). `cmp.Compare` treats NaN as equal, so a zero-weight queue could be ordered as if it had normal share instead of always being disadvantaged. Handle `lqWeight == 0` explicitly (e.g., return +Inf/MaxFloat) so zero-weight always sorts after non-zero weight, even when usage is zero.
