## Pull request overview

This PR is a small cleanup/refactor in the workload handling path to centralize the “should this DRA workload be handled via Reconcile?” decision into a single helper, reducing duplicated feature-gate + DRA detection logic across controllers and queue management.

**Changes:**
- Added `workload.NeedsDRAReconcile(*kueue.Workload)` helper (feature-gated by `DynamicResourceAllocation`).
- Replaced duplicated `features.Enabled(DynamicResourceAllocation) && workload.HasDRA(...)` checks with the new helper in controller event handling and queue manager logic.

### Reviewed changes

Copilot reviewed 3 out of 3 changed files in this pull request and generated no comments.

| File | Description |
| ---- | ----------- |
| pkg/workload/workload.go | Introduces `NeedsDRAReconcile` helper to encapsulate the DRA + feature gate check. |
| pkg/controller/core/workload_controller.go | Uses `NeedsDRAReconcile` in Create/Update event handlers to skip queue ops for DRA workloads. |
| pkg/cache/queue/manager.go | Uses `NeedsDRAReconcile` when processing existing workloads during LocalQueue creation. |
